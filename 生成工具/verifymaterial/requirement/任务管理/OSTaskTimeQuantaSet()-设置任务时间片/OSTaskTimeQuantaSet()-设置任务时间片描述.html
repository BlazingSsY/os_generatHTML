需求描述

需求编号：HLR_OS_01_18

工作描述：HLR：uC/OS-Ⅲ应提供设置任务时间片长度的接口。

是否需求：是

验证方法：测试验证

补充信息：

  1. 函数名称
  2. OSTaskTimeQuantaSet( ) – 设置任务时间片
  3. 函数框架
  void OSTaskTimeQuantaSet(
     OS_TCB p_tcb             /*in: 指向任务控制块指针 */
     OS_TICK time_quanta      /*in: 时间片长度（数量） */
     OS_ERR  p_err            /*out: 错误码指针 */
     )
  1. 函数描述
  2. 该函数用于设置任务时间片长度，以供轮转调度算法使用
  3. 参数说明

（1）p_tcb

类型：OS_TCB*

作用：指向要修改时间片的任务的任务控制块（TCB）。

（2）time_quanta

类型：OS_TICK

作用：要设置的时间片长度，也可以说是时间片数量，单位是OS_TICK，即系统的时钟节拍

（3）p_err

类型：OS_ERR*

作用：返回函数调用的结果状态码。

 返回码：

OS_ERR_NONE - 成功设置任务时间片
OS_ERR_SET_ISR - 在中断中设置任务时间片
  1. 函数算法逻辑

 （1）检查是否在 ISR 中调用

    当OS_CFG_CALLED_FROM_ISR_CHK_EN功能启用时，检查是否在中断服务程序（ISR）中调用 OSTaskTimeQuantaSet()，若是，则返回错误码OS_ERR_SET_ISR。

（2）检查输入参数p_tcb

    进入临界区，如果输入参数p_tcb为空，则将p_tcb指向当前任务

（3）检查输入参数time_quanta

     如果输入参数time_quanta为0，使用OSSchedRoundRobinDfltTimeQuanta设置任务时间片长度p_tcb->TimeQuanta；

   如果输入参数time_quanta不为0，则使用输入参数time_quanta设置任务时间片长度p_tcb->TimeQuanta；

（4）检查新设置的任务时间片长度是否大于任务剩余时间片长度

  如果新设置的任务时间片长度p_tcb->TimeQuanta大于任务的剩余时间片长度p_tcb->TimeQuantaCtr，将任务剩余时间片长度更新为新设置的任务时间片长度p_tcb->TimeQuanta；

（5）设置错误码

   退出临界区，设置错误码为OS_ERR_NONE

  1. 返回值
  2. 无
  3. 健康管理错误码
  4. 无
