需求描述

需求编号：HLR_OS_01_22

工作描述：HLR：uC/OS-Ⅲ应提供更改任务优先级接口。

是否需求：是

验证方法：测试验证

补充信息：

  1. 函数名称
  2. OS_TaskChangePrio() - 更改任务的优先级。
  3. 函数框架
void OS_TaskChangePrio(
OS_TCB *p_tcb, /*in: 指向要更改优先级的任务的任务控制块 */
OS_PRIO prio_new /*in: 新的任务优先级 */
);
  1. 函数描述

   该函数用于更改任务的优先级。它是由内核调用来实现任务优先级的实际变更操作，并根据需要更新优先级继承等情况。

  1. 参数说明

（1）p_tcb

类型：OS_TCB*

作用：指向要更改优先级的任务的任务控制块（TCB）。

注意事项：

    如果任务不存在或无效，函数的行为是未定义的。

（2）prio_new

类型：OS_PRIO

作用：指定任务的新优先级。

  1. 函数算法逻辑

（1）

• 初始化 局部变量p_tcb_owner 为 NULL。

• 如果启用互斥量功能（OS_CFG_MUTEX_EN > 0u），将当前任务的优先级（p_tcb->Prio）保存到变量 prio_cur 中。

• 根据任务的当前状态（p_tcb->TaskState）进行不同处理：

    • • 就绪状态（OS_TASK_STATE_RDY）：

         • • • 调用OS_RdyListRemove函数从当前的就绪列表中移除任务。

         • • • 更新任务的优先级p_tcb->Prio为 prio_new。

         • • • 调用OS_PrioInsert函数将任务的优先级插入优先级表中

         • • • 判断任务是否是当前任务（OSTCBCurPtr）：

             • • • • 如果是，调用OS_RdyListInsertHead函数将任务插入就绪列表的头部。

             • • • • 如果不是，调用OS_RdyListInsertTail函数将任务插入就绪列表的尾部。

    • • 延迟、挂起或延迟挂起状态（OS_TASK_STATE_DLY、OS_TASK_STATE_SUSPENDED、OS_TASK_STATE_DLY_SUSPENDED）：

        • • • 直接更新任务的优先级为 prio_new，无需其他操作。

    • • Pend 状态（OS_TASK_STATE_PEND、OS_TASK_STATE_PEND_TIMEOUT、OS_TASK_STATE_PEND_SUSPENDED、OS_TASK_STATE_PEND_TIMEOUT_SUSPENDED）：

        • • • 更新任务的优先级为 prio_new。

        • • • 根据任务 Pend 的对象类型进行不同的处理：

            • • • • 如果 Pend对象为OS_TASK_PEND_ON_FLAG、OS_TASK_PEND_ON_Q、OS_TASK_PEND_ON_SEM，调用 OS_PendListChangePrio 函数更新 Pend 列表的优先级。

            • • • • 如果 Pend 对象为互斥量（OS_TASK_PEND_ON_MUTEX）， 如果没有启用了互斥量功能（OS_CFG_MUTEX_EN > 0u），则不做任何处理，反之，则：

                • • • • • 调用 OS_PendListChangePrio 函数更新 Pend 列表中的任务优先级

                • • • • • 获取互斥量的持有者任务的 TCB 指针， 保存至变量p_tcb_owner。

                • • • • • 如果当前任务的优先级提高（prio_cur > prio_new）：

                    • • • • • • 如果持有互斥量的任务的优先级低于新优先级（p_tcb_owner->Prio <= prio_new），不需要调整互斥量持有者的优先级，p_tcb_owner置为NULL，反之则执行"（2）更改任务指针"

                • • • • • 如果当前任务的优先级降低（prio_cur <= prio_new）：

                    • • • • • • 如果持有互斥量的任务的优先级等于当前任务优先级（p_tcb_owner->Prio == prio_cur）：。

                        • • • • • • • 调用 OS_MutexGrpPrioFindHighest 函数，获取互斥量组的最高优先级，将其设置为新优先级prio_new。

                        • • • • • • • 如果新优先级高于互斥量的基础优先级（prio_new > p_tcb_owner->BasePrio），则将其设置为基优先级（p_tcb_owner->BasePrio）。

                        • • • • • • • 如果新优先级等于互斥量持有者的当前优先级（prio_new == p_tcb_owner->Prio），说明不需要调整优先级，将 p_tcb_owner 设置为 NULL，反之则执行"（2）更改任务指针"

                    • • • • • •如果持有互斥量的任务的优先级不等于当前任务优先级（p_tcb_owner->Prio != prio_cur），不需要调整持有者的优先级，p_tcb_owner置为NULL。

            • • • • 其他 Pend 对象：OS_TASK_PEND_ON_TASK_Q、OS_TASK_PEND_ON_TASK_SEM、其他无效目标，则不做处理。

    • • 无效状态：直接返回。

（2）更改任务指针

    更改任务指针为 p_tcb_owner，如果任务指针不为空，循环执行“步骤1”，如果为空，函数返回

  1. 返回值

    无

  1. 健康管理错误码

    无

备注：此函数是 uC/OS-III 的内部函数，应用程序不应直接调用它。
