需求描述

需求编号：HLR_OS_01_15

工作描述：HLR：uC/OS-Ⅲ应提供设置任务信号量的接口。

是否需求：是

验证方法：测试验证

补充信息：

  1. 函数名称
  2. OSTaskSemSet( ) -设置任务信号量
  3. 函数框架
  void OSTaskSemPost(
OS_TCB *p_tcb, / in: 指向目标任务的任务控制块（TCB） */
OS_SEM_CTR cnt, / in: 设置给指定任务的任务信号量数量 */
OS_ERR *p_err / out: 错误码指针 */
);
  1. 函数描述

该函数用于设置任务的任务信号量数量。如果目标任务正处于等待任务信号量的状态，则返回错误码OS_ERR_TASK_WAITING，返回值为0， 否则，获取目标任务信号量计数器的值作为返回值，然后更新目标任务的任务计数器的值为输入的信号量数量，返回码为OS_ERR_NONE。

  1. 参数说明

（1）p_tcb

类型：OS_TCB*

作用：指向目标任务的任务控制块（TCB）。

注意事项：

    如果 p_tcb 为 NULL，将设置当前任务的信号量数量。

（2）cnt

类型：OS_SEM_CTR

作用：任务信号量计数器的值，表示要设置给目标任务的任务信号量数量

（3）p_err

类型：OS_ERR*

作用：返回函数调用的结果状态码。

返回代码：

    OS_ERR_NONE：任务信号量数量设置成功。
    OS_ERR_SET_ISR: 当前任务处于中断中
    OS_ERR_TASK_WAITING: 当前任务正在等待任务信号量
  1. 函数算法逻辑

（1）中断检查

     当OS_CFG_CALLED_FROM_ISR_CHK_EN功能启用时，检查OSIntNestingCtr是否大于0，如果大于0，表明当前处于中断中，返回错误码OS_ERR_SET_ISR，返回值为0

（2）检查输入参数p_tcb

    进入临界区，如果 输入的参数p_tcb 为 NULL，则将其指向当前任务。

（3）检查目标任务是否正在等待任务信号量

    如果目标任务处于等待态，等待超时态，等待挂起态，等待超时挂起态这四种状态中的一种，且其等待的事件为任务信号量，则退出临界区，设置错误码为OS_ERR_TASK_WAITING，返回值为0

（4）更新目标任务的任务信号量计数器

    获取目标任务的信号量计数器的值，作为返回值，更新目标任务的任务信号量计数器的值为传入的信号量数量，退出临界区，返回码设置为OS_ERR_NONE


  1. 返回值

类型：OS_SEM_CTR

作用：目标任务的任务信号量计数器的原始值

  1. 健康管理错误码
  2. 无
