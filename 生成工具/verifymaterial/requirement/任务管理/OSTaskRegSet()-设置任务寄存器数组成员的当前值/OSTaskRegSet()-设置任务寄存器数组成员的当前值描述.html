需求描述

需求编号：HLR_OS_01_10

工作描述：HLR：uC/OS-Ⅲ应提供设置任务寄存器数组元素的当前值接口。

是否需求：是

验证方法：测试验证

补充信息：

  1. 函数名称
  2. OSTaskRegSet() - 设置任务寄存器数组元素的当前值
  3. 函数框架
void OSTaskRegSet (
OS_TCB *p_tcb /* 任务控制块指针*/
OS_REG_ID id /* 任务寄存器数组元素的标识符*/
OS_REG value  /* 要设置的寄存器新值*/
OS_ERR *p_err /* 返回码指针*/
)
  1. 函数描述

该函数用于设置任务寄存器数组指定元素的当前值。任务寄存器是应用程序特定的，可以存储任务特定的值

  1. 参数说明

（（1）p_tcb

    类型：OS_TCB*

    作用：指向要接收消息的任务的任务控制块（TCB）。如果传入 NULL， 将指向当前任务。

（2）id

    类型：OS_REG_ID

    作用： 数组元素标识符，用于指定要设置的寄存器数组元素。

（3）value

    类型：OS_REG

    作用：用于设置寄存器数组元素的新值

（4）p_err

    类型：OS_ERR*

    作用：返回码，用于保存返回函数执行结果状态。

返回代码：

OS_ERR_NONE：成功设置。
    OS_ERR_REG_ID_INVALID：标识符无效。
  1. 函数算法逻辑

（1）检查输入参数id：

    当OS_CFG_ARG_CHK_EN功能启用时，如果输入的参数id大于等OS_CFG_TASK_REG_TBL_SIZE，返回错误码OS_ERR_REG_ID_INVALID

（2）检查输入参数p_tcb

进入临界区，如果输入参数p_tcb为空，则将其指向当前任务

（3）使用value和id设置目标任务的寄存器数组元素的值

使用输入的value和id设置目标任务的寄存器数组元素的值，退出临界区，设置错误码为OS_ERR_NONE

  1. 返回值

    无

  1. 健康管理错误码

    无
