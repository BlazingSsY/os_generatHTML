需求描述

需求编号：HLR_OS_01_07

工作描述：HLR：uC/OS-Ⅲ应提供向任务发送任务消息接口。

是否需求：是

验证方法：测试验证

补充信息：

  1. 函数名称
  2. OSTaskQPost( ) -向任务发送任务消息
  3. 函数框架
  void OSTaskQPost(
OS_TCB *p_tcb, / in: 指向接收消息的任务的任务控制块（TCB） */
void *p_void, / in: 指向要发送的消息 /
OS_MSG_SIZE msg_size,/ in: 消息的大小（以字节为单位） /
OS_OPT opt, / in: 发送消息时的选项 */
OS_ERR *p_err / out: 错误码指针 */
);
  1. 函数描述

该函数用于向任务发送任务消息。如果指定的任务正在等待消息，该消息将立即传递给任务。否则，消息将存储在任务的消息队列中，直到任务调用 OSTaskQPend 时接收消息。

  1. 参数说明

（1）p_tcb

类型：OS_TCB*

作用：指向接收消息的任务的任务控制块（TCB）。

注意事项：

    如果 p_tcb 为 NULL，消息将发送到调用任务自己的消息队列中。

（2）p_void

类型：void*

作用：指向要发送的消息。

（3）msg_size

类型：OS_MSG_SIZE

作用：指定消息的大小（以字节为单位）。

（4）opt

类型：OS_OPT

作用：指定发送消息的行为选项。

取值范围：

    OS_OPT_POST_FIFO：消息将添加到队列的末尾（FIFO 模式）。

    OS_OPT_POST_LIFO：消息将添加到队列的前端（LIFO 模式）。

    OS_OPT_POST_NO_SCHED：不调用调度程序，需手动调用。

注意事项：

    无效的选项将导致 OS_ERR_OPT_INVALID 错误。

（5）p_err

类型：OS_ERR*

作用：返回函数调用的结果状态码。

返回代码：

    OS_ERR_NONE：消息成功发送。
    OS_ERR_MSG_POOL_EMPTY：消息池中没有可用的消息缓冲区。
    OS_ERR_OPT_INVALID：无效的 opt 参数。
    OS_ERR_OS_NOT_RUNNING：uC/OS-III 尚未运行。
    OS_ERR_Q_MAX：消息队列已满。
    OS_ERR_STATE_INVALID：任务状态无效。
  1. 函数算法逻辑

（1）内核运行状态检查

    OS_CFG_INVALID_OS_CALLS_CHK_EN功能启用时，检查OSRunning是否不等于OS_STATE_OS_RUNNING。若不等于，则说明当前内核处于停止态，返回错误码OS_ERR_OS_NOT_RUNNING

（2）参数合法性检查

    OS_CFG_ARG_CHK_EN功能启用时，如果 opt 参数不是 OS_OPT_POST_FIFO、OS_OPT_POST_LIFO 或它们与 OS_OPT_POST_NO_SCHED 的组合，返回 OS_ERR_OPT_INVALID。

  (3)  检查输入参数p_tcb

    进入临界区，检测 输入参数p_tcb指针，如果其为空，则将p_tcb指向当前任务

  (4)如果目标任务处于就绪、延迟、挂起、延迟挂起态四种之一，调用OS_MsgQPut函数将消息添加到目标任务的任务消息队列中，然后退出临界区。

  (5）如果目标任务处于等待态、等待超时态、等待挂起态、等待超时挂起态四种之一且等待的事件为任务消息队列，则直接将消息传递给任务并退出临界区，此时如果输入参数opt不包含OS_OPT_POST_NO_SCHED，将执行任务调度

  (6)如果目标任务处于等待态、等待超时态、等待挂起态、等待超时挂起态四种之一且等待的事件非任务消息队列，则调用OS_MsgQPut函数将消息添加到目标任务的任务消息队列中并退出临界区

  (7)如果任务状态无效，退出临界区，返回 OS_ERR_STATE_INVALID。

  2. 无
  3. 健康管理错误码
  4. 无
