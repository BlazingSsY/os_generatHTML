需求描述

需求编号：HLR_OS_01_17

工作描述：HLR：uC/OS-Ⅲ应提供挂起任务的接口。

是否需求：是

验证方法：测试验证

补充信息：

  1. 函数名称
  2. OSTaskSuspend( ) – 挂起任务
  3. 函数框架
  void OSTaskSuspend(
  OS_TCB *p_tcb    /*in: 任务控制块指针*/
  OS_ERR *p_err    /*out: 错误码指针*/
)
  1. 函数描述
  2. 该函数用于挂起一个任务，使其不再参与系统的调度和执行
  3. 参数说明

（1）p_tcb

类型：OS_TCB*

作用：指向要挂起的任务的任务控制块（TCB）。

注意事项：

    在调度器锁定的情况下，不能挂起“自我”（即当前任务）。

（2）p_err

类型：OS_ERR*

作用：返回函数调用的结果状态码。

返回码：

  OS_ERR_NONE - 成功挂起目标任务
  OS_ERR_OS_NOT_RUNNING - uC/OS-III 尚未运行
  OS_ERR_SCHED_LOCKED - 在任务调度被锁定的情况下挂起当前任务
  OS_ERR_STATE_INVALID - 目标任务处于无效状态
  OS_ERR_TASK_SUSPEND_CTR_OVF - 目标任务挂起计数器异常
  OS_ERR_TASK_SUSPEND_ISR -  从中断服务程序中挂起任务
  OS_ERR_TASK_SUSPEND_IDLE -  尝试挂起空闲任务
  1. 函数算法逻辑

 （1）检查是否在 ISR 中调用

    当OS_CFG_CALLED_FROM_ISR_CHK_EN功能启用时，检查是否在中断中调用，如果当前在中断服务程序（ISR）中调用 OSTaskResume()，则返回错误码OS_ERR_TASK_SUSPEND_ISR。

（2）检查是否挂起空闲任务

    当OS_CFG_TASK_IDLE_EN功能启用时，检查要挂起的目标任务是否为空闲任务，若是，则返回错误码OS_ERR_TASK_SUSPEND_IDLE

（3）检查输入参数p_tcb

    进入临界区，如果输入参数p_tcb为空，则检查当前内核是否处于停止态：如果内核处于停止态，则退出临界区，返回错误码OS_ERR_OS_NOT_RUNNING；如果内核处于运行态，将p_tcb指向当前任务。

（4）检查是否锁定任务调度

    如果输入参数p_tcb指向当前任务且任务调度被锁定的时，退出临界区，返回错误码OS_ERR_SCHED_LOCKED

（5）挂起任务

将错误码设置为OS_ERR_NONE，检查目标任务的状态：

  1. 目标任务处于就绪态，更新任务状态为挂起态，挂起计数器设置为1，将目标任务从就绪列表中移除，退出临界区；
  2. 目标任务处于延时态，更新任务状态为延时挂起态，挂起计数器设置为1，退出临界区；
  3. 目标任务处于等待态，更新任务状态为等待挂起态，挂起计数器设置为1，退出临界区；
  4. 目标任务处于等待超时态，更新任务状态为等待超时挂起态，挂起计数器设置为1，退出临界区；
  5. 目标任务处于挂起态、延时挂起态、等待挂起态、等待超时挂起态四种状态之一，检查挂起计数器是否异常：如果异常，退出临界区，函数返回，返回错误码OS_ERR_TASK_SUSPEND_CTR_OVF，如果正常，退出临界区，挂起计数器加1；
  6. 目标任务处于其他状态，退出临界区，返回错误码OS_ERR_STATE_INVALID。

（6）任务调度

    挂起任务结束后，如果内核处于运行态，将执行任务调度

  1. 返回值
  2. 无
  3. 健康管理错误码
  4. 无
