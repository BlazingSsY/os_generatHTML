需求描述

需求编号：HLR_OS_01_06

工作描述：HLR：uC/OS-Ⅲ应提供终止任务等待任务消息接口。

是否需求：是

验证方法：测试验证

补充信息：

  1. 函数名称
  2. OSTaskQPendAbort( ) ---终止任务等待消息
  3. 函数框架
  CPU_BOOLEAN OSTaskQPendAbort(
OS_TCB *p_tcb, / in: 要中止 Pend 的任务控制块指针 /
OS_OPT opt, / in: 中止 Pend 时的选项 */
OS_ERR *p_err / out: 错误码指针 */
);
  1. 函数描述

    该函数用于终止并就绪指定的任务，通常用于异常终止任务对任务消息的等待（而不是通过正常的消息发送方式）。

  1. 参数说明

（1）p_tcb

类型：OS_TCB*

作用：指向要中止 Pend 的任务的任务控制块（TCB）。

注意事项：

    如果 p_tcb 为 NULL，表示尝试中止自身，将返回 OS_ERR_PEND_ABORT_SELF。

    如果 p_tcb 指向的任务未处于等待消息的状态，将返回 OS_ERR_PEND_ABORT_NONE。

（2）opt

类型：OS_OPT

作用：指定中止 Pend 时的行为选项。

取值范围：

    OS_OPT_POST_NONE：无特殊选项，调用后会调用调度程序。

    OS_OPT_POST_NO_SCHED：不调用调度程序，需手动调用。

注意事项：无效的选项将导致 OS_ERR_OPT_INVALID 错误。

（3）p_err

类型：OS_ERR*

作用：返回函数调用的结果状态码。

返回代码：

OS_ERR_NONE：成功中止 Pend 并就绪任务。
    OS_ERR_OPT_INVALID：无效的 opt 参数。
    OS_ERR_OS_NOT_RUNNING：uC/OS-III 尚未运行。
    OS_ERR_PEND_ABORT_ISR：从 ISR 中调用 Pend 中止。
    OS_ERR_PEND_ABORT_NONE：任务未处于等待消息状态。
    OS_ERR_PEND_ABORT_SELF：尝试中止自身或当前任务。
  1. 函数算法逻辑

(1)检查是否在中断环境中调用

如果 OS_CFG_CALLED_FROM_ISR_CHK_EN 功能启用，检测当前是否处于中断服务例程（ISR）中。若处于 ISR 中，将错误码 OS_ERR_PEND_ABORT_ISR 存储到 p_err 中，并返回 OS_FALSE。

(2)检查内核是否处于运行状态

如果 OS_CFG_INVALID_OS_CALLS_CHK_EN 功能启用，判断 OSRunning 是否等于 OS_STATE_OS_RUNNING 。若不等于，说明内核未正常运行，将错误码 OS_ERR_OS_NOT_RUNNING 存储到 p_err 中，并返回 OS_FALSE。

(3)检查终止选项的有效性

如果 OS_CFG_ARG_CHK_EN 功能启用，检查 opt 参数是否为有效的终止选项。有效的选项包括 OS_OPT_POST_NONE 和 OS_OPT_POST_NO_SCHED 。若无效，将错误码 OS_ERR_OPT_INVALID 存储到 p_err 中，并返回 OS_FALSE。

(4)检查是否为终止自身任务等待

进入临界区。

检查 p_tcb 是否为 NULL 或等于当前任务控制块（OSTCBCurPtr）。如果是，退出临界区，将错误码 OS_ERR_PEND_ABORT_SELF 存储到 p_err 中，并返回 OS_FALSE。

(5)检查任务是否处于等待状态

检查任务的等待状态是否为 OS_TASK_PEND_ON_TASK_Q 。如果不是，退出临界区，将错误码 OS_ERR_PEND_ABORT_NONE 存储到 p_err 中，并返回 OS_FALSE。

(6)终止任务等待并指向任务调度

  1. 调用 OS_PendAbort() 函数终止任务的等待操作，设置任务的等待状态为 OS_STATUS_PEND_ABORT。
  2. 退出临界区
  3. 如果 opt 中不包含 OS_OPT_POST_NO_SCHED 选项，调用 OSSched() 进行任务调度，选择下一个最高优先级的就绪任务来运行。
  4. 将错误码 OS_ERR_NONE 存储到 p_err 中，并返回 OS_TRUE。
  5. 返回值

类型：CPU_BOOLEAN

作用：

  OS_TRUE：任务处于等待消息状态并被成功中止。

  OS_FALSE：任务未处于等待消息状态或发生错误。

  1. 健康管理错误码
  2. 无
