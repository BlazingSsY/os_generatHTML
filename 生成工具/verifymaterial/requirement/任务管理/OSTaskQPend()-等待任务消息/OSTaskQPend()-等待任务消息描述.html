需求描述

需求编号：HLR_OS_01_05

工作描述：HLR：uC/OS-Ⅲ应提供任务等待队列消息接口。

是否需求：是

验证方法：测试验证

补充信息：

  1. 函数名称
  2. OSTaskQPend( ) – 任务等待队列消息
  3. 函数框架
  void* OSTaskQPend(
OS_TICK timeout, / in: 超时时间 /
OS_OPT opt, / in: 等待选项 */
OS_MSG_SIZE *p_msg_size,/ out: 返回消息的大小 */
CPU_TS *p_ts, / out: 返回消息的时间戳 */
OS_ERR *p_err / out: 错误码指针 */
);
  1. 函数描述

该函数获取当前任务消息队列中的消息。如果队列中没有消息，任务将根据指定的 opt 参数决定是否等待，或者直到超时或被中断。

  1. 参数说明

（1）timeout

类型：OS_TICK

作用：指定任务等待消息的超时时间（以SysTick为单位）。

注意事项：

    如果设置为 0，表示无限期等待（除非被中断）。

（2）opt

类型：OS_OPT

作用：指定任务的等待行为。

取值范围：

    OS_OPT_PEND_BLOCKING：任务将等待消息的等待态，直到消息到达或超时。

    OS_OPT_PEND_NON_BLOCKING：如果当前没有消息，任务将立即返回错误 OS_ERR_PEND_WOULD_BLOCK。

注意事项：无效的选项将导致 OS_ERR_OPT_INVALID 错误。

（3）p_msg_size

类型：OS_MSG_SIZE*

作用：返回消息的大小。如果消息队列中存在消息，p_msg_size 将被填充为消息的大小。

注意事项：如果传入 NULL 指针，函数将返回 OS_ERR_PTR_INVALID。

（4）p_ts

类型：CPU_TS*

作用：返回消息的时间戳。如果消息队列中存在消息，p_ts 将被填充为消息接收的时间戳。

注意事项：如果传入 NULL 指针，函数将忽略时间戳。

（5）p_err

类型：OS_ERR*

作用：返回函数调用的结果状态码。

返回代码：

  OS_ERR_NONE：成功收到消息。
    OS_ERR_OPT_INVALID：无效的 opt 参数。
    OS_ERR_OS_NOT_RUNNING：uC/OS-III 尚未运行。
    OS_ERR_PEND_ABORT：Pend 被中断。
    OS_ERR_PEND_ISR：从 ISR中 调用 Pend 函数。
    OS_ERR_PEND_WOULD_BLOCK：非等待模式下队列为空。
    OS_ERR_PTR_INVALID：p_msg_size 为 NULL。
    OS_ERR_SCHED_LOCKED：调度程序被锁定。
    OS_ERR_TIMEOUT：在超时时间内未收到消息。
  1. 函数算法逻辑

（1）中断调用检查

  1. OS_CFG_CALLED_FROM_ISR_CHK_EN功能启用，检查OSIntNestingCtr是否大于0。若大于0，则说明当前函数在ISR中执行，返回错误码OS_ERR_PEND_ISR

  (2) 内核运行状态检查

  1. OS_CFG_INVALID_OS_CALLS_CHK_EN功能启用，检查OSRunning是否不等于OS_STATE_OS_RUNNING。若不等于，说明当前内核不在运行状态，返回错误码OS_ERR_OS_NOT_RUNNING

（3）输入参数p_msg_size合法性检查

  1. OS_CFG_ARG_CHK_EN功能启用时，如果 p_msg_size 为 NULL，返回 OS_ERR_PTR_INVALID。

（4）输入参数opt合法性检查

  1. OS_CFG_ARG_CHK_EN功能启用时，如果 opt 不是 OS_OPT_PEND_BLOCKING 或 OS_OPT_PEND_NON_BLOCKING，返回 OS_ERR_OPT_INVALID。

  (5) 初始化时间戳变量

  1. 如果输入的时间戳变量指针p_ts不为空，则将时间戳变量初始化为0

（6）从当前任务的消息队列中获取消息

  1.  进入临界区，调用 OS_MsgQGet() 尝试从当前任务的队列中获取消息，调用时，将函数的输入指针参数p_msg_size、p_ts、p_err传递给给OS_MsgQGet()函数
  2. 如果消息成功获取即OS_MsgQGet()返回的错误码为OS_ERR_NONE, 则退出临界区，并返回消息指针。

 （7）判断opt是否为非阻塞模式

如果任务消息队列中没有消息 ，则判断opt是否设置为非阻塞模式OS_OPT_PEND_NON_BLOCKING，若是则退出临界区返回 OS_ERR_PEND_WOULD_BLOCK，返回空的消息指针

  （8）判断是否在阻塞模式且调度器锁定情况下等待任务消息队列

如果任务消息队列中没有消息且opt为阻塞模式时，判断任务调度是否锁定，若任务调度被锁定，则退出临界区返回OS_ERR_SCHED_LOCKED，返回空的消息指针

（9）阻塞等待当前任务

    当任务消息队列中没有消息且opt为阻塞模式且调度器没有被锁定时：

  1. 调用OS_Pend函数使任务进入等待状态并等待任务消息，若输入参数timeout为0，则任务进入等待态，若timeout大于0，则任务进入等待超时态
  2. 退出临界区，并调用 OSSched() 执行任务调度， 选择执行下一个最高优先级任务。

（10）根据等待状态做处理

         进入临界区，根据任务的等待状态进行处理：

  1. 等待状态为 OS_STATUS_PEND_OK时，设置消息指针和消息大小，设置错误码为OS_ERR_NONE；
  2. 等待状态为 OS_STATUS_PEND_ABORT时，设置消息指针为空和消息大小为0，如果输入的时间戳指针不为空，则设置时间戳变量为0，设置错误码为OS_ERR_PEND_ABORT；
  3. 等待状态为OS_STATUS_PEND_TIMEOUT或其他状态时，消息指针为空和消息大小为0，设置错误码为OS_ERR_TIMEOUT；

（11）返回消息指针

       退出临界区，返回消息指针

   

  1. 返回值
  2. 类型： void *
  3. 作用：指向任务消息的指针
  4. 健康管理错误码
  5. 无
