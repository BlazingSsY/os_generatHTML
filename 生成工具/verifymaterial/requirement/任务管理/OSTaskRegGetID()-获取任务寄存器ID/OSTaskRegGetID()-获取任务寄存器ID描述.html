需求描述

需求编号：HLR_OS_01_09

工作描述：HLR：uC/OS-Ⅲ应提供获取下一个可用的任务寄存器ID接口。

是否需求：是

验证方法：测试验证

补充信息：

  1. 函数名称
  2. OSTaskRegGetID() - 获取下一个可用的任务寄存器ID。
  3. 函数框架
OS_REG_ID OSTaskRegGetID(OS_ERR *p_err / out: 错误码指针 */
);
  1. 函数描述

该函数用于动态获取下一个可用的任务寄存器ID，避免了静态分配寄存器ID的局限性，使任务寄存器ID的分配更加灵活和高效。

  1. 参数说明

（1）p_err

类型：OS_ERR*

作用：返回函数调用的结果状态码。

返回代码：

OS_ERR_NONE：调用成功，成功分配了任务寄存器ID。
    OS_ERR_NO_MORE_ID_AVAIL：无法分配更多任务寄存器ID，已超过通过OS_CFG_TASK_REG_TBL_SIZE配置的可用数量。
  1. 函数算法逻辑

（1）检查可用任务寄存器 ID：

    检查全局变量OSTaskRegNextAvailID是否已超过配置的最大值OS_CFG_TASK_REG_TBL_SIZE。

    如果超过，设置错误码为OS_ERR_NO_MORE_ID_AVAIL，退出临界区，并返回OS_CFG_TASK_REG_TBL_SIZE表示无法分配。

（2）分配任务寄存器 ID：

  1. 将当前的OSTaskRegNextAvailID作为分配的ID。
  2. 增量更新OSTaskRegNextAvailID以便下次分配。
  3. 设置p_err为OS_ERR_NONE，表示分配成功，并返回分配的任务寄存器ID。
  4. 返回值

   类型: OS_REG_ID

   作用： 成功时，返回下一个可用的任务寄存器ID。如果没有可用的ID，则返回OS_CFG_TASK_REG_TBL_SIZE。

  1. 健康管理错误码

    无
