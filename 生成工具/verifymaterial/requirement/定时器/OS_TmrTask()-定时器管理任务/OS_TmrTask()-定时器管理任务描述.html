需求描述

需求编号：HLR_OS_06_12

工作描述：HLR：uC/OS-Ⅲ应提供定时器任务管理的功能。

是否需求：是

验证方法：测试验证

补充信息：

1.函数名称

OS_TmrTask() --- 定时器任务管理函数

2.函数框架

void OS_TmrTask (
    void *p_arg /* 输入：任务参数 */
)

3.函数描述

该函数是定时器任务的入口点，负责处理定时器链表中的定时器。它是一个无限循环，用于周期性地检查定时器链表中的定时器是否到期，并执行相应的回调函数。定时器任务通过条件变量等待和处理定时器事件，确保定时器能够准确地触发。

4.参数说明

（1）p_arg

类型：void*

作用：任务参数，用于传递给定时器任务的参数。在本函数中未使用该参数，以防止编译器警告。

5.算法逻辑

1）锁定定时器模块

  1. 调用 OS_TmrLock() 函数锁定定时器模块，防止在定时器任务执行过程中被其他任务或中断干扰，并进入无线循环，后续步骤在无线循环里处理

2）无限循环处理定时器，确定超时时间

进入无限循环（for (;;)），持续处理定时器事件：

  1. 若定时器链表为空，将超时时间（timeout）设置为 0。
  2. 否则，将超时时间设置为链表头节点定时器的剩余时间。

3）等待条件变量

调用 OS_TmrCondWait(timeout) 函数，使定时器任务暂停执行，等待条件变量触发。该函数会释放互斥锁，允许其他任务添加或移除定时器。

4）处理空链表情况

若定时器链表为空，继续下一次循环（continue）。

5）获取当前时间、计算经过时间并更新定时器剩余时间

  1. 进入临界区（CPU_CRITICAL_ENTER()），获取当前滴答计数（OSTickCtr）作为当前时间（time），然后退出临界区（CPU_CRITICAL_EXIT()）。
  2. 计算从上次处理到当前的经过时间（elapsed = time - OSTmrTaskTickBase）。
  3. 更新 OSTmrTaskTickBase 为当前时间。
  4. 遍历定时器链表，更新每个定时器的剩余时间。若经过时间大于定时器的剩余时间，则将定时器的剩余时间设置为 0，并减少经过时间；否则，减少定时器的剩余时间并设置经过时间为 0。

6）处理到期定时器

遍历定时器链表，处理已到期的定时器（Remain == 0）：

  1. 将定时器状态设置为超时（OS_TMR_STATE_TIMEOUT）。
  2. 若定时器有回调函数，执行回调函数。
  3. 若定时器为周期性定时器，将其从链表中移除（OS_TmrUnlink），重新设置定时器的剩余时间为周期时间，并将定时器重新链接到链表中（OS_TmrLink）；否则，将定时器标记为完成状态（OS_TMR_STATE_COMPLETED），并重置其指针和剩余时间。

6.返回值

无。

7.健康管理错误码

无。

备注：此函数是uC/OS-III的内部函数，您的应用程序不得调用它。
