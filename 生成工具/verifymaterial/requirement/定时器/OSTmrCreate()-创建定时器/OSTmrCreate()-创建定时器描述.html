需求描述

需求编号：HLR_OS_06_01

工作描述：HLR：uC/OS-Ⅲ应提供定时器创建接口，以创建和初始化定时器。

是否需求：是

验证方法：测试验证

补充信息：

1. 函数名称

OSTmrCreate()---定时器创建

2. 函数框架

void OSTmrCreate {
    OS_TMR              *p_tmr,       /* 输入：指向定时器控制块的指针 */
    CPU_CHAR            *p_name,      /* 输入：定时器的名称 */
    OS_TICK              dly,         /* 输入：定时器启动延迟时间（滴答数） */
    OS_TICK              period,      /* 输入：定时器周期时间（滴答数） */
    OS_OPT               opt,         /* 输入：定时器选项 */
    OS_TMR_CALLBACK_PTR  p_callback,  /* 输入：定时器回调函数指针 */
    void                *p_callback_arg,/* 输入：回调函数参数 */
    OS_ERR              *p_err        /* 输出：错误码指针 */
}

3. 函数描述

该函数用于创建一个定时器。定时器可以是周期性的或单次触发的，具体由选项参数决定。函数会初始化定时器控制块，并将其设置为停止状态。

4. 参数说明

(1) p_tmr

  1. 类型：OS_TMR*
  2. 作用：指向定时器控制块的指针，用于管理定时器。

(2) p_name

  1. 类型：CPU_CHAR*
  2. 作用：定时器的名称，用于调试和跟踪。

(3) dly

  1. 类型：OS_TICK
  2. 作用：定时器启动前的延迟时间（以滴答数为单位）。

(4) period

  1. 类型：OS_TICK
  2. 作用：定时器的周期时间（以滴答数为单位），仅适用于周期性定时器。

(5) opt

  1. 类型：OS_OPT
  2. 作用：指定定时器的类型，可以是以下两种取值：

              OS_OPT_TMR_PERIODIC：创建周期性定时器。

              OS_OPT_TMR_ONE_SHOT：创建单次触发定时器。

(6) p_callback

  1. 类型：OS_TMR_CALLBACK_PTR
  2. 作用：指向定时器回调函数的指针，当定时器到期时调用该函数。

(7) p_callback_arg

  1. 类型：void*
  2. 作用：传递给回调函数的参数。

(8) p_err

  1. 类型：OS_ERR*
  2. 作用：用于返回函数执行的错误码，指示函数调用是否成功以及失败的原因。
  3. 错误码：
OS_ERR_TMR_ISR：从中断服务例程调用该函数，这是不允许的。
OS_ERR_OBJ_PTR_NULL：传入的定时器控制块指针无效。
OS_ERR_OPT_INVALID：传入的 opt 参数不是有效的选项。
OS_ERR_TMR_INVALID_PERIOD：对于周期性定时器，周期时间设置为 0。
OS_ERR_TMR_INVALID_CALLBACK：对于周期性定时器，回调函数指针为 NULL。
OS_ERR_TMR_INVALID_DLY：对于单次触发定时器，延迟时间设置为 0。
OS_ERR_OBJ_CREATED：定时器控制块已被创建。
OS_ERR_NONE：函数执行成功，无错误。


5. 函数算法逻辑

1). 检查是否在中断环境中调用

如果 OS_CFG_CALLED_FROM_ISR_CHK_EN 功能启用，检测当前是否处于中断服务例程（ISR）中。若处于 ISR 中，将错误码 OS_ERR_TMR_ISR 存储到 p_err 中，并返回。

2). 参数检查

如果 OS_CFG_ARG_CHK_EN 功能启用：

  1. 检查定时器控制块指针的有效性：检查 p_tmr 是否为 NULL。若为 NULL，将错误码 OS_ERR_OBJ_PTR_NULL 存储到 p_err 中，并返回。
  2. 检查定时器选项的有效性：根据 opt 参数的值进行分支判断：

              当 opt 为 OS_OPT_TMR_PERIODIC 时：检查 period 是否为 0。若是，将错误OS_ERR_TMR_INVALID_PERIOD 存储到 p_err 中，并返回。检查 p_callback 是否为 NULL。若是，将错误码 OS_ERR_TMR_INVALID_CALLBACK 存储到 p_err 中，并返回。

              当 opt 为 OS_OPT_TMR_ONE_SHOT 时：检查 dly 是否为 0。若是，将错误码 OS_ERR_TMR_INVALID_DLY 存储到 p_err 中，并返回。

              其他情况：将错误码 OS_ERR_OPT_INVALID 存储到 p_err 中，并返回。

3). 锁定定时器模块（如果内核运行）

如果内核处于运行状态（OSRunning == OS_STATE_OS_RUNNING），调用 OS_TmrLock() 锁定定时器模块，防止在定时器创建过程中被其他任务或中断干扰。

4). 检查定时器控制块是否已被创建

如果启用了对象类型检查（OS_OBJ_TYPE_REQ > 0u）并且启用了已创建对象检查（OS_CFG_OBJ_CREATED_CHK_EN > 0u），检查定时器控制块的类型是否为定时器类型（OS_OBJ_TYPE_TMR）。如果是，解锁定时器模块（如果内核运行），将错误码 OS_ERR_OBJ_CREATED 存储到 p_err 中，并返回。

5). 初始化定时器控制块

将定时器控制块的类型设置为 OS_OBJ_TYPE_TMR（如果 OS_OBJ_TYPE_REQ > 0u）。

将定时器的状态设置为 OS_TMR_STATE_STOPPED，表示定时器处于停止状态。

将定时器的启动延迟时间（Dly）设置为 dly 乘以滴答数转换因子（OSTmrToTicksMult）。

将定时器的剩余时间（Remain）初始化为 0。

将定时器的周期时间（Period）设置为 period 乘以滴答数转换因子（OSTmrToTicksMult）。

将定时器的选项（Opt）、回调函数指针（CallbackPtr）、回调函数参数（CallbackPtrArg）分别设置为对应的输入参数。

将定时器的 NextPtr 和 PrevPtr 指针初始化为 NULL，表示定时器未被插入到任何链表中。

6). 解锁定时器模块（如果内核运行）

如果内核处于运行状态，调用 OS_TmrUnlock() 解锁定时器模块。

7). 设置错误码

验证在所有参数有效的情况下，将错误码 OS_ERR_NONE 存储到 p_err 中，表示定时器创建成功。

6. 返回值

类型 ：无返回值（void）

作用 ：该函数不返回值，直接修改定时器控制块的内容，并通过 p_err 参数返回执行状态。

7. 健康管理错误码

无。
