需求描述

需求编号：HLR_OS_06_04

工作描述：HLR：uC/OS-Ⅲ应提供设置定时器参数功能。

是否需求：是

验证方法：测试验证

补充信息：

1.函数名称

OSTmrSet()---设置定时器

2.函数框架

void OSTmrSet (OS_TMR *p_tmr,  /* 输入：指向定时器控制块的指针 */
                OS_TICK dly,  /* 输入：定时器启动延迟时间（滴答数） */
                OS_TICK period,  /* 输入：定时器周期时间（滴答数） */
                OS_TMR_CALLBACK_PTR p_callback,  /* 输入：定时器回调函数指针 */
                void *p_callback_arg,  /* 输入：回调函数参数 */
                OS_ERR *p_err)  /* 输出：错误码指针 */

3.函数描述

该函数用于重新配置已创建定时器的参数（延迟时间、周期、回调函数等）。定时器必须处于停止状态（OS_TMR_STATE_STOPPED）或已删除状态才能被重新配置，否则需先通过 OSTmrStop() 停止定时器。

4.参数说明

（1）p_tmr

类型：OS_TMR*

作用：指向目标定时器的控制块（TCB）。

注意事项：必须指向有效且已创建的定时器，不可为 NULL。

（2）dly

类型：OS_TICK

作用：新的初始延迟时间（用户定义单位，如毫秒）。

限制：单次模式（OS_OPT_TMR_ONE_SHOT）下必须大于 0。

（3）period

类型：OS_TICK

作用：新的周期时间（用户定义单位，如毫秒）。

限制：周期模式（OS_OPT_TMR_PERIODIC）下必须大于 0。

（4）p_callback

类型：OS_TMR_CALLBACK_PTR

作用：新的回调函数指针。

限制：周期模式下不可为 NULL。

（5）p_callback_arg

类型：void*

作用：传递给新回调函数的参数。

（6）p_err

类型：OS_ERR*

作用：返回函数调用的结果状态码。

返回代码：

OS_ERR_NONE：参数设置成功。
OS_ERR_TMR_ISR：从中断服务程序（ISR）中调用。
OS_ERR_OS_NOT_RUNNING：uC/OS-III 尚未运行。
OS_ERR_TMR_INVALID：p_tmr 为 NULL。
OS_ERR_OBJ_TYPE：p_tmr 非定时器类型。
OS_ERR_TMR_INVALID_DLY：单次模式下 dly 为 0。
OS_ERR_TMR_INVALID_PERIOD：周期模式下 period 为 0。
OS_ERR_TMR_INVALID_CALLBACK：周期模式下 p_callback 为 NULL。

5.算法逻辑

1）检查是否在中断环境中调用

若启用了从中断服务例程调用检查（OS_CFG_CALLED_FROM_ISR_CHK_EN），通过判断 OSIntNestingCtr 是否大于 0 来检测是否处于中断服务例程中。若处于中断服务例程中，将错误码 OS_ERR_TMR_ISR 存入 p_err，并返回。

2）检查内核运行状态

若启用了无效操作系统调用检查（OS_CFG_INVALID_OS_CALLS_CHK_EN），判断 OSRunning 是否等于 OS_STATE_OS_RUNNING 来确定内核是否处于运行状态。若内核未运行，将错误码 OS_ERR_OS_NOT_RUNNING 存入 p_err，并返回。

3）检查定时器控制块指针有效性

若启用了参数检查（OS_CFG_ARG_CHK_EN），判断 p_tmr 是否为 NULL。若为 NULL，将错误码 OS_ERR_TMR_INVALID 存入 p_err，并返回。

4）检查定时器控制块类型

若启用了对象类型检查（OS_CFG_OBJ_TYPE_CHK_EN），判断 p_tmr->Type 是否等于 OS_OBJ_TYPE_TMR。若不等于，将错误码 OS_ERR_OBJ_TYPE 存入 p_err，并返回。

5）检查定时器选项及参数有效性

若启用了参数检查（OS_CFG_ARG_CHK_EN），根据定时器的选项（p_tmr->Opt）进行分支判断：

  1. 对于周期性定时器（OS_OPT_TMR_PERIODIC） ：检查 period 是否为 0。若为 0，将错误码 OS_ERR_TMR_INVALID_PERIOD 存入 p_err，并返回。同时，检查 p_callback 是否为 NULL。若为 NULL，将错误码 OS_ERR_TMR_INVALID_CALLBACK 存入 p_err，并返回。
  2. 对于单次触发定时器（OS_OPT_TMR_ONE_SHOT） ：检查 dly 是否为 0。若为 0，将错误码 OS_ERR_TMR_INVALID_DLY 存入 p_err，并返回。
  3. 其他情况 ：将错误码 OS_ERR_TMR_INVALID 存入 p_err，并返回。

6）锁定定时器模块

调用 OS_TmrLock() 函数锁定定时器模块，防止在设置定时器参数过程中定时器状态被其他任务或中断改变。

7）更新定时器参数

  1. 将定时器的启动延迟时间（Dly）设置为 dly 乘以滴答数转换因子（OSTmrToTicksMult）。
  2. 将定时器的周期时间（Period）设置为 period 乘以滴答数转换因子（OSTmrToTicksMult）。
  3. 更新定时器的回调函数指针（CallbackPtr）为 p_callback。
  4. 更新定时器的回调函数参数（CallbackPtrArg）为 p_callback_arg。

8）设置错误码

将错误码 OS_ERR_NONE 存入 p_err，表示定时器参数设置成功。

9）解锁定时器模块

调用 OS_TmrUnlock() 函数解锁定时器模块。

6.返回值

无。

7.健康管理错误码

无。

备注：此函数可以在运行中的计时器上调用，对延迟和周期的更改只有在当前周期或延迟过去后才会生效，对回调的更改将立即生效。
