需求描述

需求编号：HLR_OS_06_10

工作描述：HLR：uC/OS-Ⅲ应提供链接定时器到链表的功能。

是否需求：是

验证方法：测试验证

补充信息：

1.函数名称

OS_TmrLink() --- 链接定时器到链表

2.函数框架

void OS_TmrLink (
    OS_TMR *p_tmr, /* 输入：指向定时器控制块的指针 */
    OS_TICK time /* 输入：当前时间 */
)

3.函数描述

该函数用于将定时器插入到定时器链表中。根据当前时间计算定时器的剩余时间，并将定时器插入到链表中的正确位置，以确保定时器按正确的顺序触发。

4.参数说明

（1）p_tmr

类型：OS_TMR*

作用：指向待链接的定时器控制块，需已初始化且状态为 RUNNING。

注意事项：调用前需确保定时器未被其他链表引用，且已通过互斥锁锁定。

（2）time

类型：OS_TICK

作用：调用时的系统当前时间（时钟节拍数），通常为 OSTickCtr 或定时器任务的节拍基准。

5.算法逻辑

1）处理定时器链表为空的情况

若定时器链表为空（OSTmrListPtr 为 NULL），将当前定时器设置为链表的第一个节点。设置定时器的前驱指针（PrevPtr）和后继指针（NextPtr）为 NULL，并将 OSTmrListPtr 指向该定时器。更新 OSTmrTaskTickBase 为输入time，并发送条件变量信号（OS_TmrCondSignal()），然后返回。

2）计算时间差值，并检查是否插入链表头部

若链表不为空，计算输入的time与 OSTmrTaskTickBase 的时间差值（delta），并加上定时器的剩余时间（p_tmr->Remain）。

  1. 获取链表的第一个节点（p_tmr2 = OSTmrListPtr），并获取其剩余时间（remain = p_tmr2->Remain）。
  2. 若计算的时间差值（delta）小于第一个节点的剩余时间，并且第一个节点没有前驱节点（p_tmr2->PrevPtr == NULL），则将当前定时器插入链表头部。更新相关指针，并调整第一个节点的剩余时间，然后更新 OSTmrTaskTickBase 和发送条件变量信号。

3）遍历链表，并将定时器插入链表

若不满足插入头部的条件，调整 delta 为相对于链表头部的时间差值。遍历链表，找到合适的位置插入当前定时器。在遍历过程中，更新 delta 值，比较当前节点的剩余时间和 delta，直到找到 delta 小于当前节点剩余时间的位置：

  1. 若找到插入位置且该位置不是链表末尾，更新当前节点和相邻节点的指针，调整剩余时间，并将当前定时器插入链表中。
  2. 若遍历到链表末尾仍未找到合适位置，则将当前定时器插入链表末尾，设置其前驱指针为最后一个节点，后继指针为 NULL，并更新最后一个节点的后继指针。

6.返回值

无。

7.健康管理错误码

无。

备注：此函数是uC/OS-III的内部函数，您的应用程序不得调用它。
