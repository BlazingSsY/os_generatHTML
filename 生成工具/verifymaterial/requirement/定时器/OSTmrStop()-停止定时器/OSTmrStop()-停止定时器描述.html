需求描述

需求编号：HLR_OS_06_07

工作描述：HLR：uC/OS-Ⅲ应提供停止定时器的功能。

是否需求：是

验证方法：测试验证

补充信息：

1.函数名称

OSTmrStop() --- 停止定时器

2.函数框架

CPU_BOOLEAN OSTmrStop (
    OS_TMR *p_tmr, /* 输入：指向定时器控制块的指针 */
    OS_OPT opt, /* 输入：停止选项 */
    void *p_callback_arg,/* 输入：回调函数参数 */
    OS_ERR *p_err /* 输出：错误码指针 */
);

3.函数描述

该函数用于停止一个正在运行的定时器，可选择在停止时执行其回调函数。定时器停止后进入 OS_TMR_STATE_STOPPED 状态，可通过 OSTmrStart() 重新启动。

4.参数说明

（1）p_tmr

类型：OS_TMR*

作用：指向目标定时器的控制块（TCB）。

注意事项：不可为 NULL，且必须为已创建的定时器。

（2）opt

类型：OS_OPT

作用：停止时的行为选项：

           OS_OPT_TMR_NONE：停止定时器，不执行回调函数。

           OS_OPT_TMR_CALLBACK：停止定时器，并执行创建时配置的回调函数。

           OS_OPT_TMR_CALLBACK_ARG：停止定时器，并执行回调函数，使用本次传入的 p_callback_arg 作为参数。

限制：若定时器无回调函数（p_tmr->CallbackPtr == NULL），执行回调相关选项会返回错误。

（3）p_callback_arg

类型：void*

作用：当 opt 为 OS_OPT_TMR_CALLBACK_ARG 时，传递给回调函数的参数。

（4）p_err

类型：OS_ERR*

作用：返回函数调用的结果状态码。

错误码：

OS_ERR_NONE：定时器停止成功。
OS_ERR_TMR_ISR：从中断服务程序（ISR）中调用。
OS_ERR_OS_NOT_RUNNING：uC/OS-III 尚未运行。
OS_ERR_TMR_INVALID：p_tmr 为 NULL。
OS_ERR_OBJ_TYPE：p_tmr 非定时器类型。
OS_ERR_OPT_INVALID：opt 为无效选项（非预定义值）。
OS_ERR_TMR_NO_CALLBACK：尝试执行回调但定时器无回调函数。
OS_ERR_TMR_STOPPED：定时器已处于停止或完成状态（非错误，仅提示）。
OS_ERR_TMR_INACTIVE：定时器处于未使用状态（已被删除）。
OS_ERR_TMR_INVALID_STATE：定时器状态无效（如未知状态值）。

5.算法逻辑

1）检查是否在中断环境中调用

若启用了从中断服务例程调用检查（OS_CFG_CALLED_FROM_ISR_CHK_EN），检测当前是否处于中断服务例程中（通过判断 OSIntNestingCtr 是否大于 0）。若处于中断服务例程中，将错误码 OS_ERR_TMR_ISR 存入 p_err 中，并返回 OS_FALSE。

2）检查内核运行状态

若启用了无效操作系统调用检查（OS_CFG_INVALID_OS_CALLS_CHK_EN），判断内核是否处于运行状态（OSRunning 是否等于 OS_STATE_OS_RUNNING）。若内核未运行，将错误码 OS_ERR_OS_NOT_RUNNING 存入 p_err 中，并返回 OS_FALSE。

3）检查定时器控制块指针有效性

若启用了参数检查（OS_CFG_ARG_CHK_EN），判断传入的定时器控制块指针 p_tmr 是否为 NULL。若为 NULL，将错误码 OS_ERR_TMR_INVALID 存入 p_err 中，并返回 OS_FALSE。

4）检查定时器控制块类型

若启用了对象类型检查（OS_CFG_OBJ_TYPE_CHK_EN），判断定时器控制块的类型（p_tmr->Type）是否为定时器类型（OS_OBJ_TYPE_TMR）。若不是，将错误码 OS_ERR_OBJ_TYPE 存入 p_err 中，并返回 OS_FALSE。

5）锁定定时器模块

调用 OS_TmrLock() 函数锁定定时器模块，防止在停止定时器过程中被其他任务或中断干扰。

6）获取当前时间基准

进入临界区（CPU_CRITICAL_ENTER()），判断当前处于哪个任务上下文。若处于定时器任务上下文（OSTCBCurPtr 等于 OSTmrTaskTCB），则使用定时器任务的滴答基准时间（OSTmrTaskTickBase）作为当前时间；否则，使用当前的滴答计数（OSTickCtr）作为当前时间。然后退出临界区（CPU_CRITICAL_EXIT()）。

7）根据定时器状态进行不同操作

（1）定时器处于运行中（OS_TMR_STATE_RUNNING）或超时（OS_TMR_STATE_TIMEOUT）状态 ，根据 opt 参数的值进行分支判断：

  1. 1）OS_OPT_TMR_CALLBACK ：将定时器状态设置为停止（OS_TMR_STATE_STOPPED），从定时器链表中移除定时器（调用 OS_TmrUnlink()），获取定时器的回调函数指针（p_fnct = p_tmr->CallbackPtr）。若回调函数存在，调用该回调函数，并传递定时器创建时的回调参数（p_tmr->CallbackPtrArg），将错误码 OS_ERR_NONE 存入 p_err；否则，将错误码 OS_ERR_TMR_NO_CALLBACK 存入 p_err。
  2. 2）OS_OPT_TMR_CALLBACK_ARG ：将定时器状态设置为停止（OS_TMR_STATE_STOPPED），从定时器链表中移除定时器，获取回调函数指针。若回调函数存在，调用该回调函数，并传递传入的 p_callback_arg 参数，将错误码 OS_ERR_NONE 存入 p_err；否则，将错误码 OS_ERR_TMR_NO_CALLBACK 存入 p_err。
  3. 3）OS_OPT_TMR_NONE ：从定时器链表中移除定时器，将错误码 OS_ERR_NONE 存入 p_err。
  4. 4）其他非法选项 ：解锁定时器模块，将错误码 OS_ERR_OPT_INVALID 存入 p_err，并返回 OS_FALSE。

  最后设置返回值为OS_TRUE。

（2）定时器处于完成（OS_TMR_STATE_COMPLETED）或停止（OS_TMR_STATE_STOPPED）状态 ：将定时器状态设置为停止，并将错误码 OS_ERR_TMR_STOPPED 存入 p_err，设置返回值为OS_TRUE。

（3）定时器处于未使用（OS_TMR_STATE_UNUSED）状态 ：说明定时器未被创建，将错误码 OS_ERR_TMR_INACTIVE 存入 p_err，设置返回值为 OS_FALSE。

（4）其他非法状态 ：将错误码 OS_ERR_TMR_INVALID_STATE 存入 p_err，设置返回值为 OS_FALSE。

8）解锁定时器模块

调用 OS_TmrUnlock() 函数解锁定时器模块

9) 返回执行结果

将已设置的返回值变量返回

6.返回值

类型 ：CPU_BOOLEAN

作用 ：表示定时器停止操作是否成功。若成功返回 OS_TRUE，否则返回 OS_FALSE

7.健康管理错误码

无。
