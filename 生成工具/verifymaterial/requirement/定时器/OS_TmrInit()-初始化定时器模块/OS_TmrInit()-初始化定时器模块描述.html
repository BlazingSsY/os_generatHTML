需求描述

需求编号：HLR_OS_06_09

工作描述：HLR：uC/OS-Ⅲ应提供初始化定时器模块的功能。

是否需求：是

验证方法：测试验证

补充信息：

1.函数名称

OS_TmrInit()-初始化定时器

2.函数框架

void OS_TmrInit (
    OS_ERR *p_err /* 输出：错误码指针 */
)

3.函数描述

该函数用于初始化定时器模块。它执行一系列操作来设置定时器链表、计算滴答数转换因子、创建互斥信号量和条件变量，并创建定时器任务。这些步骤为定时器的正常运行奠定了基础。

4.参数说明

（1）p_err

类型：OS_ERR*

作用：返回初始化过程中的错误码。

注意事项：不可为 NULL，需指向有效内存以存储错误状态。

错误码：

OS_ERR_NONE：初始化成功。
OS_ERR_TMR_STK_INVALID：定时器任务栈指针无效
OS_ERR_TMR_STK_SIZE_INVALID：定时器任务栈大小小于最小允许值
OS_ERR_TMR_PRIO_INVALID：定时器任务优先级无效。
OS_ERR_OS_OBJ_CREATION_FAILED：互斥锁或条件变量创建失败（

5.算法逻辑

1）初始化定时器链表

将定时器链表指针 OSTmrListPtr 设置为空。通过 OSCfg_TickRate_Hz 除以 OSCfg_TmrTaskRate_Hz 计算得到 OSTmrToTicksMult，用于将定时器时间转换为滴答数。

2）创建互斥信号量

  1. 调用 OSMutexCreate() 函数创建一个互斥信号量 OSTmrMutex，用于保护定时器操作。
  2. 若创建互斥信号量失败（即 p_err 不等于 OS_ERR_NONE），则直接返回。

3）创建条件变量

调用 OS_TmrCondCreate() 函数创建条件变量。

4）检查定时器任务栈基址

检查 OSCfg_TmrTaskStkBasePtr 是否为空。若为空，将错误码 OS_ERR_TMR_STK_INVALID 存入 p_err，并返回。

5）检查定时器任务栈大小

检查 OSCfg_TmrTaskStkSize 是否小于 OSCfg_StkSizeMin。若是，将错误码 OS_ERR_TMR_STK_SIZE_INVALID 存入 p_err，并返回。

6）检查定时器任务优先级

检查 OSCfg_TmrTaskPrio 是否大于或等于 (OS_CFG_PRIO_MAX - 1u)。若是，将错误码 OS_ERR_TMR_PRIO_INVALID 存入 p_err，并返回。

7）创建定时器任务

  1. 调用 OSTaskCreate() 函数创建定时器任务。
  2. 定时器任务的控制块为 OSTmrTaskTCB。
  3. 定时器任务的名称设置为空。
  4. 定时器任务的入口函数为 OS_TmrTask。
  5. 定时器任务的栈基址为 OSCfg_TmrTaskStkBasePtr，栈限制为 OSCfg_TmrTaskStkLimit，栈大小为 OSCfg_TmrTaskStkSize。
  6. 定时器任务的选项包括 OS_OPT_TASK_STK_CHK、OS_OPT_TASK_STK_CLR 和 OS_OPT_TASK_NO_TLS。

6.返回值

无。

7.健康管理错误码

无

备注：此函数是uC/OS-III的内部函数，您的应用程序不得调用它。
