需求描述

需求编号：HLR_OS_06_11

工作描述：HLR：uC/OS-Ⅲ应提供将定时器移除链表的功能。

是否需求：是

验证方法：测试验证

补充信息：

1.函数名称

OS_TmrUnlink() --- 从链表中移除定时器

2.函数框架

void OS_TmrUnlink (
    OS_TMR *p_tmr, /* 输入：指向定时器控制块的指针 */
    OS_TICK time /* 输入：当前时间 */
)

3.函数描述

该函数用于从定时器链表中移除指定的定时器。根据定时器在链表中的位置，调整链表中其他定时器的时间剩余值，并更新链表的结构。

4.参数说明

（1）p_tmr

类型：OS_TMR*

作用：指向待解链的定时器控制块，需已链接到定时器列表中。

注意事项：调用前需确保定时器已通过互斥锁锁定，且状态合法。

（2）time

类型：OS_TICK

作用：当前系统时间（时钟节拍数），用于更新链表中剩余定时器的时间差。

5.算法逻辑

1）处理定时器是定时器链表唯一节点的情况

若定时器没有前驱节点且定时器也没有后继节点 ：

说明链表中只有这一个定时器。将 OSTmrListPtr 设置为 NULL，表示链表为空。将定时器的剩余时间（Remain）设置为 0，并更新 OSTmrTaskTickBase 为当前时间time，发送条件变量信号（OS_TmrCondSignal()）。

2）处理定时器是定时器链表第一节点的情况（有其他节点）

若定时器没有前驱节点且定时器有后继节点 ：

计算从 OSTmrTaskTickBase 到当前时间的经过时间（elapsed = time - OSTmrTaskTickBase）。将后继节点的前驱指针设置为 NULL，并将后继节点的剩余时间加上被移除定时器的剩余时间。将 OSTmrListPtr 更新为后继节点。然后进入循环，调整链表中后续节点的剩余时间，直到 elapsed 为 0 或遍历到链表末尾。如果链表头部节点的剩余时间发生变化或为 0，则更新 OSTmrTaskTickBase 并发送条件变量信号。最后，将被移除定时器的 NextPtr、Remain 设置为 NULL 和 0。

2）处理定时器有前驱节点的情况

若定时器有前驱节点：

  1. 将前驱节点的后继指针指向定时器的后继节点。
  2. 若后继节点存在，将后继节点的前驱指针指向定时器的前驱节点，并将其剩余时间加上被移除定时器的剩余时间
  3. 将被移除定时器的 PrevPtr、NextPtr 设置为 NULL，并将 Remain 设置为 0。

6.返回值

无。

7.健康管理错误码

无。

备注：此函数是uC/OS-III的内部函数，您的应用程序不得调用它。
