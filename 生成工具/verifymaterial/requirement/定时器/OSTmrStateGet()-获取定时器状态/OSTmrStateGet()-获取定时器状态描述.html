需求描述

需求编号：HLR_OS_06_06

工作描述：HLR：uC/OS-Ⅲ应提供获取定时器状态的功能。

是否需求：是

验证方法：测试验证

补充信息：

1.函数名称

OSTmrStateGet() -- 获取定时器状态

2.函数框架

OS_STATE OSTmrStateGet (
    OS_TMR *p_tmr, /* 输入：指向定时器控制块的指针 */
    OS_ERR *p_err /* 输出：错误码指针 */
);

3.函数描述

该函数用于获取指定定时器的当前状态，支持查询定时器是否处于运行、停止、完成、超时或未使用状态。状态查询通过定时器控制块（TCB）的 State 字段实现，适用于需要动态监控定时器生命周期的场景。

4.参数说明

（1）p_tmr

类型：OS_TMR*

作用：指向目标定时器的控制块（TCB）。

注意事项：必须指向有效且已创建的定时器，不可为 NULL 或未初始化状态。

（2）p_err

类型：OS_ERR*

作用：返回函数调用的结果状态码。

返回代码：

OS_ERR_NONE：成功获取状态。
OS_ERR_TMR_ISR：从中断服务程序（ISR）中调用。
OS_ERR_OS_NOT_RUNNING：uC/OS-III 尚未运行。
OS_ERR_TMR_INVALID：p_tmr 为 NULL。
OS_ERR_OBJ_TYPE：p_tmr 非定时器类型。
OS_ERR_TMR_INVALID_STATE：定时器状态无效（非预定义状态值）。

5.算法逻辑

1）检查是否在中断环境中调用

若启用了从中断服务例程调用检查（OS_CFG_CALLED_FROM_ISR_CHK_EN），通过判断 OSIntNestingCtr 是否大于 0 来检测是否处于中断服务例程中。若处于中断服务例程中，将错误码 OS_ERR_TMR_ISR 存入 p_err，并返回 OS_TMR_STATE_UNUSED。

2）检查内核运行状态

若启用了无效操作系统调用检查（OS_CFG_INVALID_OS_CALLS_CHK_EN），判断 OSRunning 是否等于 OS_STATE_OS_RUNNING 来确定内核是否处于运行状态。若内核未运行，将错误码 OS_ERR_OS_NOT_RUNNING 存入 p_err，并返回 OS_TMR_STATE_UNUSED。

3）检查定时器控制块指针有效性

若启用了参数检查（OS_CFG_ARG_CHK_EN），判断 p_tmr 是否为 NULL。若为 NULL，将错误码 OS_ERR_TMR_INVALID 存入 p_err，并返回 OS_TMR_STATE_UNUSED。

4）检查定时器控制块类型

若启用了对象类型检查（OS_CFG_OBJ_TYPE_CHK_EN），判断 p_tmr->Type 是否等于 OS_OBJ_TYPE_TMR。若不等于，将错误码 OS_ERR_OBJ_TYPE 存入 p_err，并返回 OS_TMR_STATE_UNUSED。

5）锁定定时器模块

调用 OS_TmrLock() 函数锁定定时器模块，防止在获取定时器状态过程中定时器状态被其他任务或中断改变。

6）获取并检查定时器状态

获取定时器的当前状态（state = p_tmr->State）。

根据 state 的值进行分支判断：

  1. 对于合法状态（OS_TMR_STATE_UNUSED、OS_TMR_STATE_STOPPED、OS_TMR_STATE_COMPLETED、OS_TMR_STATE_RUNNING、OS_TMR_STATE_TIMEOUT） ：将错误码 OS_ERR_NONE 存入 p_err。
  2. 对于其他非法状态 ：将错误码 OS_ERR_TMR_INVALID_STATE 存入 p_err。

7）解锁定时器模块并返回定时器状态

  1. 调用 OS_TmrUnlock() 函数解锁定时器模块

8）函数返回

  1. 返回定时器状态

6.返回值

类型 ：OS_STATE

作用 ：返回定时器的当前状态，可能的取值包括 OS_TMR_STATE_UNUSED、OS_TMR_STATE_STOPPED、OS_TMR_STATE_COMPLETED、OS_TMR_STATE_RUNNING、OS_TMR_STATE_TIMEOUT 等。

7.健康管理错误码

无
