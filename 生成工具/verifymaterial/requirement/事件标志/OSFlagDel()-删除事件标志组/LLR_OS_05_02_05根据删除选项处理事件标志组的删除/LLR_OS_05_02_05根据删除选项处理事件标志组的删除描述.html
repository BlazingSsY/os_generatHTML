需求描述

需求编号：LLR_OS_05_02_05

需求描述：

1.进入临界区，获取事件标志组的挂起列表 p_pend_list 和初始化任务计数 nbr_tasks 为 0。

2.根据 opt 参数的值进行分支判断：

  1. 当 opt 为 OS_OPT_DEL_NO_PEND 时

               检查等待列表是否为空。若为空，调用 OS_FlagClr() 清除事件标志组，退出临界区，设置错误码为 OS_ERR_NONE 并返回任务计数（0）。若不为空，退出临界区，设置错误码为 OS_ERR_TASK_WAITING 并返回 0 。

  1. 当 opt 为 OS_OPT_DEL_ALWAYS 时

               循环遍历等待列表，只要等待列表的头部指针不为空，就获取当前等待列表头部的 TCB 指针 p_tcb，然后调用 OS_PendAbort() 函数来中止该任务的等待操作，将任务从等待列表中移除，并设置任务的 pend 状态为 OS_STATUS_PEND_DEL，同时任务计数 nbr_tasks 增加 1。循环结束后，调用 OS_FlagClr() 清除事件标志组，退出临界区，然后调用 OSSched() 进行任务调度，设置错误码为 OS_ERR_NONE 并返回任务计数。

  1. 其他情况

                退出临界区，设置错误码为 OS_ERR_OPT_INVALID 并返回 0 。
