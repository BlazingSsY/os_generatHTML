需求描述

需求编号：HLR_OS_05_02

工作描述：HLR：uC/OS-Ⅲ应提供删除事件标志组得接口

是否需求：是

验证方法：测试验证

补充信息：

    1. 函数名称

OSFlagDel() ---事件标志组删除函数

    2. 函数框架

OS_OBJ_QTY OSFlagDel (
         OS_FLAG_GRP *p_grp, /*in: 指向要删除的事件标志组控制块的指针*/
         OS_OPT  opt,  /*in: 删除选项*/
         OS_ERR *p_err  /*out: 错误码指针*/
         )

    3. 函数描述

该函数用于删除一个事件标志组。它可以根据选项选择仅在没有任务等待时删除或强制删除并处理等待任务

    4. 参数说明

        (1) p_grp

        类型：OS_FLAG_GRP*

        作用：指向要删除的事件标志组控制块的指针。

        (2) opt

        类型：OS_OPT

        作用：指定删除选项。可以是以下两种取值：

  1. OS_OPT_DEL_NO_PEND：仅在没有任务因等待该事件标志组而挂起时才删除。
  2. OS_OPT_DEL_ALWAYS：无论如何都删除事件标志组。

       (3) p_err

       类型：OS_ERR*

       作用：用于返回函数执行的错误代码，指示函数调用是否成功以及失败的原因。

       返回代码

OS_ERR_NONE     //成功删除事件标志组
OS_ERR_DEL_ISR                   //试图从中断服务例程（ISR）中删除事件标志组。
OS_ERR_OS_NOT_RUNNING            //内核尚未运行时试图删除事件标志组。
OS_ERR_OBJ_PTR_NULL              //传入的事件标志组控制块指针 p_grp 为 NULL。
OS_ERR_OBJ_TYPE                  //传入的 p_grp 指向的对象类型不是事件标志组类型。
OS_ERR_TASK_WAITING              //当 opt 为 OS_OPT_DEL_NO_PEND 时，一个或多个任务在等待事件标志组。
OS_ERR_OPT_INVALID               //传入的 opt 参数不是有效的选项。

    5. 函数逻辑

    (1) 检查是否在中断环境中调用

  1. 如果 OS_CFG_CALLED_FROM_ISR_CHK_EN功能启用，检测当前是否处于中断服务例程（ISR）中。若处于 ISR 中，将错误码 OS_ERR_DEL_ISR 存储到 p_err 中，并返回 0 。

    (2) 检查内核是否已经运行

  1. 如果OS_CFG_INVALID_OS_CALLS_CHK_EN功能启用，判断 OSRunning 是否等于 OS_STATE_OS_RUNNING 。若不等于，说明内核未正常运行，将错误码 OS_ERR_OS_NOT_RUNNING 存储到 p_err 中，并返回 0 。

    (3) 检查事件标志组控制块指针的有效性

  1. 如果OS_CFG_ARG_CHK_EN功能启用，检查 p_grp 是否为 NULL。若是则将错误码 OS_ERR_OBJ_PTR_NULL 存储到 p_err 中，并返回 0 。

  （4）检查事件标志组控制块的类型

  1. 如果 OS_CFG_OBJ_TYPE_CHK_EN 功能启用，检查 p_grp->Type 是否等于 OS_OBJ_TYPE_FLAG 。若不等于，将错误码 OS_ERR_OBJ_TYPE 存储到 p_err 中，并返回 0 。

  （5）根据删除选项处理事件标志组的删除

  1. 进入临界区，获取事件标志组的挂起列表 p_pend_list 和初始化任务计数 nbr_tasks 为 0。
  2. 根据 opt 参数的值进行分支判断：

                * 当 opt 为 OS_OPT_DEL_NO_PEND 时

                        检查等待列表的头部指针是否为空。若为空，调用 OS_FlagClr() 清除事件标志组，退出临界区，设置错误码为 OS_ERR_NONE 并返回任务计数（0）。若不为空，退出临界区，设置错误码为 OS_ERR_TASK_WAITING 并返回 0 。

                * 当 opt 为 OS_OPT_DEL_ALWAYS 时

                        循环遍历等待列表，只要等待列表的头部指针不为空，就获取当前等待列表头部的 TCB 指针，然后调用 OS_PendAbort() 函数来中止该任务的等待操作，将任务从等待列表中移除，并设置任务的 pend 状态为 OS_STATUS_PEND_DEL，同时任务计数 nbr_tasks 增加 1。循环结束后，调用 OS_FlagClr() 清除事件标志组，退出临界区，然后调用 OSSched() 进行任务调度，设置错误码为 OS_ERR_NONE 并返回任务计数。

                 * 其他情况

                         退出临界区，设置错误码为 OS_ERR_OPT_INVALID 并返回 0 。

    6. 健康管理错误码

类型 ：OS_OBJ_QTY

作用 ：返回被中止等待该事件标志组的任务数量。如果是无条件删除（OS_OPT_DEL_ALWAYS），返回被中止的任务数；如果是仅在无任务等待时删除（OS_OPT_DEL_NO_PEND）且删除成功，则返回 0；如果删除失败（如有任务等待且选择 OS_OPT_DEL_NO_PEND），也返回 0 。

    7. 健康管理错误码

    无
