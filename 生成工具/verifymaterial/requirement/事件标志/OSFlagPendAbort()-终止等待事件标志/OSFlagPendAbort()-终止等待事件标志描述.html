需求描述

需求编号：HLR_OS_05_04

工作描述：HLR：uC/OS-Ⅲ应提供终止等待事件标志的接口

是否需求：是

验证方法：测试验证

补充信息：

1. 函数名称

OSFlagPendAbort() ---终止等待事件标志组中的事件

    2. 函数框架

OS_OBJ_QTY OSFlagPendAbort (
    OS_FLAG_GRP *p_grp, /* 输入：指向事件标志组控制块的指针 */
    OS_OPT opt, /* 输入：中止选项 */
    OS_ERR *p_err /* 输出：错误码指针 */
)

    3. 函数描述

    该函数用于终止一个或多个任务对事件标志组中的事件标志的等待。可以通过选项指定是中止一个任务还是所有任务，并且可以选择是否进行任务调度。

    4. 参数说明

        (1) p_grp

        类型：OS_FLAG_GRP*

        作用：指向事件标志组控制块的指针，用于标识要中止等待的事件标志组。

        (2) opt

        类型：OS_OPT

        作用：指定中止选项。可以是以下两种取值：

                OS_OPT_PEND_ABORT_1：中止一个等待任务。

                OS_OPT_PEND_ABORT_ALL：中止所有等待任务。

                OS_OPT_PEND_ABORT_1 | OS_OPT_POST_NO_SCHED：中止一个等待任务且不进行任务调度。

                OS_OPT_PEND_ABORT_ALL | OS_OPT_POST_NO_SCHED：中止所有等待任务且不进行任务调度。

        (3) p_err

        类型：OS_ERR*

        作用：用于返回函数执行的错误代码，指示函数调用是否成功以及失败的原因。

       错误码

OS_ERR_NONE        //成功终止任务等待事件标志
OS_ERR_PEND_ABORT_ISR //从中断服务例程（ISR）中调用该函数。
OS_ERR_OS_NOT_RUNNING //操作系统内核尚未运行时调用了该函数。
OS_ERR_OBJ_PTR_NULL //事件标志组控制块指针 p_grp 为 NULL。
OS_ERR_OBJ_TYPE    //p_grp 指向的对象类型不是事件标志组类型。
OS_ERR_PEND_ABORT_NONE //没有任务在等待该事件标志组。
OS_ERR_OPT_INVALID //无效opt选项

    5. 函数逻辑

    (1) 检查是否在中断环境中调用

  1. 如果 OS_CFG_CALLED_FROM_ISR_CHK_EN 功能启用，检测当前是否处于中断服务例程（ISR）中。若处于 ISR 中，将错误码 OS_ERR_PEND_ABORT_ISR 存储到 p_err 中，并返回 0 。

    (2) 检查内核是否处于运行状态

  1. 如果 OS_CFG_INVALID_OS_CALLS_CHK_EN 功能启用，判断 OSRunning 是否等于 OS_STATE_OS_RUNNING 。若不等于，说明内核未正常运行，将错误码 OS_ERR_OS_NOT_RUNNING 存储到 p_err 中，并返回 0 。

    (3) 检查参数有效性

       如果 OS_CFG_ARG_CHK_EN 功能启用,则

  1. 验证事件标志组控制块指针的有效性：检查 p_grp 是否为 NULL。若为 NULL，将错误码 OS_ERR_OBJ_PTR_NULL 存储到 p_err 中，并返回 0
  2. 检查opt 参数有效性：

                   * OS_OPT_PEND_ABORT_1： 有效选项，正常处理

                   * OS_OPT_PEND_ABORT_ALL：有效选项，正常处理

                   * OS_OPT_PEND_ABORT_1 | OS_OPT_POST_NO_SCHED：有效选项，正常处理

                   * OS_OPT_PEND_ABORT_ALL | OS_OPT_POST_NO_SCHED：有效选项，正常处理

                   * 其他：无效选项，将错误码 OS_ERR_OPT_INVALID 存储到 p_err 中，并返回 0

    (4) 检查事件标志组对象类型有效性

  1. 如果 OS_CFG_OBJ_TYPE_CHK_EN 功能启用，检查 p_grp->Type 是否等于 OS_OBJ_TYPE_FLAG 。若不等于，将错误码 OS_ERR_OBJ_TYPE 存储到 p_err 中，并返回 0 。

    (5) 终止等待处理:

  1. 进入临界区，获取事件标志组的挂起列表 p_pend_list。
  2. 检查挂起列表的头部指针是否为空。若为空，说明没有任务在等待该事件标志组，退出临界区，将错误码 OS_ERR_PEND_ABORT_NONE 存储到 p_err 中，并返回 0 。
  3.  初始化任务计数 nbr_tasks 为 0 和时间戳变量 ts 为 0 。
  4. 循环遍历挂起列表，只要挂起列表的头部指针不为空，就获取当前挂起列表头部的 TCB 指针 p_tcb，调用 OS_PendAbort() 函数来终止该任务的等待操作，设置任务的 pend 状态为 OS_STATUS_PEND_ABORT，同时任务计数 nbr_tasks 增加 1 。如果 opt 不包含中止所有任务选项，则在中止一个任务后跳出循环。
  5. 退出临界区

    (6) 任务调度:

  1. 如果 opt 中不包含 OS_OPT_POST_NO_SCHED 选项，调用 OSSched() 进行任务调度，选择下一个最高优先级的就绪任务来运行

    (7) 返回结果:

  1. 将错误码 OS_ERR_NONE 存储到 p_err 中，并返回中止的任务计数。

    6. 返回值

       类型 ：OS_OBJ_QTY

       作用 ：返回终止等待事件标志组的任务数量。如果终止成功，返回终止的任务数；如果没有任务被中止，则返回 0


    7. 健康管理错误码

    无
