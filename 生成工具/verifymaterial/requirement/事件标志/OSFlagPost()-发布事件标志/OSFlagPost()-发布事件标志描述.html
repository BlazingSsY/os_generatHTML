需求描述

需求编号：HLR_OS_05_06

工作描述：HLR：uC/OS-Ⅲ应提供发布事件标志的接口

是否需求：是

验证方法：测试验证

补充信息：

    1. 函数名称

OSFlagPost()---事件标志发布函数

    2. 函数框架

OS_FLAGS OSFlagPost (
    OS_FLAG_GRP *p_grp, /* 输入：指向事件标志组控制块的指针 */
    OS_FLAGS flags, /* 输入：要设置或清除的事件标志位 */
    OS_OPT opt, /* 输入：发布选项 */
    OS_ERR *p_err /* 输出：错误码指针 */
)

    3. 函数描述

该函数用于向事件标志组发布事件标志。可以根据选项设置或清除指定的事件标志位，并唤醒等待该事件标志组的任务。函数返回事件标志组当前的标志位组合。

    4. 参数说明

        (1) p_grp

        类型：OS_FLAG_GRP*

        作用：指向要操作的事件标志组控制块

        (2) flags

        类型：OS_FLAGS

        作用：要设置或清除的标志位掩码

        (3) opt

        类型：OS_OPT

        作用：操作选项，支持以下组合：

                OS_OPT_POST_FLAG_SET：设置标志位

                OS_OPT_POST_FLAG_CLR：清除标志位

                OS_OPT_POST_FLAG_SET | OS_OPT_POST_NO_SCHED: 设置标志位并不立即执行任务调度

                OS_OPT_POST_FLAG_CLR | OS_OPT_POST_NO_SCHED: 清除标志位并不立即执行任务调度

        (4) p_err

        类型：OS_ERR*

        作用：返回操作结果状态

        错误码：

OS_ERR_NONE                //操作成功
OS_ERR_OS_NOT_RUNNING      //内核未运行
OS_ERR_OBJ_PTR_NULL        //无效的对象指针
OS_ERR_OBJ_TYPE            //对象类型错误
OS_ERR_OPT_INVALID         //无效的操作选项
OS_ERR_FLAG_PEND_OPT       //无效的标志等待选项


5. 函数逻辑

    (1) 检查内核是否处于运行状态

  1. 如果 OS_CFG_INVALID_OS_CALLS_CHK_EN 功能启用，判断 OSRunning 是否等于 OS_STATE_OS_RUNNING 。若不等于，说明内核未正常运行，将错误码 OS_ERR_OS_NOT_RUNNING 存储到 p_err 中，并返回 0 。

    (2) 检查事件标志组控制块指针的有效性

  1. 如果 OS_CFG_ARG_CHK_EN 功能启用，检查 p_grp 是否为 NULL。若为 NULL，将错误码 OS_ERR_OBJ_PTR_NULL 存储到 p_err 中，并返回 0 。

    (3) 检查事件标志组控制块的类型

  1. 如果 OS_CFG_OBJ_TYPE_CHK_EN 功能启用，检查 p_grp->Type 是否等于 OS_OBJ_TYPE_FLAG 。若不等于，将错误码 OS_ERR_OBJ_TYPE 存储到 p_err 中，并返回 0 。

    (4)更新事件标志组标志位

    根据 opt 参数的值进行分支判断：

  1. 当opt 为 OS_OPT_POST_FLAG_SET 或 OS_OPT_POST_FLAG_SET | OS_OPT_POST_NO_SCHED 时，进入临界区，将 flags 中指定的标志位设置到事件标志组的标志位中（p_grp->Flags |= flags）。
  2. 当 opt 为 OS_OPT_POST_FLAG_CLR 或 OS_OPT_POST_FLAG_CLR | OS_OPT_POST_NO_SCHED 时进入临界区，将 flags 中指定的标志位清除（p_grp->Flags &= ~flags）。
  3. 其他情况时，设置错误码为 OS_ERR_OPT_INVALID 并返回 0 。

    (5) 处理事件标志组等待列表为空的情况:

  1. 当等待列表为空时，退出临界区，设置错误码为 OS_ERR_NONE 并返回事件标志组当前的标志位组合。

    (6) 处理事件标志组等待列表不为空情况:

          当等待列表不为空时, 遍历等待列表中的每个任务控制块（TCB），对于每个等待任务，根据其等待模式（如等待所有指定标志位设置、等待任意指定标志位设置等）进行判断：

  1. 等待所有标志位设置模式：检查事件标志组中是否所有指定标志位都已设置。如果是，调用 OS_FlagTaskRdy() 将任务设置为就绪状态，并传递匹配的标志位。
  2. 等待任意标志位设置模式：检查事件标志组中是否有任意一个指定标志位已设置。如果是，调用 OS_FlagTaskRdy() 将任务设置为就绪状态，并传递匹配的标志位。
  3. 等待所有标志位清除模式（如果OS_CFG_FLAG_MODE_CLR_EN功能启用）：检查事件标志组中是否所有指定标志位都已清除。如果是，调用 OS_FlagTaskRdy() 将任务设置为就绪状态，并传递匹配的标志位。
  4. 等待任意标志位清除模式（如果OS_CFG_FLAG_MODE_CLR_EN功能启用）：检查事件标志组中是否有任意一个指定标志位已清除。如果是，调用 OS_FlagTaskRdy() 将任务设置为就绪状态，并传递匹配的标志位。
  5. 其他无效模式：退出临界区，设置错误码为 OS_ERR_FLAG_PEND_OPT 并返回 0 。

    (7) 任务调度:

    退出临界区，如果 opt 中不包含 OS_OPT_POST_NO_SCHED 选项，调用 OSSched() 进行任务调度，选择下一个最高优先级的就绪任务来运行。

    (8) 返回当前标志位组合

进入临界区，获取事件标志组当前的标志位组合 。

退出临界区，将错误码 OS_ERR_NONE 存储到 p_err 中，并返回标志位组合。

    6.返回值

        类型 ：OS_FLAGS

        作用 ：返回事件标志组当前的标志位组合。如果函数执行成功，返回事件标志组当前的标志位组合；如果发生错误，返回 0 。


    7. 健康管理错误码

    无
