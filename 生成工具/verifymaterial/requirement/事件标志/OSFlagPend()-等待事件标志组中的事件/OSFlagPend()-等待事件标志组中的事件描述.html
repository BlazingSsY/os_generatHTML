需求描述

需求编号：HLR_OS_05_03

工作描述：HLR：uC/OS-Ⅲ应提供等待事件标志组中的事件的接口

是否需求：是

验证方法：测试验证

补充信息：

    1. 函数名称

OSFlagPend() // 等待事件标志组中的事件的函数

    2. 函数框架

OS_FLAGS OSFlagPend (
    OS_FLAG_GRP *p_grp, /* 输入：指向事件标志组控制块的指针 */
    OS_FLAGS flags, /* 输入：要等待的事件标志 */
    OS_TICK timeout, /* 输入：等待超时时间 */
    OS_OPT opt, /* 输入：等待选项 */
    CPU_TS *p_ts, /* 输出：时间戳指针 */
    OS_ERR *p_err /* 输出：错误码指针 */
    )

    3. 函数描述

    该函数用于等待事件标志组中的事件标志，任务可以指定等待的事件标志组合、等待时间以及等待选项。函数根据不同的等待条件和选项返回相应的事件标志或错误码。

    4. 参数说明

        (1) p_grp

        类型：OS_FLAG_GRP*

        作用：指向事件标志组控制块的指针，用于标识要等待的事件标志组。

        (2) flags

        类型：OS_FLAGS

        作用：指定要等待的事件标志位组合。

        (3) timeout

        类型：OS_TICK

        作用：指定任务等待事件标志的时间。如果在指定时间内事件标志未满足条件，任务将放弃等待。

        (4) opt

        类型：OS_OPT

        作用：等待选项

OS_OPT_PEND_FLAG_CLR_ALL ：等待所有指定的事件标志位被清除。
OS_OPT_PEND_FLAG_CLR_ANY ：等待任意一个指定的事件标志位被清除。
OS_OPT_PEND_FLAG_SET_ALL ：等待所有指定的事件标志位被设置。
OS_OPT_PEND_FLAG_SET_ANY ：等待任意一个指定的事件标志位被设置。

OS_OPT_PEND_FLAG_CLR_ALL | OS_OPT_PEND_FLAG_CONSUME ：等待所有指定的事件标志位被清除，并在条件满足时复位这些标志位。
OS_OPT_PEND_FLAG_CLR_ANY | OS_OPT_PEND_FLAG_CONSUME ：等待任意一个指定的事件标志位被清除，并在条件满足时复位这些标志位。
OS_OPT_PEND_FLAG_SET_ALL | OS_OPT_PEND_FLAG_CONSUME ：等待所有指定的事件标志位被设置，并在条件满足时复位这些标志位。
OS_OPT_PEND_FLAG_SET_ANY | OS_OPT_PEND_FLAG_CONSUME ：等待任意一个指定的事件标志位被设置，并在条件满足时复位这些标志位。
OS_OPT_PEND_FLAG_CLR_ALL | OS_OPT_PEND_NON_BLOCKING ：非阻塞等待所有指定的事件标志位被清除。
OS_OPT_PEND_FLAG_CLR_ANY | OS_OPT_PEND_NON_BLOCKING ：非阻塞等待任意一个指定的事件标志位被清除。
OS_OPT_PEND_FLAG_SET_ALL | OS_OPT_PEND_NON_BLOCKING ：非阻塞等待所有指定的事件标志位被设置。
OS_OPT_PEND_FLAG_SET_ANY | OS_OPT_PEND_NON_BLOCKING ：非阻塞等待任意一个指定的事件标志位被设置。
OS_OPT_PEND_FLAG_CLR_ALL | (OS_OPT_PEND_FLAG_CONSUME | OS_OPT_PEND_NON_BLOCKING) ：非阻塞等待所有指定的事件标志位被清除，并在条件满足时复位这些标志位。
OS_OPT_PEND_FLAG_CLR_ANY | (OS_OPT_PEND_FLAG_CONSUME | OS_OPT_PEND_NON_BLOCKING) ：非阻塞等待任意一个指定的事件标志位被清除，并在条件满足时复位这些标志位。
OS_OPT_PEND_FLAG_SET_ALL | (OS_OPT_PEND_FLAG_CONSUME | OS_OPT_PEND_NON_BLOCKING) ：非阻塞等待所有指定的事件标志位被设置，并在条件满足时复位这些标志位。
OS_OPT_PEND_FLAG_SET_ANY | (OS_OPT_PEND_FLAG_CONSUME | OS_OPT_PEND_NON_BLOCKING) ：非阻塞等待任意一个指定的事件标志位被设置，并在条件满足时复位这些标志位。


        (5) p_ts

        类型：CPU_TS*

        作用：用于返回时间戳。如果传入非 NULL 指针，函数会在成功获得事件标志时将当前时间戳存储到该位置。

        (6) p_err

        类型：OS_ERR*

        作用：用于返回函数执行的错误代码，指示函数调用是否成功以及失败的原因。

        错误码

OS_ERR_NONE: 等待事件标志成功
OS_ERR_PEND_ISR：从中断服务例程调用等待函数且未指定非阻塞选项。
OS_ERR_OS_NOT_RUNNING：内核尚未运行时调用了该函数。
OS_ERR_OBJ_PTR_NULL：传入的事件标志组控制块指针 p_grp 为 NULL。
OS_ERR_OPT_INVALID：传入的 opt 参数不是有效的选项。
OS_ERR_OBJ_TYPE：传入的 p_grp 指向的对象类型不是事件标志组类型。
OS_ERR_PEND_WOULD_BLOCK：指定非阻塞等待但事件标志条件不满足。
OS_ERR_SCHED_LOCKED：调度程序锁定时尝试阻塞等待。
OS_ERR_PEND_ABORT：等待被中止。
OS_ERR_TIMEOUT：等待超时。
OS_ERR_OBJ_DEL：等待的事件标志组被删除。
OS_ERR_STATUS_INVALID：等待状态无效

    5. 函数算法逻辑

    (1) 检查是否在中断环境中调用

  1. 如果 OS_CFG_CALLED_FROM_ISR_CHK_EN 功能启用，检测当前是否处于中断服务例程（ISR）中。若处于 ISR 中且等待选项不是非阻塞模式，将错误码 OS_ERR_PEND_ISR 存储到 p_err 中，并返回 0 。

    (2) 检查内核是否处于运行状态

  1. 如果 OS_CFG_INVALID_OS_CALLS_CHK_EN 功能启用，判断 OSRunning 是否等于 OS_STATE_OS_RUNNING 。若不等于，说明内核未正常运行，将错误码 OS_ERR_OS_NOT_RUNNING 存储到 p_err 中，并返回 0

    (3) 检查参数有效性

      如果 OS_CFG_ARG_CHK_EN 功能启用，则：

  1. 检查事件标志组控制块指针的有效性：，检查 p_grp 是否为 NULL。若为 NULL，将错误码 OS_ERR_OBJ_PTR_NULL 存储到 p_err 中，并返回 0 。
  2. 检查opt参数有效性：检查opt 参数是否为有效的等待选项。若无效，将错误码 OS_ERR_OPT_INVALID 存储到 p_err 中，并返回 0，若有效，则正常处理

    (4) 检查事件标志组对象类型有效性

  1. 如果 OS_CFG_OBJ_TYPE_CHK_EN 功能启用，检查 p_grp->Type 是否等于 OS_OBJ_TYPE_FLAG 。若不等于，将错误码 OS_ERR_OBJ_TYPE 存储到 p_err 中，并返回 0 。

    (5) 根据opt选项进行等待处理

  1. 检查 opt 是否包含 OS_OPT_PEND_FLAG_CONSUME 选项，设置复位标志。
  2. 如果 p_ts 不为 NULL，将其初始化为 0。
  3. 从opt中提取等待模式，进入临界区
  4. 根据等待模式进行分支判断：

                  * 等待所有标志位设置

                           如果事件标志组中所有指定标志位都已设置，根据复位标志进行相应操作（清除已设置的标志位），将匹配的标志位存储到当前任务控制块的 FlagsRdy 中，退出临界区，设置错误码为 OS_ERR_NONE 并返回匹配的标志位

                          如果事件标志组中所有指定标志位没有全部设置，进一步判断是否为非阻塞等待：

                                  若是非阻塞等待，退出临界区，设置错误码为 OS_ERR_PEND_WOULD_BLOCK 并返回 0；

                                  若是阻塞等待，检查调度程序是否锁定：若锁定，退出临界区，设置错误码为 OS_ERR_SCHED_LOCKED 并返回 0；若未锁定，调用 OS_FlagBlock() 将任务阻塞并加入事件标志组的等待列表，然后退出临界区。

                  * 等待任意标志位设置

                           如果事件标志组中任意一个指定标志位已设置：根据复位标志位进行相应操作（清除已设置的标志位），将匹配的标志位存储到当前任务控制块的 FlagsRdy 中，退出临界区，设置错误码为 OS_ERR_NONE 并返回匹配的标志位。

                           如果事件标志组中没有任何一个指定标志位被设置，后续处理逻辑与等待所有标志位设置模式相同，包括判断非阻塞等待、调度程序锁定情况以及调用 OS_FlagBlock() 阻塞任务。

                   * 等待所有标志位清除（如果OS_CFG_FLAG_MODE_CLR_EN功能启用）

                            如果事件标志组所有指定标志位都已清除，根据复位标志进行相应操作（设置已清除的标志位），将匹配的标志位存储到当前任务控制块的 FlagsRdy 中，退出临界区，设置错误码为 OS_ERR_NONE 并返回匹配的标志位。

                            如果事件标志组指定标志位没有全部被清除，后续处理逻辑与等待所有标志位设置模式相同。

                    * 等待任意标志位清除模式（如果OS_CFG_FLAG_MODE_CLR_EN功能启用）

                            如果事件标志组任意一个指定标志位已清除。如果是，根据复位标志进行相应操作（设置已清除的标志位），将匹配的标志位存储到当前任务控制块的 FlagsRdy 中，退出临界区，设置错误码为 OS_ERR_NONE 并返回匹配的标志位。

                             如果事件标志组没有任何指定标志位被清除，后续处理逻辑与等待所有标志位设置模式相同。

                    * 其他无效模式

                             退出临界区，设置错误码为 OS_ERR_OPT_INVALID 并返回 0。

  1. 执行任务调度

    (6) 处理等待结果

         重新进入临界区后，根据当前任务的等待状态（OSTCBCurPtr->PendStatus）进行处理，处理完成后，如果错误码不为OS_ERR_NONE，将直接返回0：

  1. 若等待状态为 OS_STATUS_PEND_OK

                   表示成功获取事件标志，设置错误码为 OS_ERR_NONE。

  1. 若等待状态为 OS_STATUS_PEND_ABORT

                   表示等待被中止，退出临界区，设置错误码为 OS_ERR_PEND_ABORT。

  1. 若等待状态为 OS_STATUS_PEND_TIMEOUT

                   表示等待超时，退出临界区，设置错误码为 OS_ERR_TIMEOUT，并将时间戳设置为 0。

  1. 若挂起状态为 OS_STATUS_PEND_DEL

                  表示等待的事件标志组被删除，退出临界区，设置错误码为 OS_ERR_OBJ_DEL。

  1. 其他无效状态

                  退出临界区，设置错误码为 OS_ERR_STATUS_INVALID。

    (7) 检查复位标志并根据opt选项进行复位:

         如果复位标志被设置，根据opt选项中的等待模式进行复位：

  1. 等待所有标志位设置：清除已设置的标志位
  2. 等待任意标志位设置：清除已设置的标志位
  3. 等待所有标志位清除：设置已清除的标志位
  4. 等待任意标志位清除模式：设置已清除的标志位
  5. 其他无效模式：退出临界区，返回错误码OS_ERR_OPT_INVALID，返回值为0

    (8) 退出临界段并返回:

  1. 退出临界段。
  2. 设置错误码为OS_ERR_NONE
  3. 返回事件标志位

    6. 返回值

类型 ：OS_FLAGS

作用 ：返回匹配的事件标志位。如果成功获取事件标志，返回匹配的标志位组合；如果因超时、中止等原因未获取事件标志，返回 0。

    7. 健康管理错误码

    无
