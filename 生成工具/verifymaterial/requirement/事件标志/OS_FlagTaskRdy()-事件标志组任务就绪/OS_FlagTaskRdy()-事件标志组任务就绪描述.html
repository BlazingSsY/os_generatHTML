需求描述

需求编号：HLR_OS_05_09

工作描述：HLR：uC/OS-Ⅲ应提供使事件标志关联任务变为就绪状态的接口

是否需求：是

验证方法：测试验证

补充信息：

1. 函数名称

OS_FlagTaskRdy ()--- 事件标志组任务就绪处理函数

    2. 函数框架

void OS_FlagTaskRdy (
    OS_TCB *p_tcb, /* 输入：任务控制块指针 */
    OS_FLAGS flags_rdy, /* 输入：已就绪的标志位 */
    CPU_TS ts /* 输入：时间戳 */
)

    3. 函数描述

    该函数是 uC/OS-III 的内部函数，用于用于处理因事件标志组条件满足而需要转为就绪状态的任务，使其转为就绪态并将其从挂起列表中移除

    4. 参数说明

        (1) p_tcb

        类型：OS_TCB*

        作用：指向要设置为就绪状态的任务控制块的指针。

        (2) flags_rdy

        类型：OS_FLAGS

        作用：表示匹配的事件标志位组合，这些标志位导致任务等待结束。

        (3) ts

        类型：CPU_TS

        作用：时间戳，用于记录任务等待结束的时间。

    5. 函数逻辑

    (1) 更新任务控制块，设置就绪标志

    将任务匹配标志 (FlagsRdy) 设置为传入的 flags_rdy。

    清除任务的等待状态 (PendStatus = OS_STATUS_PEND_OK)。

    设置 PendOn 为 OS_TASK_PEND_ON_NOTHING，指示任务不再等待任何对象

    (2) 任务状态处理:

  根据任务当前的状态（TaskState）进行分支判断：

  1. 如果任务处于等待状态（OS_TASK_STATE_PEND）或等待超时状态（OS_TASK_STATE_PEND_TIMEOUT):

              * 如果OS_CFG_TICK_EN 启用且任务处于等待超时状态，则将其从节拍列表中移除。

              * 通过调用 OS_RdyListInsert(p_tcb)将任务插入到就绪列表中，。

              * 更新任务的状态为就绪状态（OS_TASK_STATE_RDY）。

  1. 如果任务处于挂起等待状态（OS_TASK_STATE_PEND_SUSPENDED）或挂起等待超时状态（OS_TASK_STATE_PEND_TIMEOUT_SUSPENDED）:

              * 如果OS_CFG_TICK_EN 启用且任务处于挂起等待超时状态，则将其从节拍列表中移除。

              * 更新任务的状态为挂起状态（OS_TASK_STATE_SUSPENDED）。

  1. 其他状态: 不进行任何操作。

    (3) 从等待列表中移除任务:

    调用 OS_PendListRemove函数将任务从等待列表中移除，表示任务不再等待事件标志组。

    6. 返回值

    无

    7. 健康管理错误码

    无

    7. 注意事项

    此函数是 uC/OS-III 的内部函数，不能被用户直接调用
