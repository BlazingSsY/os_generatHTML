需求描述

需求编号：HLR_OS_05_05

工作描述：HLR：uC/OS-Ⅲ应提供获取当前任务等待到的事件标志的接口

是否需求：是

验证方法：测试验证

补充信息：

1. 函数名称

OSFlagPendGetFlagsRdy() ---获取当前任务等待到的事件

    2. 函数框架

OSFlagPendGetFlagsRdy (
    OS_ERR *p_err /*out: 错误码指针*/
    )

    3. 函数描述

    该函数用于获取当前任务等待到的事件标志。它通常在任务成功等待事件标志后调用，以获取实际匹配的标志位。

    4. 参数说明

        (1) p_err

        类型：OS_ERR*

        作用：用于返回函数执行的错误代码，指示函数调用是否成功。

        错误码：

OS_ERR_NONE                 //成功等待到的获取事件标志
OS_ERR_OS_NOT_RUNNING       //内核尚未运行时调用了该函数。
OS_ERR_PEND_ISR             //从中断服务例程调用该函数


    5. 函数逻辑

    (1) 检查内核是否处于运行状态

  1. 如果 OS_CFG_INVALID_OS_CALLS_CHK_EN 功能启用，判断 OSRunning 是否等于 OS_STATE_OS_RUNNING 。若不等于，说明内核未正常运行，将错误码 OS_ERR_OS_NOT_RUNNING 存储到 p_err 中，并返回 0 。

    (2) 检查是否在中断环境中调用

  1. 如果 OS_CFG_CALLED_FROM_ISR_CHK_EN 功能启用，检测当前是否处于中断服务例程（ISR）中。若处于 ISR 中，将错误码 OS_ERR_PEND_ISR 存储到 p_err 中，并返回 0 。

    (3) 获取当前任务等待到的事件标志

  1. 进入临界区，获取当前任务控制块（OSTCBCurPtr）中的 FlagsRdy 值，该值表示任务等待到的事件标志位组合。
  2. 退出临界区，将错误码 OS_ERR_NONE 存储到 p_err 中，并返回获取到的事件标志。

    6. 返回值

       类型 ：OS_FLAGS

       作用 ：返回当前任务等待到事件标志位。如果函数执行成功，返回等待到事件标志位；如果发生错误，返回 0 。

    7. 健康管理错误码

    无
