需求描述

需求编号：HLR_OS_04_01

工作描述：HLR：uC/OS-Ⅲ应提供互斥量创建接口。

是否需求：是

验证方法：测试验证

补充信息：

1.函数名称

OSMutexCreate() --- 互斥量创建函数

2.函数框架

void OSMutexCreate (
    OS_MUTEX *p_mutex, /* 输入：指向互斥量控制块的指针 */
    CPU_CHAR *p_name, /* 输入：互斥量的名称 */
    OS_ERR *p_err /* 输出：错误码指针 */
)

3.函数描述

该函数用于创建一个互斥量。互斥量用于保护共享资源，确保同一时间只有一个任务可以访问该资源。函数会初始化互斥量控制块，并将其标记为可用状态

4.参数说明

    (1) p_mutex

        类型：OS_MUTEX*

        作用：指向互斥量控制块的指针，用于管理互斥量。

    (2) p_name

        类型：CPU_CHAR*

        作用：互斥量的名称，用于调试和跟踪。

    (3) p_err

        类型：OS_ERR*

        作用：用于返回函数执行的错误代码，指示函数调用是否成功以及失败的原因。

        错误码：

OS_ERR_NONE：函数执行成功，无错误。
OS_ERR_CREATE_ISR：从中断服务例程调用该函数，这是不允许的。
OS_ERR_OBJ_PTR_NULL：传入的互斥量控制块指针无效。
OS_ERR_OBJ_CREATED：互斥量控制块已被创建。


5.函数算法逻辑

(1)检查是否在中断环境中调用

如果 OS_CFG_CALLED_FROM_ISR_CHK_EN 功能启用，检测当前是否处于中断服务例程（ISR）中。若处于 ISR 中，将错误码 OS_ERR_CREATE_ISR 存储到 p_err 中，并返回。


(2)检查互斥量控制块指针的有效性

如果 OS_CFG_ARG_CHK_EN 功能启用，检查 p_mutex 是否为 NULL。若为 NULL，将错误码 OS_ERR_OBJ_PTR_NULL 存储到 p_err 中，并返回。


(3)对象类型检查

进入临界区。

如果对象类型检查（OS_OBJ_TYPE_REQ 和创建对象检查（OS_CFG_OBJ_CREATED_CHK_EN启用，检查互斥量控制块的类型是否已设置为互斥量类型。如果是，退出临界区，将错误码 OS_ERR_OBJ_CREATED 存储到 p_err 中，并返回。


(4)初始化互斥量控制块

将互斥量控制块的类型设置为互斥量类型（OS_OBJ_TYPE_MUTEX）。

初始化互斥量的下一个互斥量指针为 NULL，表示该互斥量未链接到其他互斥量。

将互斥量的所有者任务控制块指针设置为 NULL，表示当前没有任务拥有该互斥量。

将互斥量的所有者嵌套计数设置为 0，表示该互斥量未被任何任务锁定。

调用 OS_PendListInit() 函数初始化互斥量的挂起列表，该列表用于管理等待该互斥量的任务。


(5)函数返回

退出临界区，将错误码 OS_ERR_NONE 存储到 p_err 中。

6.返回值

无

7.健康管理错误码

无
