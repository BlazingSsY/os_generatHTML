需求描述

需求编号：HLR_OS_04_10

工作描述：HLR：uC/OS-Ⅲ应提供释放任务互斥量组中所有互斥量的接口。

是否需求：是

验证方法：测试验证

补充信息：

1.函数名称

OS_MutexGrpPostAll() - 释放任务互斥量组中所有互斥量

2.函数框架

void OS_MutexGrpPostAll (
    OS_TCB *p_tcb /* 输入：指向任务控制块的指针 */
)

3.函数描述

该函数用于释放任务拥有的所有互斥量，并将这些互斥量授予等待任务（如果有）。它会遍历任务的互斥量组链表，逐个处理每个互斥量。

4.参数说明

    (1)p_tcb

        类型：OS_TCB*

        作用：指向任务控制块的指针，用于标识要释放其拥有的所有互斥量的任务。

5.函数算法逻辑

遍历目标任务的互斥量组链表

  1. 调用 OS_MutexGrpRemove() 函数将当前互斥量从任务的互斥量组中移除
  2. 检查互斥量等待列表

                   获取互斥量的等待列表 p_pend_list。

                   如果等待列表为空：

                           将互斥量的所有者嵌套计数设置为 0。

                           将互斥量的所有者任务控制块指针设置为 NULL。

                   如果等待列表不为空：

                          获取等待列表头部的TCB 指针p_tcb_new。

                          将互斥量的所有者任务控制块指针更新为 p_tcb_new。

                          将互斥量的所有者嵌套计数设置为 1。

                          调用 OS_MutexGrpAdd() 函数将互斥量添加到新所有者的互斥量组中。

                         调用 OS_Post() 函数唤醒等待任务。

7.返回值

    无

8.健康管理错误码

    无

备注：此函数是 uC/OS-III 的内部函数，应用程序不应直接调用它。
