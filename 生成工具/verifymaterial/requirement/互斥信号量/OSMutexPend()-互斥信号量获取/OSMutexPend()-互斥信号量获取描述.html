需求描述

需求编号：HLR_OS_04_03

工作描述：HLR：uC/OS-Ⅲ应提供互斥量获取接口。

是否需求：是

验证方法：测试验证

补充信息：

1.函数名称

OSMutexPend() ---互斥量获取函数

2.函数框架

void OSMutexPend (
    OS_MUTEX *p_mutex, /* 输入：指向互斥量控制块的指针 */
    OS_TICK timeout, /* 输入：等待超时时间 */
    OS_OPT opt, /* 输入：等待选项 */
    CPU_TS *p_ts, /* 输出：时间戳 */
    OS_ERR *p_err /* 输出：错误码指针 */
)

3.函数描述

该函数用于等待获取一个互斥量。如果互斥量当前不可用，任务可以指定等待的超时时间和等待模式（阻塞或非阻塞）。函数根据不同的等待结果返回相应的错误码。

4.参数说明

    (1) p_mutex

        类型：OS_MUTEX*

        作用：指向互斥量控制块的指针，用于标识要等待的互斥量。

    (2) timeout

        类型：OS_TICK

        作用：指定任务等待互斥量的最长时间。如果在指定时间内互斥量不可用，任务将放弃等待。

    (3) opt

        类型：OS_OPT

        作用：指定等待选项，可以是以下两种取值：

            OS_OPT_PEND_BLOCKING：阻塞等待，任务会挂起直到互斥量可用或超时。

            OS_OPT_PEND_NON_BLOCKING：非阻塞等待，如果互斥量不可用，任务不会挂起，函数立即返回。

    (4) p_ts

        类型：CPU_TS*

        作用：用于返回时间戳（如果需要）。如果传入非 NULL 指针，函数会在成功获取互斥量时将当前时间戳存储到该位置。

    (5) p_err

        类型：OS_ERR*

        作用：用于返回函数执行的错误代码，指示函数调用是否成功以及失败的原因。

        错误码：

OS_ERR_NONE： 函数执行成功，无错误。
OS_ERR_PEND_ISR：    从中断服务例程调用等待函数且未指定非阻塞选项。
OS_ERR_OS_NOT_RUNNING：    内核尚未运行时调用了该函数。
OS_ERR_OBJ_PTR_NULL：    传入的互斥量控制块指针无效。
OS_ERR_OPT_INVALID：    传入的 opt 参数不是有效的选项。
OS_ERR_OBJ_TYPE：    传入的互斥量控制块类型不正确。
OS_ERR_PEND_WOULD_BLOCK：    指定非阻塞等待但互斥量不可用。
OS_ERR_SCHED_LOCKED：    调度程序锁定时尝试阻塞等待。
OS_ERR_PEND_ABORT：    等待被中止。
OS_ERR_TIMEOUT：    等待超时。
OS_ERR_OBJ_DEL：    等待的互斥量被删除。
OS_ERR_STATUS_INVALID：    挂起状态无效。
OS_ERR_MUTEX_OVF：    互斥量嵌套计数溢出。
OS_ERR_MUTEX_OWNER：    当前任务已是互斥量的所有者。


5.函数算法逻辑

(1)检查是否在中断环境中调用

如果 OS_CFG_CALLED_FROM_ISR_CHK_EN 功能启用，检测当前是否处于中断服务例程（ISR）中。若处于 ISR 中，将错误码 OS_ERR_PEND_ISR 存储到 p_err 中，并返回。


(2)检查内核是否处于运行状态

如果 OS_CFG_INVALID_OS_CALLS_CHK_EN 功能启用，判断 OSRunning 是否等于 OS_STATE_OS_RUNNING 。若不等于，说明内核未正常运行，将错误码 OS_ERR_OS_NOT_RUNNING 存储到 p_err 中，并返回。


(3)检查互斥量控制块指针的有效性

如果 OS_CFG_ARG_CHK_EN 功能启用，检查 p_mutex 是否为 NULL。若为 NULL，将错误码 OS_ERR_OBJ_PTR_NULL 存储到 p_err 中，并返回。


(4)检查等待选项

如果 OS_CFG_ARG_CHK_EN 功能启用，检查 opt 参数： 若为OS_OPT_PEND_BLOCKING和OS_OPT_PEND_NON_BLOCKING，则正常处理； 其他选项，则将错误码 OS_ERR_OPT_INVALID 存储到 p_err 中，并返回。


(5)检查互斥量控制块的类型

如果 OS_CFG_OBJ_TYPE_CHK_EN 功能启用，检查 p_mutex->Type 是否等于 OS_OBJ_TYPE_MUTEX 。若不等于，将错误码 OS_ERR_OBJ_TYPE 存储到 p_err 中，并返回。

(6)检查互斥量是否可用

  1. 进入临界区。
  2. 检查互斥量的所有者嵌套计数是否为 0。如果是，表示互斥量可用：

                   将互斥量的所有者任务控制块指针设置为当前任务控制块（OSTCBCurPtr）。

                   将互斥量的所有者嵌套计数设置为 1。

                   将当前任务添加到互斥量所有者组。

                   退出临界区，将错误码 OS_ERR_NONE 存储到 p_err 中，并返回。

(7)检查当前任务是否已持有该互斥量：

                  检查互斥量的嵌套计数是否达到最大值（OS_NESTING_CTR 的最大值）。如果是，退出临界区，将错误码                 OS_ERR_MUTEX_OVF 存储到 p_err 中，并返回。

                 增加互斥量的所有者嵌套计数。

                 退出临界区，将错误码 OS_ERR_MUTEX_OWNER 存储到 p_err 中，并返回。

(8) 如果互斥量不可用且等待选项为非阻塞模式：

                  退出临界区，将错误码 OS_ERR_PEND_WOULD_BLOCK 存储到 p_err 中，并返回。

(9）如果互斥量不可用且等待选项为阻塞模式

  1. 检查调度器是否被锁定，若已锁定，则退出临界区，将错误码 OS_ERR_SCHED_LOCKED 存储到 p_err 中，并返回。

(10) 判断是否需要提升互斥量持有者任务的优先级

  1. 如果互斥量的所有者任务的优先级低于当前任务，则提升所有者任务的优先级至当前任务的优先级。

(11) 阻塞当前任务

  1. 调用 OS_Pend() 函数将当前任务阻塞，等待互斥量可用或超时。

(12)执行任务调度

退出临界区，调用 OSSched() 进行任务调度，选择下一个最高优先级的就绪任务来运行。

(13)处理等待结果

重新进入临界区，根据当前任务的挂起状态（OSTCBCurPtr->PendStatus）设置相应的错误码：

  1. OS_STATUS_PEND_OK：成功获取互斥量，设置错误码为 OS_ERR_NONE。
  2. OS_STATUS_PEND_ABORT：等待被中止，设置错误码为 OS_ERR_PEND_ABORT。
  3. OS_STATUS_PEND_TIMEOUT：等待超时，设置错误码为 OS_ERR_TIMEOUT。
  4. OS_STATUS_PEND_DEL：等待的互斥量被删除，设置错误码为 OS_ERR_OBJ_DEL。
  5. 其他状态：设置错误码为 OS_ERR_STATUS_INVALID。

退出临界区。

6.返回值

无

7.健康管理错误码

无

备注：该函数不能在定时器回调函数中调用
