需求描述

需求编号：HLR_OS_04_02

工作描述：HLR：uC/OS-Ⅲ应提供互斥量删除接口。

是否需求：是

验证方法：测试验证

补充信息：

1.函数名称

OSMutexDel() ---互斥量删除函数

2.函数框架

OS_OBJ_QTY OSMutexDel (
    OS_MUTEX *p_mutex, /* 输入：指向互斥量控制块的指针 */
    OS_OPT opt, /* 输入：删除选项 */
    OS_ERR *p_err /* 输出：错误码指针 */
)

3.函数描述

该函数用于删除一个互斥量。它可以有条件地删除互斥量（仅当没有任务挂起时）或无条件地删除互斥量（强制删除并处理挂起任务）。

4.参数说明

    (1) p_mutex

        类型：OS_MUTEX*

        作用：指向要删除的互斥量控制块的指针。

    (2) opt

        类型：OS_OPT

        作用：指定删除选项。可以是以下两种取值：

                OS_OPT_DEL_NO_PEND：仅在没有任务因等待该互斥量而挂起时才删除。

                OS_OPT_DEL_ALWAYS：无论如何都删除互斥量。如果存在等待该互斥量的任务，会将这些任务从挂起列表中移除，并设置它们的 pend 状态为 OS_STATUS_PEND_DEL，表示因互斥量被删除而终止等待。

    (3) p_err

        类型：OS_ERR*

        作用：用于返回函数执行的错误代码，指示函数调用是否成功以及失败的原因。

        错误码：

OS_ERR_NONE：函数执行成功，无错误。
OS_ERR_DEL_ISR：试图从中断服务例程（ISR）中删除互斥量，这是不允许的。
OS_ERR_OS_NOT_RUNNING：内核尚未运行时调用了该函数。
OS_ERR_OBJ_PTR_NULL：传入的互斥量控制块指针 p_mutex 为 NULL。
OS_ERR_OBJ_TYPE：传入的 p_mutex 指向的对象类型不是互斥量类型。
OS_ERR_TASK_WAITING：当 opt 为 OS_OPT_DEL_NO_PEND 时，发现有任务正在等待该互斥量，所以无法删除。
OS_ERR_OPT_INVALID：传入的 opt 参数不是有效的选项。


5.函数算法逻辑

(1)检查是否在中断环境中调用

如果 OS_CFG_CALLED_FROM_ISR_CHK_EN 功能启用，检测当前是否处于中断服务例程（ISR）中。若处于 ISR 中，将错误码 OS_ERR_DEL_ISR 存储到 p_err 中，并返回 0 。


(2)检查内核是否处于运行状态

如果 OS_CFG_INVALID_OS_CALLS_CHK_EN 功能启用，判断 OSRunning 是否等于 OS_STATE_OS_RUNNING 。若不等于，说明内核未正常运行，将错误码 OS_ERR_OS_NOT_RUNNING 存储到 p_err 中，并返回 0 。


(3)检查互斥量控制块指针的有效性

如果 OS_CFG_ARG_CHK_EN 功能启用，检查 p_mutex 是否为 NULL。若为 NULL，将错误码 OS_ERR_OBJ_PTR_NULL 存储到 p_err 中，并返回 0 。


(4)检查互斥量控制块的类型

如果 OS_CFG_OBJ_TYPE_CHK_EN 功能启用，检查 p_mutex->Type 是否等于 OS_OBJ_TYPE_MUTEX 。若不等于，将错误码 OS_ERR_OBJ_TYPE 存储到 p_err 中，并返回 0 。


(5)根据删除选项处理互斥量删除

进入临界区。

获取互斥量的挂起列表 p_pend_list 和初始化任务计数 nbr_tasks 为 0。

根据 opt 参数的值进行分支判断：

    当 opt 为 OS_OPT_DEL_NO_PEND时，检查等待列表的头部指针是否为空：

  1. 若为空，表示没有任务在等待该互斥量。如果互斥量有持有者任务，将其从持有者任务的互斥量组中移除。调用 OS_MutexClr() 清除互斥量，退出临界区，设置错误码为 OS_ERR_NONE 并返回任务计数0
  2. 若不为空，退出临界区，设置错误码为 OS_ERR_TASK_WAITING 并返回 0 。

    当 opt 为 OS_OPT_DEL_ALWAYS时，初始化时间戳变量 ts 为 0：

  1. 循环遍历等待列表，只要等待列表的头部指针不为空，就获取当前等待列表头部的 TCB 指针 p_tcb，然后调用 OS_PendAbort() 函数来中止任务的等待操作，设置任务的 pend 状态为 OS_STATUS_PEND_DEL，同时任务计数 nbr_tasks 增加 1。
  2. 如果互斥量有持有者任务，将其从持有者任务的互斥量组中移除。
  3. 如果持有者任务的当前优先级不等于其基础优先级，计算新的优先级并调用 OS_TaskChangePrio() 更新任务优先级。
  4. 调用 OS_MutexClr() 清除互斥量，退出临界区
  5. 调用 OSSched() 进行任务调度，设置错误码为 OS_ERR_NONE 并返回任务计数。

    其他情况：退出临界区，设置错误码为 OS_ERR_OPT_INVALID 并返回 0 。


6.返回值

    类型 ：OS_OBJ_QTY

    作用 ：返回被中止等待该互斥量的任务数量。如果是无条件删除（OS_OPT_DEL_ALWAYS），返回被中止的任务数；如果是仅在无任务等待时删除（OS_OPT_DEL_NO_PEND）且删除成功，则返回 0；如果删除失败（如有任务等待且选择 OS_OPT_DEL_NO_PEND），也返回 0

7.健康管理错误码

无
