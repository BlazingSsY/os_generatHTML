需求描述

需求编号：HLR_OS_04_06

工作描述：HLR：uC/OS-Ⅲ应提清除互斥量的接口。

是否需求：是

验证方法：测试验证

补充信息：

1.函数名称

OS_MutexClr() ---互斥量清除函数

2.函数框架

void OS_MutexClr (
    OS_MUTEX *p_mutex /* 输入：指向互斥量控制块的指针 */
)

3.函数描述

该函数用于清除互斥量控制块，将其重置为初始状态。将互斥量的类型设置为无类型、重置互斥量的所有者和嵌套计数，并初始化互斥量的等待列表。

4.参数说明

    (1)p_mutex

        类型: OS_MUTEX*

        作用：指向需要清除的互斥量对象

5.函数算法逻辑

  1. 如果OS_OBJ_TYPE_REQ功能启用，将互斥量控制块的 Type 成员设置为 OS_OBJ_TYPE_NONE，表示该数据结构不再是一个互斥量。
  2. 将互斥量的 MutexGrpNextPtr 成员设置为 NULL，表示该互斥量不再链接到其他互斥量。
  3. 将互斥量的 OwnerTCBPtr 成员设置为 NULL，表示当前没有任务拥有该互斥量。
  4. 将互斥量的 OwnerNestingCtr 成员设置为 0，表示该互斥量的嵌套计数已重置。
  5. 调用 OS_PendListInit() 函数初始化互斥量的等待列表

6.返回值

  1. 无

7.健康管理错误码

  1. 无

备注：此函数是 uC/OS-III 的内部函数，应用程序不应直接调用它。
