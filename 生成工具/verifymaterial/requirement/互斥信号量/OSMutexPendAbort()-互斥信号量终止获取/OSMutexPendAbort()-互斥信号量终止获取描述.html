需求描述

需求编号：HLR_OS_04_04

工作描述：HLR：uC/OS-Ⅲ应提供互斥量终止获取的接口。

是否需求：是

验证方法：测试验证

补充信息：

1.函数名称

OSMutexPendAbort() ---互斥量终止获取函数

2.函数框架

OS_OBJ_QTY OSMutexPendAbort (
    OS_MUTEX *p_mutex, /* 输入：指向互斥量控制块的指针 */
    OS_OPT opt, /* 输入：中止选项 */
    OS_ERR *p_err /* 输出：错误码指针 */
)

3.函数描述

该函数用于中止一个或多个任务对互斥量的等待。可以通过选项指定是中止一个任务还是所有任务，并且可以选择是否进行任务调度。

4.参数说明

    (1) p_mutex

       类型：OS_MUTEX*

        作用：指向互斥量控制块的指针，用于标识要中止等待的互斥量。

    (2) opt

        类型：OS_OPT

        作用：指定中止选项。可以是以下几种取值：

            OS_OPT_PEND_ABORT_1：中止一个等待任务。

            OS_OPT_PEND_ABORT_ALL：中止所有等待任务。

            OS_OPT_PEND_ABORT_1 | OS_OPT_POST_NO_SCHED：中止一个等待任务且不进行任务调度。

            OS_OPT_PEND_ABORT_ALL | OS_OPT_POST_NO_SCHED：中止所有等待任务且不进行任务调度。

    (3) p_err

        类型：OS_ERR*

        作用：用于返回函数执行的错误代码，指示函数调用是否成功以及失败的原因。

        错误码：

OS_ERR_NONE：函数执行成功，无错误。
OS_ERR_PEND_ABORT_ISR：从中断服务例程调用该函数。
OS_ERR_OS_NOT_RUNNING：内核尚未运行时调用了该函数。
OS_ERR_OBJ_PTR_NULL：传入的互斥量控制块指针无效。
OS_ERR_OPT_INVALID：传入的 opt 参数不是有效的选项。
OS_ERR_OBJ_TYPE：传入的互斥量控制块类型不正确。
OS_ERR_PEND_ABORT_NONE：没有任务在等待该互斥量。

5.函数算法逻辑

(1)检查是否在中断环境中调用

如果 OS_CFG_CALLED_FROM_ISR_CHK_EN 功能启用，检测当前是否处于中断服务例程（ISR）中。若处于 ISR 中，将错误码 OS_ERR_PEND_ABORT_ISR 存储到 p_err 中，并返回 0 。

(2)检查内核是否处于运行状态

如果 OS_CFG_INVALID_OS_CALLS_CHK_EN 功能启用，判断 OSRunning 是否等于 OS_STATE_OS_RUNNING 。若不等于，说明内核未正常运行，将错误码 OS_ERR_OS_NOT_RUNNING 存储到 p_err 中，并返回 0 。

(3)检查互斥量控制块指针的有效性

如果 OS_CFG_ARG_CHK_EN 功能启用，检查 p_mutex 是否为 NULL。若为 NULL，将错误码 OS_ERR_OBJ_PTR_NULL 存储到 p_err 中，并返回 0 。

(4)检查中止选项的有效性

如果 OS_CFG_ARG_CHK_EN 功能启用，验证 opt 参数是否为有效的中止选项。若无效，将错误码 OS_ERR_OPT_INVALID 存储到 p_err 中，并返回 0 。

(5)检查互斥量控制块的类型

如果 OS_CFG_OBJ_TYPE_CHK_EN 功能启用，检查 p_mutex->Type 是否等于 OS_OBJ_TYPE_MUTEX 。若不等于，将错误码 OS_ERR_OBJ_TYPE 存储到 p_err 中，并返回 0 。

(6)检查等待列表是否为空

进入临界区。

获取互斥量的等待列表 p_pend_list。

检查等待列表是否为空。若为空，退出临界区，将错误码 OS_ERR_PEND_ABORT_NONE 存储到 p_err 中，并返回 0 。

(7)中止等待处理并调整优先级

设置化任务计数 nbr_tasks 为 0 和时间戳变量 ts 为 0 。

循环遍历等待列表，只要等待列表的头部指针不为空，就获取当前等待列表头部的 TCB 指针 p_tcb，调用 OS_PendAbort() 函数来中止该任务的等待操作。

获取互斥量的所有者任务控制块指针 p_tcb_owner 和新的优先级 prio_new。

如果所有者任务的当前优先级不是其基础优先级，并且当前优先级等于被中止任务的优先级：

  1. 调用 OS_MutexGrpPrioFindHighest 函数获取互斥量任务组的最高优先级。
  2. 如果计算出的新优先级低于所有者任务的基础优先级，则将新优先级设置为基础优先级。
  3. 如果新优先级不等于所有者任务的当前优先级，则调用 OS_TaskChangePrio() 函数更新所有者任务的优先级。

增加任务计数 nbr_tasks。

如果 opt 不是中止所有任务选项，则在中止一个任务后跳出循环。

退出临界区。

(8)任务调度

如果 opt 中不包含 OS_OPT_POST_NO_SCHED 选项，调用 OSSched() 进行任务调度，选择下一个最高优先级的就绪任务来运行。

(9)设置返回值

将错误码 OS_ERR_NONE 存储到 p_err 中，并返回中止的任务计数 nbr_tasks。

6.返回值

类型 ：OS_OBJ_QTY

作用 ：返回中止等待互斥量的任务数量。如果中止成功，返回中止的任务数；如果等待列表为空且没有任务被中止，则返回 0 。

7.健康管理错误码

无
