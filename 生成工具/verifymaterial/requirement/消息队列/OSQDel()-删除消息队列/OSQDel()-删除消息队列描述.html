需求描述

需求编号：HLR_OS_02_02

工作描述：HLR：uC/OS-Ⅲ应提供消息队列删除接口，以删除指定的消息队列，释放相关资源。

是否需求：是

验证方法：测试验证

补充信息：

1.函数名称

OSQDel()  ---  删除消息队列                         

2.函数框架

OS_OBJ_QTY  OSQDel
(
OS_Q       *p_q,                   /* 输入: 消息队列指针                    */
OS_OPT      opt,                   /* 输入: 删除操作选项                    */
OS_ERR     *p_err                  /* 指向变量的指针，该变量将包含此函数返回的错误代码                           */
)

3.函数描述

    OSQDel()函数用于删除一个消息队列。执行删除后，消息队列的所有信息会被系统清空，无法再次使用。

4.参数说明

(1) p_q

类型：OS_Q*

作用：指向消息队列的指针

(2) opt

类型：OS_OPT

作用：执行的中止类型：

OS_OPT_DEL_NO_PEND：只有在没有任务等待时才删除消息队列。

OS_OPT_DEL_ALWAYS：无论是否有任务等待，都删除消息队列。在这种情况下，所有等待任务将被置于就绪状态。

(3) p_err

类型：OS_ERR*

作用：返回操作错误码

错误类型：

OS_ERR_NONE                        /* 无错误    */
OS_ERR_DEL_ISR                     /* 不可在中断中执行操作   */
OS_ERR_OBJ_PTR_NULL                /* 消息队列指针为空          */
OS_ERR_OBJ_TYPE                    /* 消息队列不存在                        */
OS_ERR_OPT_INVALID                 /* 删除操作选项无效                      */
OS_ERR_OS_NOT_RUNNING              /* UCOS Ⅲ未运行                        */
OS_ERR_TASK_WAITING                /* 一个或多个任务正在等待该消息队列       */

5.函数算法逻辑

（1）中断上下文检查

  1. 如果OS_CFG_CALLED_FROM_ISR_CHK_EN启用且当前中断嵌套计数器OSIntNestingCtr > 0，表示在ISR中调用，设置错误码OS_ERR_DEL_ISR，返回0

（2）系统运行状态检查

  1. 如果OS_CFG_INVALID_OS_CALLS_CHK_EN启用，检查内核是否运行，若未运行，设置错误码OS_ERR_OS_NOT_RUNNING，返回0

（3）参数有效性检查

  1. 如果OS_CFG_ARG_CHK_EN启用, 检查p_q是否为NULL，若是则设置错误码OS_ERR_OBJ_PTR_NULL，返回0

（4）对象类型检查

  1. 如果OS_CFG_OBJ_TYPE_CHK_EN启用，检查p_q->Type是否为OS_OBJ_TYPE_Q，若不是，设置错误码OS_ERR_OBJ_TYPE，返回0


（5）根据删除选项处理消息队列删除

  1. * 进入临界区，获取消息队列的等待列表和初始化任务计数。
  2. * 根据 opt 参数分支判断：
  3.     当 opt 为 OS_OPT_DEL_NO_PEND 时
  4.     检查等待列表是否为空。若为空，调用OS_QClr函数清空消息队列的内容，退出临界区，设置错误码为OS_ERR_NONE 并返回任务计数（0）。若不为空，退出临界区，设置错误码为 OS_ERR_TASK_WAITING 并返回 0。
  5.     当 opt 为 OS_OPT_DEL_ALWAYS 时
  6. 遍历等待列表，调用OS_PendAbort函数移除所有等待任务，增加任务计数。
  7. 调用OS_QClr函数清空消息队列的内容，退出临界区，调度任务，设置错误码为OS_ERR_NONE 并返回任务计数。
  8.     其他情况
  9. 退出临界区，设置错误码为 OS_ERR_OPT_INVALID
  10. * 返回终止的等待任务个数


6 返回值

  1.  类型 ：nbr_tasks
  2. 作用 ：返回被中止等待队列的任务的数量。如果是无条件删除（OS_OPT_DEL_ALWAYS），返回被中止的任务数；如果是仅在无任务等待时删除（OS_OPT_DEL_NO_PEND）且删除成功，则返回 0.


7.健康管理错误码:

    无

   备注：

 1) 未被定义的消息队列无法删除。

 2) 当有任务正在等待消息，则不应进行删除操作，
