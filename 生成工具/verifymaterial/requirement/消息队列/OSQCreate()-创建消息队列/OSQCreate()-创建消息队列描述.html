需求描述

需求编号：HLR_OS_02_01

工作描述：HLR：uC/OS-Ⅲ应提供消息队列创建接口，以创建具有指定优先级和选项的消息队列。

是否需求：是

验证方法：测试验证

补充信息：

1.函数名称

OSQCreate()                          创建消息队列                                  

2.函数框架

void OSQCreate
(
OS_Q        *p_q,                    /* 输入:消息队列指针                             */
CPU_CHAR    *p_name,                 /* 输入:消息队列名称                             */
OS_MSG_QTY   max_qty,                /* 输入:消息队列大小（不能为0）                   */
OS_ERR      *p_err                   /* 指向变量的指针，该变量将包含此函数返回的错误代码                    */
)

3.函数描述

       OSQCreate()函数用于创建一个新的队列。队列就是一个数据结构， 用于任务间的数据的传递。

4.参数说明

(1) p_q

类型：OS_Q*

作用：指向消息队列的指针

(2) p_name

类型：CPU_CHAR *

作用：消息队列名称

(3) max_qty

类型：OS_MSG_QTY

作用：消息队列大小

(4) p_err

类型：OS_ERR*

作用：返回操作错误码

错误码：

OS_ERR_NONE                          /* 无错误                                        */
OS_ERR_CREATE_ISR                    /* 不可在中断中执行操作                           */
OS_ERR_OBJ_PTR_NULL                  /* 消息队列指针为空                               */
OS_ERR_Q_SIZE                        /* 消息队列大小为0                                */
OS_ERR_OBJ_CREATED  /* 消息队列已经创建 */

5.函数算法逻辑

（1）中断上下文检查

  1. 如果OS_CFG_CALLED_FROM_ISR_CHK_EN启用且OSIntNestingCtr > 0u，表示在ISR中调用，返回OS_ERR_CREATE_ISR

（2）参数有效性检查

  1. 如果OS_CFG_ARG_CHK_EN启用:
  2. 检查p_q是否为NULL，若是则返回OS_ERR_OBJ_PTR_NULL
  3. 验证max_qty参数是否合法，如果max_qty为0，则返回OS_ERR_Q_SIZE

（3）对象类型检查

            进入临界区。

  1. 如果OS_OBJ_TYPE_REQ > 0u且OS_CFG_OBJ_CREATED_CHK_EN > 0u的话，检查p_q->Type是否为OS_OBJ_TYPE_Q，若是的话，退出临界区，返回OS_ERR_OBJ_TYPE

（4）初始化队列数据结构

​设置队列类型：

标记 p_q->Type = OS_OBJ_TYPE_Q，表明该数据结构为消息队列。

​初始化消息队列：

调用 OS_MsgQInit()，初始化消息链表，设置队列最大容量。

​初始化等待链表：

调用 OS_PendListInit()，初始化任务等待链表，用于管理等待消息的任务。

（5）返回结果

​退出临界区

设置错误码: 通过 *p_err 返回操作结果，返回值为OS_ERR_NONE，表示创建成功。


6 . 返回值

  1. 无

7.健康管理错误码

    无

   备注：要使用 μC/OS的消息队列必须先声明和创建消息队列。
