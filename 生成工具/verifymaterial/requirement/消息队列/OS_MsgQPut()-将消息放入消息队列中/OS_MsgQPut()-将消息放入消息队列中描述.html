需求描述

需求编号：HLR_OS_02_0﻿9

工作描述：HLR：uC/OS-Ⅲ应提供向消息队列插入消息的接口。

是否需求：是

验证方法：测试验证

补充信息：

1.函数名称

OS_MsgQPut()                        向消息队列插入消息                                 

2.函数框架

void OS_MsgQPut (
    OS_MSG_Q *p_msg_q, /* 输入：指向消息队列控制块的指针 */
    void *p_void, /* 输入：指向要发送的消息的指针 */
    OS_MSG_SIZE msg_size, /* 输入：消息的大小 */
    OS_OPT opt, /* 输入：发送选项 */
    CPU_TS ts, /* 输入：时间戳 */
    OS_ERR *p_err /* 输出：错误码指针 */
)
3.

3. 函数描述

该函数用于将消息放入向消息队列中。如果消息队列已满或消息池中没有空闲消息块，函数会立即返回相应的错误码。

4. 参数说明

(1) p_msg_q

类型：OS_MSG_Q*

作用：指向消息队列中消息链表控制块的指针，用于标识要发送消息的消息队列的消息链表。

(2) p_void

类型：void*

作用：指向要发送的消息的指针。

(3) msg_size

类型：OS_MSG_SIZE

作用：消息的大小。

(4) opt

类型：OS_OPT

作用：指定发送选项，可以是以下取值：

OS_OPT_POST_FIFO：以先进先出（FIFO）顺序添加消息。

OS_OPT_POST_LIFO：以后进先出（LIFO）顺序添加消息。

(5) ts

类型：CPU_TS

作用：时间戳，用于记录消息发送的时间。

(6) p_err

类型：OS_ERR*

作用：用于返回函数执行的错误码，指示函数调用是否成功以及失败的原因。

错误码：

OS_ERR_Q_MAX：消息队列已满，无法添加更多消息。

OS_ERR_MSG_POOL_EMPTY：消息池中没有空闲消息块。

OS_ERR_NONE：函数执行成功，无错误。


5. 函数算法逻辑

（1）检查消息队列是否已满

如果消息队列中的消息数量（p_msg_q->NbrEntries）大于等于消息队列的最大消息数量（p_msg_q->NbrEntriesSize），将错误码 OS_ERR_Q_MAX 存储到 p_err 中，并返回。

（2）检查消息池中是否有空闲消息块

如果消息池的空闲消息块数量（OSMsgPool.NbrFree）为 0，将错误码 OS_ERR_MSG_POOL_EMPTY 存储到 p_err 中，并返回。

（3）向消息队列添加消息

(a)从消息池获取空闲消息块

获取消息池空闲链表的第一个消息块（OSMsgPool.NextPtr）。

更新消息池空闲链表的头部指针为该消息块的下一个消息块。

减少消息池的空闲消息块计数（OSMsgPool.NbrFree）。

增加消息池的已用消息块计数（OSMsgPool.NbrUsed）。

（b）处理消息队列为空的情况

如果消息队列当前为空（p_msg_q->NbrEntries == 0u）：

将消息队列的消息链表的入队指针（InPtr）和出队指针（OutPtr）都设置为获取到的消息块。

将消息队列的消息数量（NbrEntries）设置为 1。

将消息块的下一个指针（NextPtr）设置为 NULL。

（c）处理消息队列不为空的情况

根据发送选项决定消息添加位置：

如果是先进先出（FIFO）模式：

将新消息块的下一个指针设置为当前入队指针所指消息块的下一个指针。

更新入队指针所指消息块的下一个指针为新消息块。

更新入队指针为新消息块。

如果是后进先出（LIFO）模式：

将新消息块的下一个指针设置为当前出队指针所指消息块。

更新消息队列的出队指针为新消息块。

增加消息队列的消息数量（NbrEntries）。

（d）存储消息内容和大小

将消息指针（MsgPtr）和消息大小（MsgSize）存储到新消息块中。

（e）设置返回值

   将错误码 OS_ERR_NONE 存储到 p_err 中，表示消息发送成功。

6 返回值

    无

7.健康管理错误码

    无
