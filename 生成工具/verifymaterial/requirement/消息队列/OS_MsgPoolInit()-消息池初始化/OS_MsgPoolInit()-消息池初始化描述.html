需求描述

需求编号：HLR_OS_02_12

工作描述：HLR：uC/OS-Ⅲ应提供系统消息池初始化接口，以便初始化系统消息池。

是否需求：是

验证方法：测试验证

补充信息：

1.函数名称

OS_MsgPoolInit()---消息池初始化                                

2.函数框架

void OS_MsgPoolInit(
OS_ERR *p_err,   /*输出：错误码指针 */

)

3.函数描述

该函数用于初始化系统的消息池。消息池是一组预分配的消息缓冲区，任务可以从中申请和释放消息缓冲区。

4.参数说明

(1) p_err

类型：OS_ERR *

作用：用于返回函数执行的错误码，指示函数调用是否成功以及失败的原因。

错误码：

OS_ERR_MSG_POOL_NULL_PTR：消息池基址指针为空。
OS_ERR_MSG_POOL_EMPTY：消息池大小为 0。
OS_ERR_NONE：函数执行成功，无错误。


5.函数算法逻辑

（1）检查消息池基址指针的有效性

如果 OS_CFG_ARG_CHK_EN 功能启用，检查 OSCfg_MsgPoolBasePtr 是否为 NULL。若为 NULL，将错误码 OS_ERR_MSG_POOL_NULL_PTR 存储到 p_err 中，并返回。

（2）检查消息池大小的有效性

如果 OS_CFG_ARG_CHK_EN 功能启用，检查 OSCfg_MsgPoolSize 是否为 0。若为 0，将错误码 OS_ERR_MSG_POOL_EMPTY 存储到 p_err 中，并返回。

（3）初始化消息池中的所有消息节点后，,更新消息池状态并设置返回值。

  1. 获取消息池的起始消息节点指针 p_msg1 和下一个消息节点指针 p_msg2。
  2. 循环遍历消息池中的消息节点（除最后一个外）：

                  将当前消息节点 p_msg1 的 NextPtr 指针设置为下一个消息节点 p_msg2。

                  将当前消息节点的 MsgPtr 和 MsgSize 成员初始化为 0。

                  移动 p_msg1 和 p_msg2 指针到下一个消息节点。

  1. 将最后一个消息节点的 NextPtr 指针设置为 NULL。
  2. 将最后一个消息节点的 MsgPtr 和 MsgSize 成员初始化为 0。
  3. 将消息池的 NextPtr 设置为消息池的基址指针，表示空闲消息链表的头部。
  4. 将消息池的 NbrFree 设置为消息池的大小，表示所有消息节点都处于空闲状态。
  5. 将消息池的 NbrUsed 设置为 0，表示当前没有消息节点被使用。

     设置返回值

将错误码 OS_ERR_NONE 存储到 p_err 中，表示消息池初始化成功,返回消息指针。


6 返回值

    无

7.健康管理错误码

    无

备注：该函数是内部函数，不能被用户直接调用
