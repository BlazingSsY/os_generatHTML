需求描述

需求编号：HLR_OS_03_04

工作描述：HLR：uC/OS-Ⅲ应提供终止等待获取信号量的接口

是否需求：是

验证方法：测试验证

补充信息：

1.函数名称

OSSemPendAbort（） -- 信号量等待终止函数

2.函数框架

OS_OBJ_QTY OSSemPendAbort (
    OS_SEM *p_sem, /* 输入：信号量控制块指针 */
    OS_OPT opt, /* 输入：终止选项 */
    OS_ERR *p_err /* 输出：错误码 */
)

3.函数描述

OSSemPendAbort（）用于强制终止正在等待指定信号量的任务，可选择终止一个或全部等待任务

4.参数说明

        (1) p_sem

        类型：OS_SEM*

        作用：指向要操作的信号量控制块

        (2) opt

        类型：OS_OPT

        作用：中止选项：

                OS_OPT_PEND_ABORT_1：中止一个等待任务

                OS_OPT_PEND_ABORT_ALL：中止所有等待任务

                OS_OPT_PEND_ABORT_1 | OS_OPT_POST_NO_SCHED: 中止一个等待任务，不立即触发调度

                OS_OPT_PEND_ABORT_ALL | OS_OPT_POST_NO_SCHED: 中止一个等待任务，不立即触发调度

        (3) p_err

        类型：OS_ERR*

        作用：返回操作错误码

        错误码：

OS_ERR_NONE：操作成功

OS_ERR_PEND_TERMINATE_ISR：在ISR中调用

OS_ERR_OS_NOT_RUNNING：内核未运行

OS_ERR_OBJ_PTR_NULL：参数p_sem为NULL

OS_ERR_OPT_INVALID：无效的opt参数

OS_ERR_OBJ_TYPE：对象类型错误

OS_ERR_PEND_TERMINATE_NONE：无等待任务可终止

5.函数算法逻辑

(1) 中断上下文检查

若OS_CFG_CALLED_FROM_ISR_CHK_EN启用且当前中断嵌套计数器OSIntNestingCtr > 0，表示在ISR中调用，设置错误码OS_ERR_PEND_ABORT_ISR，并立即返回0

(2) 系统运行状态检查

若系统配置宏OS_CFG_INVALID_OS_CALLS_CHK_EN启用且内核状态OSRunning != OS_STATE_OS_RUNNING，设置错误码OS_ERR_OS_NOT_RUNNING，立即返回0

(3) 参数有效性检查

若系统配置宏OS_CFG_ARG_CHK_EN启用：

检查p_sem是否为NULL指针，若是，设置错误码OS_ERR_OBJ_PTR_NULL并返回0

检查opt参数是否合法，若opt不合法，设置错误码OS_ERR_OPT_INVALID并返回0

    允许的选项值：

        OS_OPT_PEND_ABORT_1

        OS_OPT_PEND_ABORT_ALL

        上述选项与OS_OPT_POST_NO_SCHED的组合

(4) 对象类型检查

若系统配置宏OS_CFG_OBJ_TYPE_CHK_EN启用，检查p_sem->Type 是否为 OS_OBJ_TYPE_SEM，若不是，设置错误码OS_ERR_OBJ_TYPE并返回0

(5) 等待任务检查

进入临界区，

获取信号量的等待列表指针

检查等待列表是否为空，若为空，退出临界区，设置错误码OS_ERR_PEND_ABORT_NONE并返回0

(6) 终止等待任务处理

初始化被终止任务计数器nbr_tasks = 0

初始化时间戳ts = 0

循环处理等待列表中的任务：

    获取列表中的第一个任务TCB指针，调用OS_PendAbort()终止该任务：

    增加终止计数器nbr_tasks++

    检查终止选项：

        若opt为OS_OPT_PEND_ABORT_1，只终止一个任务后退出循环

        若opt为OS_OPT_PEND_ABORT_ALL，继续处理下一个任务

退出临界区（CPU_CRITICAL_EXIT()）

(7) 调度控制

检查是否设置了OS_OPT_POST_NO_SCHED标志，若未设置，调用OSSched()触发任务调度；若设置，跳过调度步骤

(8) 返回结果

设置错误码OS_ERR_NONE

返回被终止的任务数量nbr_tasks

6. 返回值

类型：OS_OBJ_QTY

作用：返回实际被终止的任务数量

7.健康管理错误码

  1. 无
