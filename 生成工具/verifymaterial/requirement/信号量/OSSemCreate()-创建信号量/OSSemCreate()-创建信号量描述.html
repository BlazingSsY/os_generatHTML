需求描述

需求编号：HLR_OS_03_01

工作描述：HLR：uC/OS-Ⅲ应提供信号量创建接口，以创建二值信号量或计数信号量。

是否需求：是

验证方法：测试验证

补充信息：

1. 函数名称

OSSemCreate() —— 信号量创建函数

2. 函数框架

void OSSemCreate (
    OS_SEM *p_sem,     /* 输入：信号量控制块指针 */
    CPU_CHAR *p_name,    /* 输入：信号量名称（调试用） */
    OS_SEM_CTR cnt,       /* 输入：信号量初始计数值 */
    OS_ERR *p_err /* 输出：错误码 */);

3. 函数描述

用于创建一个二值信号量或计数信号量，并初始化其计数值和等待列表。

4. 参数说明

(1) p_sem

  1. 类型：OS_SEM*
  2. 作用：指向信号量控制块的指针，用于管理信号量的状态和等待任务列表。

(2) p_name

  1. 类型：CPU_CHAR*
  2. 作用：信号量名称字符串（调试用），仅在 OS_CFG_DBG_EN 宏启用时有效。

(3) cnt

  1. 类型：OS_SEM_CTR
  2. 作用：信号量的初始计数值：
  3. 0：资源不可用，任务获取信号量时将阻塞等待
  4. 1：资源可用，实现二值信号量（互斥访问）
  5. >1：资源可用，实现计数信号量（允许多次获取）

(4) p_err

  1. 类型：OS_ERR*
  2. 作用：返回操作错误码
  3. 错误码：
  4. OS_ERR_NONE：创建成功
  5. OS_ERR_CREATE_ISR：禁止在中断服务程序（ISR）中调用
  6. OS_ERR_OBJ_PTR_NULL：p_sem 参数为 NULL
  7. OS_ERR_OBJ_CREATED：信号量已创建（重复初始化）

5. 函数算法逻辑

        （1） 中断上下文检查

  1. 如果 OS_CFG_CALLED_FROM_ISR_CHK_EN 启用，检测是否在 ISR 中调用，若是则返回 OS_ERR_CREATE_ISR

        （2）参数有效性检查

  1. 如果 OS_CFG_ARG_CHK_EN 启用，检查 p_sem 是否为 NULL，若是则返回 OS_ERR_OBJ_PTR_NULL

       （3）信号量类型检查与标记

  1. 进入临界区
  2. 如果 OS_OBJ_TYPE_REQ 和 OS_CFG_OBJ_CREATED_CHK_EN 启用：
  3. 检查 p_sem->Type 是否为 OS_OBJ_TYPE_SEM，若是则退出临界区并返回 OS_ERR_OBJ_CREATED

       （4） 初始化信号量属性

  1. 否则标记 p_sem->Type = OS_OBJ_TYPE_SEM
  2. 设置初始计数值 p_sem->Ctr = cnt

     （5）初始化等待列表

  1. 调用 OS_PendListInit(&p_sem->PendList) 初始化等待列表

     （6） 返回成功状态

  1. 退出临界区
  2. 设置错误码 *p_err = OS_ERR_NONE

6. 返回值

  1. 类型：void
  2. 作用：无直接返回值，通过 p_err 输出操作状态

7. 健康管理错误码

  1. 无（错误码已通过 p_err 返回）
