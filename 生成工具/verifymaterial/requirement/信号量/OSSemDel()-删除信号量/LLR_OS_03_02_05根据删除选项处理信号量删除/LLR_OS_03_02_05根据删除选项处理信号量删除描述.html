需求描述

需求编号 ：LLR_OS_03_02_05

需求描述 ：根据删除选项处理信号量删除操作。

1.进入临界区后，获取信号量的等待列表并初始化任务计数。

2.若删除选项为 OS_OPT_DEL_NO_PEND ，则检查等待列表是否为空。若为空，调用 OS_SemClr() 清除信号量，退出临界区，设置错误码为 OS_ERR_NONE 并返回任务计数（0）；若不为空，退出临界区，设置错误码为 OS_ERR_TASK_WAITING 并返回 0 。

3.若删除选项为 OS_OPT_DEL_ALWAYS ，并循环遍历等待列表，获取当前等待列表头部的 TCB 指针，调用 OS_PendAbort() 函数来中止任务的等待操作，将任务从等待列表中移除，同时任务计数增加 1 。循环结束后，调用 OS_SemClr() 清除信号量，退出临界区，然后调用 OSSched() 进行任务调度，设置错误码为 OS_ERR_NONE 并返回任务计数。

4.若删除选项既不是 OS_OPT_DEL_NO_PEND 也不是 OS_OPT_DEL_ALWAYS ，则设置错误码为 OS_ERR_OPT_INVALID 并返回 0 。
