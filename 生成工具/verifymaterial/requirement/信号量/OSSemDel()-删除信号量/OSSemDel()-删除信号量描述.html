需求描述

需求编号：HLR_OS_03_02

工作描述：HLR：uC/OS-Ⅲ应提供信号量删除接口，以删除指定信号量

是否需求：是

验证方法：测试验证

补充信息：

1. 函数名称

OSSemDel()---删除信号量

2. 函数框架

OS_OBJ_QTY OSSemDel (
    OS_SEM *p_sem,      /* 输入：指向要删除的信号量控制块的指针 */
    OS_OPT  opt,        /* 输入：删除选项，指示如何处理挂起任务 */
    OS_ERR *p_err      /* 输出：错误代码 */
   )



3. 函数描述

该函数用于删除一个信号量。

4. 参数说明

(1) p_sem

  1. 类型：OS_SEM*
  2. 作用：指向要删除的信号量控制块的指针。该控制块是在创建信号量时由 uCOS - III分配的，用于管理信号量的相关信息。

(2) opt

  1. 类型：OS_OPT
  2. 作用：指定删除信号量的选项。主要有两种取值：
  3. OS_OPT_DEL_NO_PEND：仅在没有任务因等待该信号量而挂起时才删除信号量。
  4. OS_OPT_DEL_ALWAYS：无论如何都删除信号量。如果存在等待该信号量的任务，会将这些任务从挂起列表中移除，并设置它们的 pend 状态为 OS_STATUS_PEND_DEL，表示因信号量被删除而终止等待。

(3) p_err

  1. 类型：OS_ERR*
  2. 作用：用于返回函数执行的错误代码，指示函数调用是否成功以及失败的原因。函数执行过程中会根据不同的错误情况将相应的错误码存储到 p_err 指向的变量中。
  3. 错误码：
OS_ERR_DEL_ISR：试图从中断服务例程（ISR）中删除信号量，这是不允许的。
OS_ERR_OS_NOT_RUNNING：内核尚未运行时调用了该函数。
OS_ERR_OBJ_PTR_NULL：传入的信号量控制块指针 p_sem 为 NULL，即无效指针。
OS_ERR_OBJ_TYPE：传入的 p_sem 指向的对象类型不是信号量类型。
OS_ERR_TASK_WAITING：当 opt 为 OS_OPT_DEL_NO_PEND 时，发现有任务正在等待该信号量，所以无法删除。
OS_ERR_OPT_INVALID：传入的 opt 参数不是有效的选项。


5. 函数算法逻辑

（1）检查是否从中断服务例程调用

  1. 如果宏 OS_CFG_CALLED_FROM_ISR_CHK_EN 启用，检查 OSIntNestingCtr 是否大于 0 。若大于 0 ，说明处于中断环境中，不能删除信号量，设置错误码为OS_ERR_DEL_ISR 并返回 0 。

（2） 检查内核是否已经运行

  1. 如果宏 OS_CFG_INVALID_OS_CALLS_CHK_EN 启用，判断 OSRunning 是否不等于 OS_STATE_OS_RUNNING 。若不等于，内核未正常运行，设置错误码为 OS_ERR_OS_NOT_RUNNING 并返回 0 。

（3） 验证传入的信号量控制块指针

  1. 如果宏 OS_CFG_ARG_CHK_EN 启用，检查 p_sem 是否为 NULL。若为 NULL，指针无效，设置错误码为OS_ERR_OBJ_PTR_NULL 并返回 0 。

（4） 验证信号量控制块的类型

  1. 如果宏 OS_CFG_OBJ_TYPE_CHK_EN 启用，检查 p_sem->Type 是否等于 OS_OBJ_TYPE_SEM 。若不等于，控制块类型错误，设置错误码为 OS_ERR_OBJ_TYPE 并返回 0 。

（5）根据删除选项处理信号量删除

  1. 进入临界区。
  2. 获取信号量的等待列表并始化任务计数。
  3. 根据 opt 参数分支判断：
  4. 当 opt 为 OS_OPT_DEL_NO_PEND 时
  5. 检查等待列表是否为空。若为空，清除信号量，退出临界区，设置错误码为OS_ERR_NONE 并返回任务计数（0）。若不为空，退出临界区，设置错误码为 OS_ERR_TASK_WAITING 并返回 0。
  6. 当 opt 为 OS_OPT_DEL_ALWAYS 时
  7. 遍历等待列表，调用OS_PendAbort函数移除所有等待任务，增加任务计数。
  8. 清除信号量，退出临界区，调度任务，设置错误码为OS_ERR_NONE 并返回任务计数。
  9. 其他情况
  10. 退出临界区，设置错误码为 OS_ERR_OPT_INVALID 并返回 0。

6. 返回值

  1. 类型 ：OS_OBJ_QTY
  2. 作用 ：返回被中止等待该信号量的任务的数量。如果是无条件删除（OS_OPT_DEL_ALWAYS），返回被中止的任务数；如果是仅在无任务等待时删除（OS_OPT_DEL_NO_PEND）且删除成功，则返回 0；如果删除失败（如有任务等待且选择 OS_OPT_DEL_NOEND_P），也返回 0。

7. 健康管理错误码

无
