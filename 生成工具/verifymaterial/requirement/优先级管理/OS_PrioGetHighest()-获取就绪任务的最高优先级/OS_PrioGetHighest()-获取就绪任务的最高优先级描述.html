需求描述

需求编号：HLR_OS_10_02

工作描述：HLR:uC/OS-Ⅲ应提供获取所有就绪任务的最高优先级的接口

是否需求：是

验证方法：测试验证

补充信息：

1. 函数名称：OS_PrioGetHighest ()— 获取就绪任务的最高优先级

2. 函数框架

OS_PRIO OS_PrioGetHighest (void)

3. 函数描述

该函数用于从优先级表位图中获取被置为1最高位所代表的优先级，即获取所有就绪任务中的最高优先级。

4. 参数说明

无参数

5. 函数算法逻辑

根据系统配置的最大优先级OS_CFG_PRIO_MAX与 CPU 数据宽度（CPU_CFG_DATA_SIZE * 8u）的关系来决定获取最高优先级的算法

•若OS_CFG_PRIO_MAX小于等于 2 倍 CPU 数据宽度(系统默认)：

 •  • 若OSPrioTbl[0]  == 0（无任务在高优先级范围就绪），则计算OSPrioTbl[1]的前导零数量，并加上 CPU 数据宽度，得到最高优先级。

 •  • 若OSPrioTbl[0]  >  0，直接计算OSPrioTbl[0]的前导零数量作为最高优先级。

6. 返回值

类型：OS_PRIO

作用：返回等待事件的最高优先级任务的优先级（值越小，优先级越高）。

7. 健康管理错误码

无
