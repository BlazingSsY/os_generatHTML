需求描述

需求编号：HLR_OS_07_02

工作描述：HLR：uC/OS-Ⅲ应提申请内存块的接口。

是否需求：是

验证方法：测试验证

补充信息：

1.函数名称

OSMemGet ()---内存块申请函数

2.函数框架

void  *OSMemGet (
    OS_MEM *p_mem, /* 输入：指向内存池控制块的指针 */
    OS_ERR *p_err /* 输出：错误码指针 */
)

3.函数描述

该函数用于从指定的内存池中获取一个空闲内存块。如果内存池中有空闲块，函数会返回该块的指针；如果没有空闲块，则返回 NULL 并设置相应的错误码。

4.参数说明

    (1) p_mem

        类型：OS_MEM*

        作用：指向内存池控制块的指针，用于标识要从中获取内存块的内存池。

    (2) p_err

        类型：OS_ERR*

        作用：用于返回函数执行的错误代码，指示函数调用是否成功以及失败的原因。

        错误码：

OS_ERR_NONE：空闲内存块申请成功
OS_ERR_MEM_INVALID_P_MEM：传入的内存池控制块指针无效。
OS_ERR_MEM_NO_FREE_BLKS：内存池没有空闲内存块可用
OS_ERR_OBJ_TYPE：传入的内存池控制块类型不正确。


5.函数算法逻辑

(1)检查内存池控制块指针的有效性

如果 OS_CFG_ARG_CHK_EN 功能启用，检查 p_mem 是否为 NULL。若为 NULL，将错误码OS_ERR_MEM_INVALID_P_MEM 存储到 p_err 中，并返回 NULL。

(2)检查内存池控制块的类型

如果 OS_CFG_OBJ_TYPE_CHK_EN 功能启用，检查 p_mem->Type 是否等于 OS_OBJ_TYPE_MEM。若不等于，将错误码OS_ERR_OBJ_TYPE 存储到 p_err 中，并返回 NULL。

(3)获取空闲内存块

  1. 进入临界区。
  2. 检查内存池中的空闲块数量 p_mem->NbrFree 是否为 0。若为 0，说明没有空闲块，退出临界区，将错误码 OS_ERR_MEM_NO_FREE_BLKS 存储到 p_err 中，并返回 NULL。
  3. 获取空闲链表的第一个内存块指针 p_blk，并更新空闲链表指针为该块指向的下一个空闲块。
  4. 减少空闲块计数 p_mem->NbrFree。
  5. 退出临界区，将错误码 OS_ERR_NONE 存储到 p_err 中，并返回获取到的内存块指针 p_blk。

6.返回值

    类型 ：void*

    作用 ：返回获取到的内存块的指针。如果成功获取内存块，返回该块的起始地址；如果内存分区中没有空闲块或发生错误，返回 NULL。

7.健康管理错误码

无
