需求描述

需求编号：HLR_OS_07_03

工作描述：HLR：uC/OS-Ⅲ应提释放内存块的接口。

是否需求：是

验证方法：测试验证

补充信息：

1.函数名称

OSMemPut ()---内存块释放函数

2.函数框架

void OSMemPut (
    OS_MEM *p_mem, /* 输入：指向内存池控制块的指针 */
    void *p_blk, /* 输入：要释放的内存块的指针 */
    OS_ERR *p_err /* 输出：错误码指针 */
)

3.函数描述

该函数用于将一个内存块释放回指定的内存池。函数会将内存块添加到空闲链表中，并更新内存池的空闲块计数。

4.参数说明

    (1) p_mem

        类型：OS_MEM*

        作用：指向内存池控制块的指针，用于标识要将内存块释放回的内存池。

    (2) p_blk

        类型：void*

        作用：要释放的内存块的指针。

    (3) p_err

        类型：OS_ERR*

        作用：用于返回函数执行的错误代码，指示函数调用是否成功以及失败的原因。

        错误码：

OS_ERR_NONE：空闲内存块成功释放
OS_ERR_MEM_INVALID_P_MEM：传入的内存池控制块指针无效。
OS_ERR_MEM_INVALID_P_BLK：传入的内存块指针无效。
OS_ERR_OBJ_TYPE：传入的内存池控制块类型不正确。
OS_ERR_MEM_FULL：内存池已满，无法释放更多内存块。


5.函数算法逻辑

(1)参数检查

如果 OS_CFG_ARG_CHK_EN 功能启用：

  1. 检查 p_mem 是否为 NULL。若为 NULL，将错误码 OS_ERR_MEM_INVALID_P_MEM 存储到 p_err 中，并返回。
  2. 检查 p_blk 是否为 NULL。若为 NULL，将错误码 OS_ERR_MEM_INVALID_P_BLK 存储到 p_err 中，并返回。

(2)检查内存池控制块的类型

如果 OS_CFG_OBJ_TYPE_CHK_EN 功能启用，检查 p_mem->Type 是否等于 OS_OBJ_TYPE_MEM。若不等于，将错误码 OS_ERR_OBJ_TYPE 存储到 p_err 中，并返回。

(3)释放内存块

  1. 进入临界区。
  2. 检查内存池中的空闲块数量 p_mem->NbrFree 是否大于等于最大块数量 p_mem->NbrMax。如果是，说明内存池已满，无法再释放内存块，退出临界区，将错误码 OS_ERR_MEM_FULL 存储到 p_err 中，并返回。
  3. 将要释放的内存块插入到空闲链表的头部。具体操作是：将当前空闲链表的第一个块的指针存储到 p_blk 所指向的内存位置，然后将 p_blk 设置为空闲链表的第一个块。
  4. 增加空闲块计数 p_mem->NbrFree。
  5. 退出临界区，将错误码 OS_ERR_NONE 存储到 p_err 中。
