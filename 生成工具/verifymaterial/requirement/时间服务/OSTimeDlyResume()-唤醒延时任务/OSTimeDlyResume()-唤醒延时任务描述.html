需求描述

需求编号：HLR_OS_08_06

工作描述：HLR:uC/OS-Ⅲ应提供唤醒延时任务的接口

是否需求：是

验证方法：测试验证

补充信息：

1.函数名称

OSTimeDlyResume() — 唤醒延时任务

2.函数框架

void OSTimeDlyResume (
    OS_TCB  *p_tcb,   /* 输入：待唤醒任务的任务控制块指针 */
    OS_ERR  *p_err    /* 输出：错误码指针 */
)

3.函数描述

该函数用于提前唤醒一个因调用 OSTimeDly() 或 OSTimeDlyHMSM() 而处于延时状态的任务。

4.参数说明

(1) p_tcb

类型：OS_TCB*

作用：指向需要被唤醒的任务控制块。

(2) p_err

类型：OS_ERR*

作用：返回的错误码：

OS_ERR_NONE — 任务已成功唤醒
OS_ERR_TCB_INVALID — 输入p_tcb 为 NULL
OS_ERR_OS_NOT_RUNNING — 内核未启动
OS_ERR_TIME_DLY_RESUME_ISR — 在中断中调用
OS_ERR_TASK_NOT_DLY — 任务并非处于延时状态
OS_ERR_TASK_SUSPENDED — 任务处于“延时并挂起”状态，仅解除延时
OS_ERR_STATE_INVALID — 任务状态异常

5.函数算法逻辑

(1) 检查是否在中断环境中调用

若 OS_CFG_CALLED_FROM_ISR_CHK_EN 启用且 OSIntNestingCtr > 0u，返回 OS_ERR_TIME_DLY_RESUME_ISR。

(2) 检查任务控制块合法性

若 OS_CFG_ARG_CHK_EN 启用且 p_tcb 为 NULL，返回 OS_ERR_TCB_INVALID。

(3) 检查内核是否运行

若 OS_CFG_INVALID_OS_CALLS_CHK_EN 启用且 OSRunning != OS_STATE_OS_RUNNING，返回 OS_ERR_OS_NOT_RUNNING。

(4) 依据任务状态分支处理

进入临界区后，根据目标任务的状态TaskState进行处理：

• OS_TASK_STATE_RDY：

• • 退出临界区，设置错误码为OS_ERR_TASK_NOT_DLY

• OS_TASK_STATE_PEND：

• • 退出临界区，设置错误码为OS_ERR_TASK_NOT_DLY

• OS_TASK_STATE_PEND_TIMEOUT：

• • 退出临界区，设置错误码为OS_ERR_TASK_NOT_DLY

• OS_TASK_STATE_SUSPENDED：

• • 退出临界区，设置错误码为OS_ERR_TASK_NOT_DLY

• OS_TASK_STATE_PEND_SUSPENDED：

• • 退出临界区，设置错误码为OS_ERR_TASK_NOT_DLY

• OS_TASK_STATE_PEND_TIMEOUT_SUSPENDED：

• • 退出临界区，设置错误码为OS_ERR_TASK_NOT_DLY

• OS_TASK_STATE_DLY：

• • 将任务状态置为 OS_TASK_STATE_RDY

• • 若系统节拍功能启用，调用 OS_TickListRemove() 从延时列表移除

• • 若系统节拍功能启用，调用 OS_RdyListInsert() 加入就绪列表

• • 退出临界区，设置错误码为OS_ERR_NONE

•OS_TASK_STATE_DLY_SUSPENDED：

• • 将状态置为 OS_TASK_STATE_SUSPENDED

• •若系统节拍功能启用，调用 OS_TickListRemove()

• • 退出临界区，设置错误码为OS_ERR_TASK_SUSPENDED

•其它状态

• • 退出临界区， 设置错误码为OS_ERR_STATE_INVALID

(5) 触发任务调度

调用 OSSched() 执行任务调度。

6. 返回值

无 

7.健康管理错误码

无
