需求描述

需求编号：HLR_OS_08_05

工作描述：HLR:uC/OS-Ⅲ应提供时分秒毫秒延时当前任务的接口

是否需求：是

验证方法：测试验证

补充信息：

1.函数名称

OSTimeDlyHMSM() - 时分秒毫秒延时当前任务

2.函数框架

void OSTimeDlyHMSM (
    CPU_INT16U   hours,   /* 输入：小时（0-99/999，取决于选项） */
    CPU_INT16U   minutes,  /* 输入：分钟（0-59/9999，取决于选项） */
    CPU_INT16U   seconds,  /* 输入：秒（0-59） */
    CPU_INT32U   milli,   /* 输入：毫秒（0-999） */
    OS_OPT     opt,    /* 输入：延时类型与范围选项 */
    OS_ERR    *p_err    /* 输出：错误码指针 */
)

3.函数描述

该函数使当前任务延时执行，直到满足系统 tick 条件（由 时-分-秒-毫秒 组合而成的时间换算而来）。支持四种延时模式：相对延时、超时等待、周期性唤醒、绝对时间匹配。

4.参数说明

(1) hours

类型：CPU_INT16U

作用：延时的小时数；在 OS_OPT_TIME_HMSM_STRICT 模式下必须 0–99，在OS_OPT_TIME_HMSM_NON_STRICT 模式下允许 0–999。

(2) minutes

类型：CPU_INT16U

作用：延时的分钟数；在 OS_OPT_TIME_HMSM_STRICT 模式下必须 0–59，在OS_OPT_TIME_HMSM_NON_STRICT 模式下允许 0–9999。

(3) seconds

类型：CPU_INT16U

作用：延时的秒数；在 OS_OPT_TIME_HMSM_STRICT 模式下必须 0–59。

(4) milli

类型：CPU_INT32U

作用：延时的毫秒数；在 OS_OPT_TIME_HMSM_STRICT 模式下必须 0–999。

(5) opt

类型：OS_OPT

作用：延时类型与范围控制位掩码：

• OS_OPT_TIME_DLY、OS_OPT_TIME_TIMEOUT、OS_OPT_TIME_MATCH、OS_OPT_TIME_PERIODIC 用于选择唤醒方式；

• OS_OPT_TIME_HMSM_STRICT（默认）或 OS_OPT_TIME_HMSM_NON_STRICT 用于控制时间范围检查。

(6) p_err

类型：OS_ERR*

作用：返回的错误码：

OS_ERR_NONE — 延时成功；
OS_ERR_OPT_INVALID — opt 非法；
OS_ERR_OS_NOT_RUNNING — 内核未运行；
OS_ERR_SCHED_LOCKED — 调度器被锁；
OS_ERR_TIME_DLY_ISR — 中断中调用；
OS_ERR_TIME_INVALID_HOURS — hours 超限；
OS_ERR_TIME_INVALID_MINUTES — minutes 超限；
OS_ERR_TIME_INVALID_SECONDS — seconds 超限；
OS_ERR_TIME_INVALID_MILLISECONDS — milli 超限；
OS_ERR_TIME_ZERO_DLY — 计算后 tick 为 0；
OS_ERR_TICK_DISABLED — 系统节拍功能未启用。

5.函数算法逻辑

(1) 检查是否在中断环境中调用

若 OS_CFG_CALLED_FROM_ISR_CHK_EN功能启用且 OSIntNestingCtr > 0u，返回错误码OS_ERR_TIME_DLY_ISR。

(2) 检查内核是否处于运行状态

若 OS_CFG_INVALID_OS_CALLS_CHK_EN功能启用且 OSRunning != OS_STATE_OS_RUNNING，返回错误码OS_ERR_OS_NOT_RUNNING。

(3) 检查调度器是否被锁定

若 OSSchedLockNestingCtr > 0u，返回错误码 OS_ERR_SCHED_LOCKED。

(4) 检查延时选项有效性

提取opt中的唤醒方式，必须为以下之一：

  1. OS_OPT_TIME_DLY
  2. OS_OPT_TIME_TIMEOUT
  3. OS_OPT_TIME_MATCH
  4. OS_OPT_TIME_PERIODIC

若非以上四种方式，则返回错误码OS_ERR_OPT_INVALID。

(5) 检查时间值范围

若启用 OS_CFG_ARG_CHK_EN：

  1. 检查输入的opt的是否包含除了OS_OPT_TIME_OPTS_MASK之外的其他标志位，如若有，则任务是非法选项，返回OS_ERR_OPT_INVALID
  2. 从opt中判断当前是OS_OPT_TIME_HMSM_NON_STRICT（严格模式）还是OS_OPT_TIME_HMSM_NON_STRICT（非严格模式）
  3. • 严格模式：
  4. • • 若小时 > 99，返回错误码OS_ERR_TIME_INVALID_HOURS
  5. • • 若分钟 > 59，返回错误码OS_ERR_TIME_INVALID_MINUTES
  6. • • 若秒 >59，返回错误码OS_ERR_TIME_INVALID_SECONDS
  7. • • 若毫秒 >999，返回错误码OS_ERR_TIME_INVALID_MILLISECONDS
  8. • 非严格模式
  9. • • 小时 >999，返回错误码OS_ERR_TIME_INVALID_HOURS
  10. • • 分钟 > 9999，返回错误码OS_ERR_TIME_INVALID_MINUTES

(6) 计算总 tick 数

根据OSCfg_TickRate_Hz将输入的时/分/秒/毫秒转换为系统节拍 tick， 毫秒转换时采用四舍五入

(7) 插入延时列表并调度

  1. 进入临界区
  2. 调用 OS_TickListInsertDly() 插入当前任务
  3. 若插入失败则退出临界区并返回OS_TickListInsertDly() 返回的错误码
  4. 调用 OS_RdyListRemove() 移除当前任务
  5. 退出临界区
  6. 调用 OSSched() 进行任务调度

6. 返回值

无

7.健康管理错误码

无
