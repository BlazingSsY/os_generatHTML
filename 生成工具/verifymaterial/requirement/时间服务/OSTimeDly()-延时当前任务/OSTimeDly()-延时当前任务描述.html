需求描述

需求编号：HLR_OS_08_04

工作描述：HLR:uC/OS-Ⅲ应提供延时当前任务的接口

是否需求：是

验证方法：测试验证

补充信息：

1.函数名称

OSTimeDly() -- 延时当前任务

2.函数框架

void OSTimeDly (
    OS_TICK dly, /* 输入：延时时间（节拍数） */
    OS_OPT opt, /* 输入：延时选项 */
    OS_ERR *p_err /* 输出：错误码指针 */
)

3.函数描述

该函数使当前任务延时执行，直到满足指定的系统 tick 条件。支持四种延时模式：相对延时、超时等待、周期性唤醒、绝对时间匹配。

4.参数说明

(1) dly

类型：OS_TICK

作用： 延时时间，单位为系统节拍Tick

(2) opt

类型：OS_OPT

作用：执行的延时类型：

OS_OPT_TIME_DLY 
OS_OPT_TIME_TIMEOUT
OS_OPT_TIME_PERIODIC
OS_OPT_TIME_MATCH

(3) p_err

类型：OS_ERR*

作用：存储返回的错误码

OS_ERR_NONE :无错误，调用成功，延时已发生
OS_ERR_OPT_INVALID :输入的opt是非法的
OS_ERR_OS_NOT_RUNNING:uC/OS没有启动
OS_ERR_SCHED_LOCKED :任务调度器被锁
OS_ERR_TIME_DLY_ISR :尝试在中断服务例程中调用该函数
OS_ERR_TIME_ZERO_DLY :输入的dly为0
OS_ERR_TICK_DISABLED :系统节拍功能未启用

5.函数算法逻辑

(1) 检查是否在中断环境中调用

若 OS_CFG_CALLED_FROM_ISR_CHK_EN 功能启用且 OSIntNestingCtr > 0u，将错误码 OS_ERR_TIME_DLY_ISR 存入 p_err 并返回。

(2) 检查内核是否处于运行状态

若 OS_CFG_INVALID_OS_CALLS_CHK_EN 功能启用且 OSRunning != OS_STATE_OS_RUNNING，将错误码 OS_ERR_OS_NOT_RUNNING 存入 p_err 并返回。

(3) 检查调度器是否被锁定

若 OSSchedLockNestingCtr > 0u，将错误码 OS_ERR_SCHED_LOCKED 存入 p_err 并返回。

(4) 检查延时选项的有效性

检查 opt 是否为以下有效值之一：

  1. OS_OPT_TIME_DLY
  2. OS_OPT_TIME_TIMEOUT
  3. OS_OPT_TIME_PERIODIC
  4. OS_OPT_TIME_MATCH

若非以上四种之一，将错误码 OS_ERR_OPT_INVALID 存入 p_err 并返回。

(5) 将任务插入延时列表并执行任务调度

  1. 进入临界区
  2. 调用 OS_TickListInsertDly() 将当前任务插入延时列表
  3. 若插入失败（OS_TickListInsertDly函数的返回码不为 OS_ERR_NONE），退出临界区并返回
  4. 调用 OS_RdyListRemove() 将当前任务从就绪列表中移除
  5. 退出临界区
  6. 调用 OSSched() 进行任务调度

6. 返回值

无

7.健康管理错误码

无
