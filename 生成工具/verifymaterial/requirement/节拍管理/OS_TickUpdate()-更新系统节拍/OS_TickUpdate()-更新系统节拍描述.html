需求描述

需求编号：HLR_OS_09_02

工作描述：HLR:uC/OS-Ⅲ应提供更新系统节拍的接口

是否需求：是

验证方法：测试验证

补充信息：

1.函数名称

OS_TickUpdate() — 系统节拍更新

2.函数框架

void OS_TickUpdate (
    OS_TICK ticks /* 输入：要增加的节拍数 */
)

3.函数描述

该函数用于更新系统全局节拍计数器 OSTickCtr，并更新延时列表。

4.参数说明

(1) ticks

类型：OS_TICK

作用：要增加的系统节拍数，通常传入 1。

5.函数算法逻辑

  1. 进入临界区
  2. 累加系统节拍计数器
  3. 更新延时列表
  4. 退出临界区

6. 返回值

无

7.健康管理错误码

无
