需求描述

需求编号：LLR_OS_09_06_02

需求描述：节拍链表不为空

    1. 更新头部任务节点

  1. 若当前任务节点剩余节拍 ≤ 输入ticks，将ticks减去当前节点的剩余节拍，并将当前节点的剩余节拍其清0
  2. 若当前任务节点的剩余节拍>输入ticks，将当前节点的剩余节拍减去输入ticks

    2. 遍历所有任务节点，检查是否到期并处理

  1. 循环遍历检查当前任务节点是否到期，若到期，退出循环，若不到期，进入如下循环处理:
  2.  • 跟据当前任务节点的任务状态进行处理：
  3.  •  • 若为OS_TASK_STATE_DLY，将当前节点的任务状态置为OS_TASK_STATE_RDY，并将其插入就绪列表
  4.  •  • 若为OS_TASK_STATE_DLY_SUSPENDED，将当前节点的任务状态置为OS_TASK_STATE_SUSPENDED
  5.  •  • 若为其他状态，进行如下处理：
  6.  •  •  • 如果OS_CFG_MUTEX_EN功能启用且当前等待的是互斥量，从互斥量中提取持有者
  7.  •  •  • 如果OS_MSG_EN功能启用，将当前任务节点的消息指针和消息大小设置为0
  8.  •  •  • 将当前任务节点从等待列表中删除
  9.  •  •  • 根据当前任务节点的任务状态进行进一步处理：
  10.  •  •  •  • 若为OS_TASK_STATE_PEND_TIMEOUT，将当前任务插入就绪列表，并将置为OS_TASK_STATE_RDY
  11.  •  •  •  • 若为OS_TASK_STATE_PEND_TIMEOUT_SUSPENDED，并将置为OS_TASK_STATE_SUSPENDED
  12.  •  •  •  • 若为其他状态，不做任何处理
  13.  •  •  •  将当前任务的等待状态设置为OS_STATUS_PEND_TIMEOUT，等待事件为OS_TASK_PEND_ON_NOTHING
  14.  •  •  •  如果OS_CFG_MUTEX_EN功能启用且互斥量持有者不为空：
  15.  •  •  •  • 如果互斥量持有者的优先级不等于基础优先级且互斥量持有者优先级等于当前任务节点的优先级：
  16.  •  •  •  •  •  从互斥量持有者的互斥量组中提取最高优先级
  17.  •  •  •  •  •  如果该最高优先级低于互斥量持有者的基础优先级：
  18.  •  •  •  •  •  •  将该最高优先级设置为互斥量持有者的基础优先级  
  19.  •  •  •  •  •  如果该最高优先级不等于互斥量持有者的当前优先级：
  20.  •  •  •  •  •  •  更改互斥量持有者的优先级为该最高优先级
  21.  •  获取当前任务节点的下一个节点，将其更新为延时列表的头节点和当前任务节点
  22.  •  如果当前任务节点为空，退出循环，反之，则进行如下处理：
  23.  •   •  将当前任务节点的前向节点设置为空
  24.  •   •  若当前任务节点剩余节拍 ≤ 输入ticks，将ticks减去当前节点的剩余节拍，并将当前节点的剩余节拍其清0
  25.  •   •  若当前任务节点的剩余节拍>输入ticks，将当前节点的剩余节拍减去输入ticks
  26.  •  继续下一次循环
