需求描述

需求编号：HLR_OS_09_04

工作描述：HLR:uC/OS-Ⅲ应提供把延时任务插入延时列表的接口

是否需求：是

验证方法：测试验证

补充信息：

1.函数名称

OS_TickListInsertDly() - 将延时任务插入延时列表

2.函数框架

void OS_TickListInsertDly (
    OS_TCB *p_tcb, /* 输入：待延时任务控制块指针 */
    OS_TICK time, /* 输入：延时时间（tick）或目标 tick 值 */
    OS_OPT opt, /* 输入：延时类型 */
    OS_ERR *p_err /* 输出：错误代码 */
)

3.函数描述

该函数根据函数输入计算经过时间、延时基准，并以此通过调用OS_TickListInsert函数将目标任务插入延时列表

4.参数说明

(1) p_tcb

类型：OS_TCB*

作用：指向需要延时的任务控制块。

(2) time

类型：OS_TICK

作用：

• 相对模式：需要延迟的节拍数。

• 绝对模式： 匹配的系统节拍数。

• 周期模式：周期延迟节拍数。

(3) opt

类型：OS_OPT

作用：延时类型：

• OS_OPT_TIME_DLY

• OS_OPT_TIME_TIMEOUT

• OS_OPT_TIME_PERIODIC

• OS_OPT_TIME_MATCH

(4) p_err

类型：OS_ERR*

作用：返回错误码：

• OS_ERR_NONE — 插入成功
• OS_ERR_TIME_ZERO_DLY — 实际延迟为 0


5.函数算法逻辑

(1) 根据 opt 计算延时基准 

      1)设置已流逝的节拍数为0(下一步计算时使用)

      2)根据 opt 计算延时基准 

  1. 若opt为OS_OPT_TIME_MATCH，设置延时基准为0
  2. 若opt为OS_OPT_TIME_PERIODIC：
  3.  • 若输入参数time为0，返回错误码OS_ERR_TIME_ZERO_DLY
  4.  • 使用目标任务的TickCtrPrev作为延时基准，并计算系统节拍计数和延时基准的差值，如果该差值不小于输入的time：
  5.  •  • 更新延时基准，将差值按照time为单位向下取整后加到延时基准上，将更新后的延时基准赋值给目标任务的TickCtrPrev
  6.  • 更新目标任务的TickCtrPrev，将TickCtrPrev加上time
  7. 若opt为其他模式（OS_OPT_TIME_DLY/OS_OPT_TIME_TIMEOUT），设置延时基准为系统节拍计数

(3) 插入延时链表

  1. 使用计算出的经过时间、延时基准和输入time调用内部 OS_TickListInsert()将目标任务插入延时列表，判断函数返回值：
  2.  • 若返回值为OS_TRUE，设置任务状态为OS_TASK_STATE_DLY，错误码为OS_ERR_NONE
  3.  •若返回值为其他值，设置错误码为OS_ERR_TIME_ZERO_DLY

6. 返回值

无

7.健康管理错误码

无
