需求描述

需求编号：HLR_OS_09_03

工作描述：HLR:uC/OS-Ⅲ应提供把任务插入延时列表的接口

是否需求：是

验证方法：测试验证

补充信息：

1.函数名称

OS_TickListInsert() — 把任务插入Tick延时列表

2.函数框架

CPU_BOOLEAN OS_TickListInsert (
    OS_TCB *p_tcb, /* 输入：待插入任务控制块 */
    OS_TICK elapsed, /* 输入：已过去的节拍数（固定为 0） */
    OS_TICK tick_base, /* 输入：延时的基准节拍值（通常为 OSTickCtr） */
    OS_TICK time /* 输入：任务还需要等待的节拍数 */
)

3.函数描述

该函数将任务按剩余节拍升序插入延时列表，延时列表的头部始终指向剩余节拍最小的任务。若剩余节拍为 0，则直接返回 FALSE，让调用方立即处理；插入成功返回 TRUE。。

4.参数说明

(1) p_tcb

类型：OS_TCB*

作用：指向待插入的任务控制块

(2) elapsed

类型：OS_TICK

作用：已流逝的节拍数， 固定传入 0。

(3) tick_base

类型：OS_TICK

作用：延时的基准节拍值（通常为当前 OSTickCtr）。

(4) time

类型：OS_TICK

作用：延时或周期长度。

5.函数算法逻辑

(1) 计算任务到期的剩余节拍delta

剩余节拍delta为输入time加上延时基准减去系统节拍计数和经过时间，若delta为0，将目标任务的TickRemain设置为0，并返回OS_FALSE

(2) 检查延时列表是否为空

若延时列表为空：

• 把任务设为延时列表的头部

• 任务的前后指针TickNextPtr 和 TickPrevPtr设置 为 NULL。

• 任务的剩余时间TickRemain设置为delta

• 返回 OS_TRUE。

(3) 插入任务到延时列表头部

若 delta < 延时列表头部任务的剩余节拍：

• 若该任务无前向指针：

•  • 调整原头节点的剩余节拍为 原剩余节拍减去delta。

•  • 把目标任务的剩余节拍TickRemain设置为delta

•  • 把目标任务插入头部，更新目标任务和原头部任务的相关指针。

•  • 返回 OS_TRUE。

• 若该任务有前向指针：

• • 返回 OS_FLASE。

(4) 插入延时列表中间或末尾

• 遍历链表，累计剩余节拍，直到找到首个 delta < 节点剩余节拍的任务节点或者遍历结束(插入末尾)

• 若找到delta < 节点剩余节拍的任务节点

• • 将目标任务插入该节点的前部，设置目标任务的剩余时间为delta，设置后续节点的剩余时间减去delta，

• 若遍历结束(插入末尾)

• •将目标任务插入延时列表末尾，并设置的其剩余时间为delta

• 返回 OS_TRUE。

6. 返回值

类型：CPU_BOOLEAN

作用：

• OS_TRUE — 任务成功插入延时链表

• OS_FALSE — 剩余时间为 0，任务无需延时

7.健康管理错误码

无
