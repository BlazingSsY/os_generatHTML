需求描述

需求编号： HLR_OS_11_11

工作描述：HLR：uC/OS-III应提供空闲任务入口函数

是否需求：是

验证方法：测试验证

1.函数名称

OS_IdleTask ()-空闲任务入口函数

2.函数框架

void OS_IdleTask (
    void *p_arg   /*输入：
)

3.函数描述

该函数是空闲任务的入口函数，即空闲任务函数，当空闲任务获取系统执行权时，该函数将被执行

4.参数说明

（1）p_arg

类型：void

作用：任务通用参数，但空闲任务通常不需要参数，此处强制类型转换避免未使用变量的警告。

5.算法逻辑

 • 检查是否启用了调试（OS_CFG_DBG_EN>0）或统计任务（OS_CFG_STAT_TASK_EN>0）。若启用了，则分配CPU状态寄存器空间为临界区做准备；若未启用，不做任何处理

 •无限循环处理。

 •  •检查是否启用了调试（OS_CFG_DBG_EN>0）或统计任务（OS_CFG_STAT_TASK_EN>0）。若未启用，不做任何处理； 若启用了，则进入临界区，禁用中断并作进一步处理：

     •  •  •若启用了调试（OS_CFG_DBG_EN>0），空闲任务计数器OSIdleTaskCtr加1；若未启用，不做任何处理

     •  •  •若启用统计任务（OS_CFG_STAT_TASK_EN>0），统计任务计数器OSStatTaskCtr数值加1；若未启用，不做任何处理

     •  •  •退出临界区 

 •  •若用户钩子函数功能启用（OS_CFG_APP_HOOKS_EN>0）， 将执行用户自定义的钩子函数，若未启用，不做任何处理。

6.返回值

无

7.健康管理错误码

无

   备注：本函数为 uC/OS-III 的内部处理函数，应用程序不应直接调用。必须启用空闲任务功能OS_CFG_TASK_IDLE_EN > 0 才会编译此函数。
