需求描述

需求编号：HLR_OS_11_17

工作描述：HLR：uC/OS-Ⅲ应提供将任务按优先级插入等待链表的接口。

是否需求：是

验证方法：测试验证

补充信息：

1.函数名称

OS_PendListInsertPrio() – 将任务按优先级插入等待链表

2.函数框架

void OS_PendListInsertPrio (
    OS_PEND_LIST *p_pend_list, /* 输入：等待链表指针 */
    OS_TCB *p_tcb /* 输入：待插入的任务控制块 */
)

3.函数描述

该函数用于将任务控制块按其优先级顺序插入指定的等待列表。任务优先级越高，插入位置越靠前（即优先级值越小，排在越前面）。

4.参数说明

（1）p_pend_list

  1.  类型：OS_PEND_LIST*
  2.  作用：指向挂起等待链表的结构体指针。
  3.  注意事项：必须为有效指针，调用前应确保其已初始化。

（2）p_tcb

  1.  类型：OS_TCB*
  2.  作用：指向待插入的任务控制块，其 Prio 字段用于确定插入位置。
  3.  注意事项：必须为有效指针，且其 PendNextPtr 与 PendPrevPtr 成员未处于其他链表中。

5.算法逻辑

  1. 获取任务优先级
  2. 判断链表是否为空
  3.  •若等待链表为空，将指定任务的前向等待指针和后向等待指针置空，等待列表的头指针和尾指针都指向指定任务控制块。
  4.  •若等待链表不为空，遍历等待列表中的任务，直到出现任务的优先级低于指定任务的优先级或到列表末尾
  5.  •  •若遍历到等待列表末尾，则将指定任务的后向等待指针置空，前向等待指针指向等待列表尾部任务，等待列表尾部任务的后向等待指针指向指定任务，等待列表的尾指针指向指定任务。
  6.  •  • 若遍历到等待列表开头，则将指定任务的前向等待指针置空，后向等待指针指向等待列表头部任务，等待列表头部任务的前向等待指针指向指定任务，等待列表的头指针指向指定任务。
  7.  •  • 否则，按照遍历任务的前向等待指针的指向任务，指定任务，遍历任务的顺序设置这三个任务的前后等待指针指向。

6.返回值

无。

7.健康管理错误码

无。

备注：此函数为 uC/OS-III 的内部函数，应用程序不得调用它。
