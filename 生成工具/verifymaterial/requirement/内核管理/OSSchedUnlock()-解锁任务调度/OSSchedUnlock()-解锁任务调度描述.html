需求描述

需求编号：HLR_OS_11_06

工作描述：HLR：uC/OS-III 应提供解锁任务调度的接口，用于恢复任务调度

是否需求： 是

验证方法： 测试验证

1. 函数名称

OSSchedUnlock() – 解锁任务调度

2. 函数框架

void OSSchedUnlock(
      OS_ERR *p_err /* 输出：错误码指针 */
)

3. 函数描述

该函数用于解锁任务调度，使系统恢复正常的任务切换操作。它与 OSSchedLock() 配套使用，必须严格一一对应。每调用一次OSSchedUnlock() ，嵌套计数器将减一，当嵌套计数归零时，任务调度被真正解锁，并立即触发一次任务调度，允许更高优先级的任务运行。

4. 参数说明

（1）p_err

  1. 类型：OS_ERR*
  2. 作用：返回函数执行状态的错误码
  3. 注意事项：不能为 NULL，必须指向有效内存
  4. 错误码：
OS_ERR_NONE：调度器成功解锁，且已执行任务调度
OS_ERR_OS_NOT_RUNNING：系统未运行，无法解锁
OS_ERR_SCHED_NOT_LOCKED：调度器未处于锁定状态
OS_ERR_SCHED_LOCKED：调度器解锁未完成，嵌套计数器仍大于0
OS_ERR_SCHED_UNLOCK_ISR：在中断服务程序中调用该函数

5. 算法逻辑

1）中断调用检查

 若OS_CFG_CALLED_FROM_ISR_CHK_EN功能启用，如果中断嵌套计数器OSIntNestingCtr > 0，表明在中断服务程序中调用该函数，则返回 OS_ERR_SCHED_UNLOCK_ISR。

2）内核运行状态检查

 若内核处于停止态，即OSRunning != OS_STATE_OS_RUNNING，则返回 OS_ERR_OS_NOT_RUNNING。

3）检查是否已锁定任务调度

 若锁定嵌套计数器OSSchedLockNestingCtr == 0，说明任务调度未处于锁定状态，则返回 OS_ERR_SCHED_NOT_LOCKED。

4）递减锁定嵌套计数器

  1. 进入临界区
  2. 递减锁定嵌套计数器OSSchedLockNestingCtr，减少锁定层数
  3. 判断锁定嵌套计数是否仍大于0，若是，则说明调度器尚未完全解锁，退出临界区并返回 OS_ERR_SCHED_LOCKED
  4. 退出临界区

5）执行任务调度

  1. 调用 OSSched() 执行任务调度
  2. 设置返回状态为 OS_ERR_NONE

6. 返回值

无。

7. 健康管理错误码

  1. 无
