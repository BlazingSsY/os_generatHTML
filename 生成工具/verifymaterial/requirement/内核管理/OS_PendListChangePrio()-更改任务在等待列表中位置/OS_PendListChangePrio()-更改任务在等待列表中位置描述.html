需求描述

需求编号： HLR_OS_11_15

工作描述： HLR：uC/OS-III 应提供更改任务在等待列表中的位置的接口。

是否需求： 是

验证方法： 测试验证

1. 函数名称

OS_PendListChangePrio() – 更改任务在等待列表中的位置

2. 函数框架

void OS_PendListChangePrio (
    OS_TCB *p_tcb /* 输入：指向待调整位置的任务控制块 */
)

3. 函数描述

该函数用于在任务的优先级发生改变时更改其在等待列表中的位置。先将任务从当前等待列表中移除，然后根据任务控制块中的新优先级重新插入到等待列表中。

4. 参数说明

（1）参数名称：p_tcb

  1. 类型：OS_TCB*
  2. 作用：指向需要调整位置的任务控制块
  3. 注意事项：在调用该函数前，需要注意p_tcb.Prio已经更新为目标优先级

5. 算法逻辑

  1. 获取任务当前等待对象的指针和该对象对应的等待列表。
  2. 判断该等待列表中是否有多个任务（两个或两个以上）：
  3.   •若有多个任务，则将指定任务从等待列表中移除，重新设置指定任务的等待对象，再将指定任务按照优先级重新插入到对应等待列表中
  4.  •如果只有一个任务，函数退出，不做处理

6. 返回值

无。

7. 健康管理错误码

无

备注：

 此函数是 uC/OS-III 内部使用函数，应用程序不得直接调用。函数假定任务控制块中的.Prio字段已存放新的优先级。
