需求描述

需求编号： HLR_OS_11_14

工作描述： HLR：uC/OS-III 应提供终止任务等待的接口。

是否需求： 是

验证方法： 测试验证

1. 函数名称

OS_PendAbort() – 终止任务等待

2. 函数框架

void OS_PendAbort (
    OS_TCB *p_tcb, /* 输入：指向将被终止等待的任务控制块 */
    CPU_TS ts, /* 输入：终止等待时的时间戳 */
    OS_STATUS reason /* 输入：终止等待的原因 */
)

3. 函数描述

该函数由 OSxxxPendAbort() 和 OSxxxDel() 等服务调用，用于终止任务的等待状态，将任务从等待列表移除并根据不同状态更新任务的状态。终止等待后，任务被重新放入就绪列表或保持挂起态（如等待状态+挂起态），并设置任务终止等待的原因。

4. 参数说明

（1）参数名称：p_tcb

  1. 类型：OS_TCB*
  2. 作用：指向待终止等待状态的任务控制块

（2）参数名称：ts

  1. 类型：CPU_TS
  2. 作用：终止等待状态的时间戳（系统时间）

（3)参数名称：reason

  1. 类型：OS_STATUS
  2. 作用：任务终止等待状态的原因，取值为OS_STATUS_PEND_DEL或OS_STATUS_PEND_ABORT

5. 算法逻辑

1）根据目标任务状态进行分支处理：

  1. 若任务处于等待或等待超时态：
  2. • 若启用消息队列功能，则设置任务控制块的消息地址为空和消息大小为0；
  3. • 将任务从等待列表中移除；
  4. • 若OS_CFG_TICK_EN功能启用，且任务处于等待超时态，则将任务从延时列表中移除；
  5. • 将任务插入就绪列表中，更新任务的状态为就绪态，设置任务的等待状态为输入的reason，任务的等待对象为OS_TASK_PEND_ON_NOTHING，退出。
  6. 若任务处于等待挂起或等待超时挂起态：
  7. • 若启用消息队列功能，则设置任务控制块的消息地址为空和消息大小为0；
  8. • 将任务从等待列表中移除；
  9. • 若OS_CFG_TICK_EN功能启用，且任务处于等待超时挂起态，则将任务从延时列表中移除；
  10. • 更新任务的状态为挂起态，设置任务的等待状态为输入的reason，任务的等待对象为OS_TASK_PEND_ON_NOTHING，退出。
  11. 若任务处于就绪、延时、挂起、延时挂起状态或其他状态，不处理，直接退出。

6. 返回值

无。

7. 健康管理错误码

无

备注：

 此函数是 uC/OS-III 内部使用函数，应用程序不得直接调用。
