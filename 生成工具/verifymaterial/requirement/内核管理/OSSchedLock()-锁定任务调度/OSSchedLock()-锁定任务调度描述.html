需求描述

需求编号：HLR_OS_11_05

工作描述：HLR：uC/OS-III 应提供锁定任务调度功能，以允许用户临时禁止任务切换。

是否需求： 是

验证方法： 测试验证

1. 函数名称

OSSchedLock() – 锁定任务调度

2. 函数框架

void OSSchedLock(
       OS_ERR *p_err   /* 输出： 错误码指针*/
    )

3. 函数描述

该函数用于锁定任务调度，使当前任务可以在不被抢占的情况下执行一段临界代码。当调度器被锁定时，即使有更高优先级任务就绪，系统也不会进行上下文切换。调度锁定是可嵌套的，每调用一次OSSchedLock() ，锁定嵌套计数器将加一，但最多只能嵌套 250 层。锁定任务调度后，必须调用 OSSchedUnlock() 来解除锁定。

4. 参数说明

（1）p_err

  1. 类型：OS_ERR*
  2. 作用：用于返回函数执行结果的错误码。

    错误码：

OS_ERR_NONE：调度器成功锁定。
OS_ERR_SCHED_LOCK_ISR：在中断服务程序中调用了此函数。
OS_ERR_OS_NOT_RUNNING：内核尚未运行。
OS_ERR_LOCK_NESTING_OVF：调度器锁嵌套次数超过上限（> 250）。

5. 算法逻辑

1）中断调用检查

 若OS_CFG_CALLED_FROM_ISR_CHK_EN功能启用，如果中断嵌套计数器OSIntNestingCtr > 0，表明在中断服务程序中调用该函数，则返回错误码OS_ERR_SCHED_LOCK_ISR。

2）内核运行状态检查

 若内核处于停止态，即OSRunning != OS_STATE_OS_RUNNING，则返回错误码OS_ERR_OS_NOT_RUNNING。

3）检查锁定嵌套是否超限

 若锁定嵌套计数器OSSchedLockNestingCtr >=  250，说明锁定嵌套层数已经超限，则返回错误码OS_ERR_LOCK_NESTING_OVF。

4）递增锁定嵌套计数器

  1. 进入临界区
  2. 锁定嵌套计数器OSSchedLockNestingCtr加一
  3. 退出临界区
  4. 返回错误码OS_ERR_NONE

6. 返回值

无。

7. 健康管理错误码

  1. 无

备注

  1. OSSchedLock() 与 OSSchedUnlock() 必须配对使用，否则将导致调度器永久锁定，系统任务无法切换。
  2. 锁定调度器仅影响任务级上下文，不会阻止中断的执行。
  3. 调度器锁定用于保证关键代码段执行的原子性，但不应过度使用，否则会降低系统实时性。
  4. 建议在锁定期间尽可能缩短操作时间，避免延迟任务响应。
