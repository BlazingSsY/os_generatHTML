需求描述

需求编号：HLR_OS_11_19

工作描述：HLR：uC/OS-Ⅲ应提供唤醒等待任务的接口

是否需求：是

验证方法：测试验证

补充信息：

1.函数名称

OS_Post() – 唤醒正在等待某个资源的等待任务

2.函数框架

void OS_Post (
    OS_PEND_OBJ *p_obj, /* 输入：等待的内核对象（可为 NULL） */
    OS_TCB *p_tcb, /* 输入：被唤醒的任务的任务控制块指针*/
    void *p_void, /* 输入：消息指针（可选） */
    OS_MSG_SIZE msg_size, /* 输入：消息大小（可选） */
    CPU_TS ts /* 输入：发送发生的时间戳 */
)

3.函数描述

 该函数用于唤醒因等待某个资源而处于等待状态的任务， 标志其等待资源已可用，其作为多个 OSxxxPost() 内核服务的通用底层实现，完成对任务状态的检查、消息发送、等待状态清除、链表移除以及任务重新就绪等操作。 如果目标任务处于等待状态，则完成任务恢复并将其加入就绪队列；若任务处于就绪、延迟或挂起状态，则不执行任何操作。该函数为内核内部使用，用户不应直接调用。

4.参数说明

（1）p_obj

  1.  类型：OS_PEND_OBJ*
  2.  作用：指向等待对象（如信号量、消息队列等）的指针。若无对象，可为 NULL。

（2）p_tcb

  1.  类型：OS_TCB*
  2.  作用：指向目标任务控制块，任务将接收事件或消息。
  3.  注意事项：必须为有效指针，任务需处于等待状态。

（3）p_void

  1.  类型：void*
  2.  作用：指向发送给任务的消息。仅在启用消息队列时有效。

（4）msg_size

  1.  类型：OS_MSG_SIZE
  2.  作用：消息的长度（字节数）。仅在启用消息队列时有效。

（5）ts

  1.  类型：CPU_TS
  2.  作用：事件发送发生的时间戳。仅在启用时间戳功能时有效。

5.算法逻辑

1）检查任务状态

  1. 若任务处于就绪、延时、挂起或延时挂起状态，直接忽略，不处理，退出。
  2. 若任务处于等待或等待超时态：
  3. • 若启用消息队列功能，则设置任务控制块的消息地址和消息大小为对应的输入参数；
  4. • 若等待对象不为空，则将任务从等待对象的等待列表中移除；
  5. • 若任务处于等待超时态，则将任务从延时列表中移除；
  6. • 将任务插入就绪列表中，更新任务的状态为就绪态，设置任务的等待状态为OS_STATUS_PEND_OK，任务的等待对象为OS_TASK_PEND_ON_NOTHING，退出。
  7. 若任务处于等待挂起或等待超时挂起态：
  8. • 若启用消息队列功能，则设置任务控制块的消息地址和消息大小为对应的输入参数；
  9. • 若发送对象不为空，则将任务从发送对象的等待列表中移除；
  10. • 若任务处于等待超时挂起态，则将任务从延时列表中移除；
  11. • 更新任务的状态为挂起态，设置任务的等待状态为OS-STATUS_PEND_OK，任务的等待对象为OS_TASK_PEND_ON_NOTHING，退出。
  12. 其他状态，不处理，退出

6.返回值

无。

7.健康管理错误码

无。

备注：该函数是 uC/OS-III 的内部函数，应用程序不得直接调用它。实际使用应通过如 OSSemPost()、OSQPost() 等对外接口间接触发。
