需求描述

需求编号：HLR_OS_11_01

工作描述：

 HLR：uC/OS-III应提供初始化内核数据结构和功能模块接口，为后续任务调度和系统启动做好准备。

是否需求： 是

验证方法： 测试验证

1. 函数名称

OSInit() – 初始化内核数据结构和功能模块

2. 函数框架

void OSInit (
    OS_ERR *p_err /* 输出：错误码指针 */
)

3. 函数描述

该函数用于初始化uC/OS-III的内核的数据结构和功能模块，是内核运行前必须调用的函数。它完成以下对象的初始化：中断嵌套计数器、内核运行状态、调度器状态、任务控制块指针、优先级管理参数、时间片轮询管理参数、中断堆栈、应用钩子函数指针、优先级表、就绪任务链表、内存管理、消息池、任务管理、空闲任务、定时器任务、系统节拍等。该函数确保整个内核环境准备就绪，为创建任务及启动多任务调度奠定基础。

4. 参数说明

（1）p_err

  1. 类型：OS_ERR*
  2. 作用：用于返回函数执行的错误代码，指示函数调用是否成功以及失败的原因
  3. 错误码：
OS_ERR_NONE：初始化成功。
other：-----


5. 算法逻辑

1）调用内核初始化钩子函数

 调用 内核初始化钩子函数，用于执行与硬件平台相关的初始化 。

2）初始化内核全局变量

  1. 初始化中断嵌套计数器、内核运行状态、调度器状态、当前任务指针、当前最高优先级任务指针、当前运行任务优先级、当前就绪任务最高优先级
  2. 若OS_CFG_SCHED_ROUND_ROBIN_EN功能启用，则初始化时间片轮询使能标志、系统默认时间片长度

3）中断堆栈初始化

若OS_CFG_ISR_STK_SIZE大于0，则清零中断堆栈区域。

4）初始化应用钩子函数指针 

 若OS_CFG_APP_HOOKS_EN功能启用，则初始化所有应用层钩子函数指针为 NULL，为应用钩子注册做准备。

5）初始化任务寄存器表

 将 OSTaskRegNextAvailID 设为0，为每个任务提供一组可扩展的"寄存器"存储空间。

6）初始化优先级位图表

调用 OS_PrioInit() 初始化优先级位图表

7）初始化就绪任务链表

 调用 OS_RdyListInit() 初始化就绪任务列表。

8）初始化内存管理

若OS_CFG_MEM_EN功能启用，则初始化内存管理器（调用 OS_MemInit()）

如果内存管理初始化失败（返回的错误码不为OS_ERR_NONE），函数立即返回。。

9）初始化消息池

若OS_MSG_EN功能启用，则初化胡消息池（调用 OS_MsgPoolInit()）

如果消息池模块初始化失败（返回的错误码不为OS_ERR_NONE），函数立即返回。

10）初始化任务管理

调用OS_TaskInit()函数初始化任务管理

若初始化失败(返回的错误码不为OS_ERR_NONE)，函数立即返回.

11）初始化空闲任务

调用OS_IdleTaskInit()函数初始化并创建空闲任务

若初始化失败(返回的错误码不为OS_ERR_NONE)，函数立即返回.

12）初始化系统节拍

调用OS_TickInit()函数初始化系统节拍和延时列表

若初始化失败(返回的错误码不为OS_ERR_NONE)，函数立即返回.

13）初始化定时器

调用OS_TmrInit()函数初始化定时器管理并创建定时器

若初始化失败(返回的错误码不为OS_ERR_NONE)，函数将立即返回。

14）初始化配置参数

调用 OSCfg_Init()，初始化OS的静态配置参数。

15)设置初始化完成标志

将 OSInitialized 设为 OS_TRUE，标志内核已完成初始化。

6. 返回值

无。

7. 健康管理错误码

无。

备注

  1. 本函数为uC/OS-III内核启动必须步骤，必须在调用任何其他uC/OS-III API函数前执行。
  2. 此函数一般只在系统启动阶段调用一次。
  3. 此函数为系统内部接口，建议应用程序不要重复调用。
