需求描述

需求编号：HLR_OS_11_08

工作描述：HLR：uC/OS-III 应提供时间片轮转让出的接口，使得当前任务在使用时间片未耗尽时，主动放弃 CPU，让同优先级的其他任务获得运行机会。

是否需求： 是

验证方法： 测试验证

1. 函数名称

OSSchedRoundRobinYield() – 时间片轮转让出

2. 函数原型

void OSSchedRoundRobinYield(
    OS_ERR *p_err   /* 输出： 错误码指针 */
    )

3. 函数描述

该函数用于在时间片轮转调度（Round-Robin）时启用时，当前任务主动让出 CPU，即在未耗尽时间片的情况下主动放弃执行权，让同优先级的其他任务获得运行机会。

4. 参数说明

（1）p_err

  1. 类型：OS_ERR*
  2. 说明：用于返回函数的执行结果的错误码
  3. 错误码：
OS_ERR_NONE：成功让出 CPU
OS_ERR_YIELD_ISR：不能在中断服务程序中调用
OS_ERR_SCHED_LOCKED：调度器处于锁定状态，无法调度
OS_ERR_ROUND_ROBIN_DISABLED：未启用轮转调度功能
OS_ERR_ROUND_ROBIN_1：当前优先级下仅有一个任务，无法让出 CPU

5. 算法逻辑

1）中断调用检查

 若OS_CFG_CALLED_FROM_ISR_CHK_EN功能启用，如果中断嵌套计数器OSIntNestingCtr > 0，表明在中断服务程序中调用该函数，则设置错误码OS_ERR_YIELD_ISR，函数返回。

2）任务调度锁定检查

若锁定嵌套计数器OSSchedLockNestingCtr > 0，表明任务调度被锁定，无法进行任务切换，设置错误码为OS_ERR_SCHED_LOCKED，函数返回。

3）轮转调度功能启用检查

若系统未启用时间片轮转调度功能，即OSSchedRoundRobinEn != OS_TRUE，设置错误码为 OS_ERR_ROUND_ROBIN_DISABLED，函数返回。

4）执行时间片让出操作

  1. 进入临界区
  2. 根据当前任务的优先级获取其所在的就绪链表
  3. 检查当前任务是否为就绪链表的唯一任务，若是则退出临界区，设置错误码为OS_ERR_ROUND_ROBIN_1，函数返回
  4. 将当前任务从就绪链表头部移动至尾部
  5. 更新就绪列表新头部任务的时间片计数器
  6. • 若新头部任务的时间片长度TimeQuanta为0，则设置其时间片计数器为默认时间片长度OSSchedRoundRobinDfltTimeQuanta
  7. • 若新头部任务的时间片长度TimeQuanta不为0，则设置其时间片计数器为时间片长度TimeQuanta
  8. 退出临界区

5）执行任务切换并设置错误码

  1. 调用 OSSched() 实施任务切换，使新任务获得 CPU 执行权
  2. 设置错误码为OS_ERR_NONE，函数返回

6. 返回值

无。

7. 健康管理错误码

  1. 无

备注

  1. 本函数只能在任务上下文中调用，不得在中断上下文中使用
  2. 系统必须启用时间片轮转调度功能（OS_CFG_SCHED_ROUND_ROBIN_EN > 0）
  3. 仅在同一优先级下存在多个就绪任务时，此函数才有实际调度意义
  4. 使用前应确保调度器未锁定，否则让出将被忽略
