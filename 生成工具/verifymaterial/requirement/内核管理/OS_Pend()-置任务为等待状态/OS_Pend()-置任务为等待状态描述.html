需求描述

需求编号： HLR_OS_11_13

工作描述： HLR：uC/OS-III 应提供将任务置为等待状态的接口。

是否需求： 是

验证方法： 测试验证

1. 函数名称

OS_Pend() – 置任务为等待状态

2. 函数框架

void OS_Pend (
    OS_PEND_OBJ *p_obj, /* 输入：指向任务等待的内核对象 */
    OS_TCB *p_tcb, /* 输入：指向将被置为等待状态的任务控制块 */
    OS_STATE pending_on, /* 输入：等待事件的类型 */
    OS_TICK timeout /* 输入：超时时间（单位为时钟节拍数） */
)

3. 函数描述

该函数用于将指定任务置于等待状态，使其等待一个事件的发生。该函数是内部通用等待机制，多个如信号量、队列、互斥锁、条件变量等 OSxxxPend() 服务都会调用它。它将根据输入参数将任务阻塞为等待态或等待超时态，并将任务加入某一等待对象的等待链表中。

4. 参数说明

（1）参数名称：p_obj

  1. 类型：OS_PEND_OBJ*
  2. 作用：指向等待对象（如信号量，互斥量）的指针；如果任务不依赖具体对象，可以为NULL

（2）参数名称：p_tcb

  1. 类型：OS_TCB*
  2. 作用：指向将被置为等待状态的任务控制块

（3）参数名称：pending_on

  1. 类型：OS_STATE
  2. 作用：指示任务正在等待的事件类型（如信号量，互斥量，消息队列等），可选如下值：
OS_TASK_PEND_ON_FLAG
OS_TASK_PEND_ON_TASK_Q
OS_TASK_PEND_ON_MUTEX
OS_TASK_PEND_ON_COND
OS_TASK_PEND_ON_Q
OS_TASK_PEND_ON_SEM
OS_TASK_PEND_ON_TASK_SEM

（4）参数名称：timeout

  1. 类型：OS_TICK
  2. 作用：设置任务等待的最大时长（系统节拍数），为0表示无限期等待

5. 算法逻辑

1）设置等待事件和等待状态并阻塞目标任务

  1. 将目标任务的等待事件设置为输入的 pending_on 参数，设置其等待状态OS_STATUS_PEND_OK
  2. 阻塞目标任务，若输入time大于0，则目标任务进入等待超时态；否则，目标任务进入等待态

2）根据等待对象是否为空分别进行处理：

  1. 若输入的p_obj 不为 NULL，则获取该对象的等待列表，将目标任务的等待对象设置为输入的p_obj，然后将其按优先级顺序插入等待的对象等待链表中。
  2. 若输入的p_obj 为 NULL，则将目标任务的等待对象设置为NULL

6. 返回值

无。

7. 健康管理错误码

无。

备注：

 本函数为 uC/OS-III 内部使用接口，应用程序不得直接调用。所有调用此函数的 API（如 OSSemPend()、OSMutexPend() 等）会统一封装错误码与状态逻辑。
