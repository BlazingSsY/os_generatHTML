需求描述

需求编号： HLR_OS_11_09

工作描述： HLR：uC/OS-III 应提供启动多任务调度的接口。

是否需求： 是

验证方法： 测试验证

1. 函数名称

OSStart() – 启动多任务调度

2. 函数框架

void OSStart (
    OS_ERR *p_err /* 输出：错误码指针 */
)

3. 函数描述

该函数用于启动 uC/OS-III 的任务调度系统。当 OSStart() 被调用时，操作系统进入运行状态，并开始调度运行就绪列表中最高优先级的任务。调用此函数前必须已成功执行 OSInit()，并至少创建一个应用任务。该函数理论上不会返回，如果返回，表示调度器未能成功启动。

4. 参数说明

（1）p_err

  1. 类型： OS_ERR*
  2. 作用： 用于返回函数调用结果的错误码。

错误码：

OS_ERR_OS_NOT_INIT---操作系统尚未初始化
OS_ERR_OS_NO_APP_TASK---没有创建任何用户应用任务
OS_ERR_OS_RUNNING---操作系统已经在运行
OS_ERR_FETAL_RETURN---开启调度成功后不会返回，否则，表示失败，并返回该错误码

5. 算法逻辑

(1)调度开启条件检查

  1. 检查是否初始化
  2.   •若初始化标志OSInitialized != OS_TRUE，表示尚未初始化，设置 *p_err = OS_ERR_OS_NOT_INIT 并返回。


  1. 检查默认应用任务是否全部启动
  2.   •设置内核任务数量的初始值为0
  3.   •若OS_CFG_TMR_EN功能启用，内核任务数量加1
  4.   •若OS_CFG_TASK_IDLE_EN功能启动，内核任务数量加1
  5.   •如果总任务数 OSTaskQty 小于或等于内核任务数量，表示没有任何用户创建的应用任务，设置 *p_err = OS_ERR_OS_NO_APP_TASK 并返回。


2）启动多任务调度

  1. 若 内核运行状态OSRunning != OS_STATE_OS_STOPPED，说明操作系统已经运行，设置 *p_err = OS_ERR_OS_RUNNING 并返回。
  2. 若内核运行状态OSRunning为OS_STATE_OS_STOPPED，则通过以下操作启动任务调度：
  3.   • 通过OS_PrioGetHighest() 获取优先级表中最高优先级，并赋值给就绪任务最高优先级变量OSPrioHighRdy。
  4.   • 设置当前运行任务优先级OSPrioCur为当前就绪任务中的最高优先级OSPrioHighRdy
  5.   • 使用就绪任务中的最高优先级OSPrioHighRdy从对应就绪列表中获取头指针并赋值给最高优先级就绪任务指针OSTCBHighRdyPtr
  6.   • 设置当前任务指针 OSTCBCurPtr为最高优先级就绪任务指针OSTCBHighRdyPtr
  7.   • 设置内核运行状态OSRunning为OS_STATE_OS_RUNNING
  8.   • 调用目标平台相关的调度器启动函数 OSStartHighRdy()，正式进入多任务调度
  9.   • 若 OSStartHighRdy() 返回，则视为调度启动失败，设置 *p_err = OS_ERR_FATAL_RETURN

6. 返回值

无。

7. 健康管理错误码

错误码：

  1. OS_ERR_OS_NOT_INIT，操作系统尚未初始化
  2. OS_ERR_OS_NO_APP_TASK，没有创建任何用户应用任务
  3. OS_ERR_OS_RUNNING，操作系统已经在运行
  4. OS_ERR_FETAL_RETURN，开启调度成功后不会返回，否则，表示失败，并返回该错误码

备注：

 该函数是启动 uC/OS-III 多任务调度的关键入口。应用程序通常在 main() 函数中最后调用 OSStart()，之后程序控制权将被操作系统接管，理论上不会再返回主函数。此函数仅能调用一次，多次调用无效。
