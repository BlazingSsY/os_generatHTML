需求描述

需求编号： HLR_OS_11_26

工作描述： HLR：uC/OS-III 应提供运行时间片轮转调度的接口。

是否需求： 是

验证方法： 测试验证

1. 函数名称

OS_SchedRoundRobin() – 运行时间片轮转调度

2. 函数框架

void OS_SchedRoundRobin (
    OS_RDY_LIST *p_rdy_list /* 输入：指定优先级就绪列表指针 */
)

3. 函数描述

该函数用于在每个系统时钟节拍（tick）触发时，更新任务剩余时间片并根据时间片轮转调度策略决定是否切换当前优先级上的任务。若当前任务的时间片用尽且有多个任务处于相同优先级队列中，则当前任务将被移动到队列尾部，调度下一任务运行。

4. 参数说明

（1）p_rdy_list

类型：OS_RDY_LIST *

作用：指向特定优先级的就绪列表

5. 算法逻辑

1）检查是否启用了时间片轮转功能（OSSchedRoundRobinEn 为 OS_TRUE）。若未启用，则直接返回。

 2）进入临界区，获取输入就绪列表的头部任务 。若该任务为空则退出临界区并返回。

 3）检查头部任务是否为空闲任务（OSIdleTaskTCB），若是，则不参与轮转，退出临界区并返回。

 4）检查头部任务时间片计数器 TimeQuantaCtr，若大于0零，则减 1；若减完后仍大于 0，说明时间片尚未耗尽，退出临界区并返回。

 5）若头部任务为该优先级的就绪列表的唯一任务（头指针与尾指针相同），则无需轮转，退出临界区并返回。

 6）若当前调度被锁定（OSSchedLockNestingCtr > 0），则退出临界区并返回。

 7）执行轮转调度操作：

  1. 调用 OS_RdyListMoveHeadToTail() 将当前任务移至就绪列表尾部；
  2. 设置新的头任务的时间片计数器 TimeQuantaCtr，检查该任务的 TimeQuanta是否为0，若是，则使用默认值 OSSchedRoundRobinDfltTimeQuanta设置TimeQuantaCtr，否则使用TimeQuanta设置TimeQuantaCtr。
  3.  退出临界区。

6. 返回值

无。

7. 健康管理错误码

无。

备注：

 本函数为 uC/OS-III 的内部调度函数，应用程序不应直接调用。必须启用时间片轮转功能 OS_CFG_SCHED_ROUND_ROBIN_EN > 0 才会编译此函数。
